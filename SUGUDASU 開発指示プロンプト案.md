# SUGUDASU 開発指示プロンプト案：`/match-board` と `/slot-board` の完全分離思想

## 0. プロンプトの目的

本ドキュメントは、SUGUDASU（sugudasu.com）における2つの主力ボードツール、`/match-board`（指名重複・ドラフト会議）**と**`/slot-board`（定員超過・引き算パレット）について、その決定的なデータ構造の違い、人間心理の差、および既存UIのバグをメタ視点で整理したものである。
開発LLM（Cursor等）は、本思想を100%インストールした上で実装に臨むこと。

---

## 1. 最上位思想：SUGUDASU憲法（前提条件）

本プロダクト群は、以下の絶対原則に従い、サーバーを介さない完全静的（HTML/JS）な使い捨てツールとして実装する。

* **【F1】登録不要：** 開いたら、アカウントなしで即3秒で使える。
* **【F2】完全非送信：** 1バイトもデータを外部サーバーに送信しない。機内モードでも100%動作する。
* **【F3】完全静的配信：** データベースを持たない。
* **【F4】要塞化の禁止：** データの永続蓄積・ファイル管理は一切行わない。画面を閉じたらすべて消える。
* **【F5】Copy-first：** 手元のExcelからコピペでIN、格闘、結果をTSVコピーしてExcelへ戻り15秒離脱。

---

## 2. メタ認知：2つのツールの決定的な違い

一般ユーザーの脳内では「グループ分け」「割り振り調整」として混同されるが、**データ構造と会議室での人間心理（ベクトルの向き）は完全に真逆**であり、絶対に1つのツールに統合してはならない。

| 論点 | `/match-board`（ドラフト会議） | `/slot-board`（枠取りパレット） |
| --- | --- | --- |
| **ドメイン例** | ゼミ配属、新卒初期配属（本人の希望あり） | 人事評価調整、予算の削り合い、掲載枠争奪 |
| **メタ本質** | **「有限な希少リソースの奪い合い」** | **「有限な檻（キャパシティ）への押し込み」** |
| **データ構造** | $1$ つの要素に対し、複数のレーンが矢印を向ける | $1$ つのレーンに対し、上限数 $N$ のバリデータがある |
| **ベクトルの向き** | **外から内**（複数のレーンが1人を同時に指名する） | **内から外**（箱から溢れた要素を外へ追い出す） |
| **現場の心理** | 「いっせーのせで指名しよう。被ったら抽選だ！」 | 「S枠は3人までだ。溢れた1人を誰に押し付ける？」 |
| **解決手段** | 抽選（確率）、またはトレード交渉 | **妥協（引き算）、他レーンへの島流し・政治的合意** |
| **終了条件** | 全員の所属先が1つずつ決まること | **すべての箱の「🚨超過明滅アラート」がゼロになること** |

## 3. なぜ、現在のUI（既存の match-board）ではダメなのか？

既存の `/match-board` のUI（名簿と枠を並べて「たたき台生成」を押す2カラム画面）には、実務家を硬直させる以下の致命的なバグ（認知の過負荷）が混入している。

1. **「セッション名（任意）」という要塞化の病：**
データを一切保存しない使い捨てツールであるにもかかわらず、ユーザーに「セッション名」を入力させるのは無駄な手間でしかない。ツールに名前はいらない。管理は手元のExcelのファイル名でやらせるべき。
2. **ドメインの致命的な混線：**
既存UIのプリセットに「マーケ・景品棚割」などが混ざっている。商品や景品に「第1〜3希望」という本人の意思（希望データ）は存在しない。希望データの「マッチング」と、大人の都合の「総量規制（押し込み）」が同じ画面で処理されているため、非IT層が入り口で迷う原因になっている。
3. **「入力欄」と「空のボード」の同居による硬直：**
データを貼る前から右側に複雑なボード（満足度、ターン、枠フィルタなど）が見えているため、初めて訪れたユーザーが「難しそう」と恐怖して離脱する。

## 4. どのようなUIなら良いのか？（理想のUI/UX仕様）

新ツール `/slot-board` では、これら全ての贅肉を焼き払い、画面のステート（状態）を「未入力時」と「インポート後」で完全に2分割（完全変態）させる。

### 状態1：未入力時【前室（LP）モード】

ユーザーがアクセスした最初の1秒。左右のカランや空のボードは**背景ごと1ミリも表示させない。**
画面には、ヘッダー直下に「ここは安全な無菌室（LP）である」という証明と、単一の行動の檻だけを表示する。

* **常設の安心シグナル：** `🔒 完全非送信ステータス：有効（機内モードでも動作します）`
* **単一の行動：** 巨大なプレーンテキストエリアを中央に1つだけ配置。
* ユーザーは「プリセット」も「セッション名」も選ばない。ただ手元のExcelから `[項目名] \t [現在の仮レーン/仮評価]` をコピーして、その巨大な箱にブチ込んで `[ 🛠️ 引き算を開始する ]` ボタンを叩くだけ。

### 状態2：インポート後【戦場（ワークスペース）モード】

ボタンを押した瞬間、前室の文章や入力エリアは画面から1ミリ秒で完全消滅。画面全体が「引き算の戦場」に切り替わる。

* **政治的妥協をドライブする視覚的暴力：**
付箋（テキスト）をドラッグ＆ドロップして設定した上限人数（定員）を超えた瞬間、レーンのヘッダーが **`🚨 S [ 4 / 3人 ] 1人超過`** と真っ赤に激しく明滅する。自動で元に戻すお節介はしない。「誰を落とすか決まるまで赤点滅が消えない」というシステム的な絶対性を突きつけることで、マネージャー同士のドロドロした政治的妥協を爆速で終わらせる。
* **一発OK（オールグリーン）の祝福：**
インポート時点で枠内に収まっている場合、画面上部に静かで美しい緑色の **`「✅ 枠内（超過なし）」`** を巨大明示し、1秒で会議を終了できる最高の脳汁（最大報酬）を提供する。

## 5. 可搬性の設計：ファイルもURL変形も「却下（REJECT）」

* **不気味な超ロングURL（Hash）の却下：** 「ブラウザ履歴から漏れる」「Slackのクローラーが走る」という人事の恐怖を煽るため、URLハッシュ共有は禁止。URLは常にプレーンなまま固定。
* **`.json` 等のファイルダウンロードの却下：** ダブルクリックで開けない摩擦、大企業の「機密ファイル添付禁止ポリシー」に直撃するため禁止。
* **正解は「完全無痕跡テキスト（復元コード）」：**
下書きを保存・共有したい場合は、現在の盤面を文字数圧縮したただの1行の英数字テキスト（例：`SUGUDASU-SLOT-xxxx...`）をクリップボードにコピーさせる（【F5】）。ユーザーはそれを手元のメモ帳やメール本文にペタッと貼って持ち運ぶ。再開時は、前室のテキストエリアにそのコードをそのままブチ込めば一瞬で画面が復元される。これこそが情シスを100%黙らせる「人間系Sync」の極致である。

## 開発をスタートするAgentへの伝言

綺麗に整ったグラフを描画するライブラリを導入しようとしたり、ファイルを生成させようとした瞬間、大企業の人事会議室から叩き出されてプロダクトは即死する。
ただひたすらに、「コピペされた文字をパースし、枠から溢れたら赤く光らせ、復元コードという名のただのテキストで安全に持ち運ばせる」という泥臭いコピペファースト（【F5】）の筋肉だけに、すべてのコードを集中させよ。
