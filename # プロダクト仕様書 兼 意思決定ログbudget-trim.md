# プロダクト仕様書 兼 意思決定ログ：`/budget-trim`（引き算パレット）

本ドキュメントは、他Agentや別セッションの開発LLMが、このツールの実装を開始する際に「なぜこのコードを書くのか」「なぜこの機能を実装してはならないのか」のナラティブ（歴史的経緯）を完全に同期するための、思想的ガードレールである。

---

## 1. 意思決定ログ（ナラティブの変遷）

本ツールは、複数AI（Grok、ChatGPT）および提督とのドロドロとした泥仕合を経て、極限まで機能が削ぎ落とされた。開発者は、過去に却下された以下の「誘惑」を絶対に再実装してはならない。

### 【変遷1】 `/budget-pie`（自動比例配分）の死

* **初期案：** 1つのスライダーを動かすと、他項目が比率を維持して「自動縮小」するオシャレな円グラフUI。
* **却下理由：** 「比例配分は人間の政治的思考とズレる」という致命的バグ。現場の会議では、広告費を増やしたからといって交通費まで均等に削られるのは困る（営業が止まる）。特定の項目だけを生贄に捧げるのがリアルな政治であり、ツールの自動介入はただのお節介であるため、**自動配分機能を完全剥ぎ取り**。

### 【変遷2】 「生贄（ドクロ）モード」「＋ーパドル」の死

* **中期案：** 政治を反映するために「削る対象（生贄）」をトグルで指定する機能、およびプロジェクター用の「＋1万」「ー10万」のパドルボタンの設置。
* **却下理由：** ツールが「意思決定エンジン」になろうとした瞬間、状態管理（ステート）が複雑化し、SUGUDASUの絶対憲法である【F4（要塞化の禁止）】【F5（実務3分）】に抵触する。130万を180万にするためにボタンを50回連打させるのも新たな摩擦でしかない。**お節介な意思決定ロジックは完全焼き払い**。

### 【変遷3】 `/budget-limit`（Bar ✕ Limit Line）の死

* **後期案：** 円グラフを捨て、上限線（Limit Line）に対して各項目（Bar）がどれだけ突出しているかを可視化する棒グラフUI。
* **却下理由：** グラフィック表現そのものが「見栄えの罠（デザインの病）」であった。会議の戦場で人間が凝視しているのは、美しいグラフではなく「生々しい数字」と「あと何円（何時間）で檻に収まるかという超過額」そのもの。また、項目名が長い場合に横幅が崩壊するリスクがあり、10年メンテナンスフリーの思想に反する。**グラフ表現（Pie、Barともに）を完全剥ぎ取り**。

### 【変遷4】 `/limit-counter`（汎用総量カウンター）の死

* **直近案：** 予算に閉じず、工数、時間、シフトなどあらゆる総量規制に対応する抽象化されたカウンター。
* **却下理由：** 「なんでもできる」は「誰にも刺さらない」と同義。実務家は「総量規制」という言葉では検索しないし、課題を認知できない。入り口は「予算の削り合い（Budget Trim）」という最も血が流れる泥臭い具体ドメインに偽装し、中身のロジックのみをシンプルに保つべきという結論に至り、**抽象的な汎用ブランディングを完全焼き払い**。

## 2. SUGUDASU憲法チェック

* **【F1】ログイン不要・ブラウザ完結：** サーバー不要、静的HTML/JSのみで動作。
* **【F2】完全非送信：** 入力データ、調整データはインターネットの向こう側へ1バイトも送信しない。
* **【F3】完全静的配信：** インフラ維持コストは永久に $O(0)$。
* **【F4】要塞化の禁止：** データの永続蓄積は一切行わない。画面を閉じたらすべて消える。
* **【F5】Copy-first：** 手元のExcelからコピペでIN、15分格闘、TSVコピーでExcelへ戻って15秒離脱。

## 3. 絶対にやらないこと（DENYガードレール）

1. **いかなる種類のグラフ（円、棒、レーダーチャート等）も描画しない。**
2. **ツール側から数値を自動計算・自動分配して変更しない。**
3. **サーバーにデータを保存する機能を絶対に付けない。**
4. **「予算」以外の文脈（工数、時間等）に全方位対応させようと、UIに過剰な設定項目（単位切り替え等）を増やさない。**

## 4. 機能要件（MUST）

### ① 入力データの超強力デトックス（パース）

ユーザーがエクセルから適当にコピーしてきた行データを、正規表現で強引に `[項目名(String), 金額(Number)]` の配列に変換する。

* `￥3,000,000-` → `3000000`
* `1,500,000 円 (暫定)` → `1500000`
* 数字以外の全ノイズ（カンマ、通貨記号、全角文字）を剥ぎ取る。行が空白ならスキップ。

### ② 支配的な「超過カウンター」の常時明示

画面上部に、固定ヘッダーとして以下の情報を巨大に表示する。

* `[ 現在合計 ] / [ 上限設定（ユーザー入力） ]`
* 合計が上限を超えている場合：`🚨 超過：＋420,000` （赤色・巨大ボールドで明滅）
* 合計が上限以下に収まった場合：`✅ 残高：15,000` （緑色・ボールド）
* JavaScriptの浮動小数点バグを排除するため、すべての計算は整数（円単位）で行い、合計値の $0$ 誤差を絶対死守する。

### ③ キーボードショートカットによる爆速「削り（Trim）」

プロジェクター前でのファシリテーターの打鍵速度を最速化する。
各項目の数値入力ボックス（input）にフォーカスしている際、マウスを使わずに数値を増減させる。

* `矢印キー（上 / 下）`：数値を `＋10,000 / ー10,000`
* `Shift ＋ 矢印キー（上 / 下）`：数値を `＋100,000 / ー100,000`
※これにより、会議中の「広告費、あと20万削って！」という怒号に1秒で対応する。

### ④ 南京錠（ロック）機能

各項目の横に視覚的な「ロック」アイコンを配置。これは「この項目はこれ以上削れない（確定枠）」という会議中の意思表示（メモリー）として機能する（計算ロジック自体は制限せず、ただの視覚的マーカーとする）。

### ⑤ Hash URLによる一時保存（ノン・ストレージ）

「画面を閉じたら消える」という恐怖を、憲法【F2（完全非送信）】を維持したまま解決する。

* 現在の「項目名、金額、ロック状態」の全オブジェクトをURLハッシュ（`#`以降）にBase64またはLZ圧縮で埋め込んだ超ロングURLをリアルタイムに生成。
* ユーザーはこのURLをコピーして手元にメモするか、Slackで同僚に送るだけで、サーバーを介さずに「会議の下書き状態」を完全匿名で共有・復元できる。

## 5. UI/UX・レイアウト仕様

長い項目名に対する耐性を100%保証するため、グリッドレイアウトによる横並びを禁止し、**スマホのリストビューに近い縦積み構造**をPC大画面でも採用する。

```text
===================================================================
 [現在合計: 3,420,000] / [上限設定: 3,000,000] 🚨 超過: ＋420,000
===================================================================

 📄 Excelからコピペ（インポートエリア）
 [ 項目名 \t 金額 のテキストエリア ] [パース実行ボタン]

-------------------------------------------------------------------

 🛠️ 引き算ワークスペース

 1. 2026年度第2四半期マーケティング施策（SNS含む）
    [ 1,200,000 ] 円  🔒[ロック]

 2. 展示会出展費用およびブース施工費
    [   800,000 ] 円  🔓[アンロック]

 3. カタログ・パンフレット印刷代
    [   250,000 ] 円  🔓[アンロック]

-------------------------------------------------------------------
 [ 💾 格闘結果をコピーしてExcelへ戻る（TSV出力） ]

```

## 6. データ構造と入出力（Copy-first）仕様

### 入力（IN）の期待値（TSVパース）

```text
広告費\t1,200,000円
展示会費\t800,000
印刷代\t250,000

```

### 出力（OUT）の期待値（クリーンテキスト）

余計な記号、カンマ、単位を完全にデトックスした、Excelにそのまま `Ctrl+V` で上書きできるプレーンなTSV。

```text
広告費\t1000000
展示会費\t750000
印刷代\t250000

```

## 開発をスタートするAgentへの伝言

このツールは「計算パレット」**であり、**「意思決定エンジン」ではない。
綺麗に整った棒グラフを描画するライブラリ（Chart.js等）を導入しようとしたり、AIによる最適配分提案ロジックを実装しようとした瞬間、このプロダクトは死ぬ。
ただひたすらに、「コピペされた汚い数字を綺麗にし、手入力に応じて超過額を1円の狂いもなく明示し、クリーンなテキストで吐き出す」という泥臭い摩擦の秒殺だけに、すべてのコードを集中させよ。
