# present（ギフト）— Reject · アーカイブ

**id:** `present`  
**判決:** **Reject**（2026-07-24）  
**判例:** [`CASE-2026-006`](../../legal/CASE_LAW.md#case-2026-006)

## 扱い

| 項目 | 内容 |
|------|------|
| Hub / ナビ / カタログ | **除外** |
| URL `/present` | アーカイブ案内のみ（`noindex`） |
| 新機能 | **停止** |
| 同系統新規 | **採用停止** |

## 理由

**SUGUDASUのコンセプトに合わない。**

SUGUDASUは仕事の前工程（操作 · 段取り · 整形）を静かに片付ける。  
AIで贈り物を提案する体験は Domain 外であり、「提案しすぎない」とも衝突する。

初期の市場・アフィリエイト検証は終了した。  
理由を隠さず、更新履歴（`changelog.json`）にも記載する — それが誠実さである。

Legacy 保守ではなく、公開カタログからの **Reject** とする。
