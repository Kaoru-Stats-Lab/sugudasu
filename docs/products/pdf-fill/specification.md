## スコープ外（MVP）

次は意図的に含めない。

- **元PDFの既存テキスト**の編集・フォント変更（追加した文字の文言差し替えは可）
- ページの結合・分割・並べ替え・削除
- フォームフィールドの作成・編集 · PDFフォーム認識
- OCR · AI欄推定 · CSV差し込み · 帳票テンプレート · フォームビルダー
- 編集ページのテキストレイヤー再構築（OCR復元）
- サーバー保存・アカウント・共有URL
- 数値による位置・サイズ指定
- 1pt単位の精密調整UI
- レイヤー／整列／回転／グループ／マルチセレクト変形
- 透明度・トリミング・色変更
- 印・画像の差し替え専用UI（消して貼り直し）
- **編集ページだけを切り出して別PDFにする**（派生保存。将来検討は philosophy）
- 保存方式の複数選択（方式はラスタ焼き付け一つのみ）
- **「移動」専用タブ**（ADR-016）
- **PDFビューア化**（ページ内スクロール、ビューア型サムネ拡張など。ADR-014）
- フォント自由選択・太字・斜体・字間・行間（ADR-019 / ADR-020）
- `prompt()` による文字入力（ADR-017）
- Input Strip のユーザー向け名称表示（内部概念のみ）
- 図形ツール · SVG編集 · Marker の線幅/色/回転（ADR-026）
- 日付欄レイアウトプリセット · slot 縦移動（ADR-031）

## PDF読込

- Drag & Drop で読込
- （実装時）クリック選択も同等の入口として可。入口は少なく保つ

## 作成ツール

タブは次の5つ（「次に足すもの」）。移動ツールはない。

- 文字（内に Marker Palette · 記号）
- **日時**（内部: Input Strip · template `datetime`）
- 印・画像
- 黒塗り
- 白塗り

---

## Objectライフサイクル

```text
作成
  ↓
配置
  ↓
選択
  ↓
移動
  ↓
サイズ変更（文字・画像/黒/白。Input Strip · Marker は対象外）
  ↓
再編集（文字 · Input Strip）
  ↓
削除
  ↓
保存
```

すべてのオブジェクトは Unified Object Model（ADR-018）に従う。Input Strip · Marker も Object。

## テキスト

- **追加:** 文字ツールで空白クリック → その場でキャレット → 入力 → Enter で改行 · 外クリック / Ctrl+Enter で確定 · Esc キャンセル
- **再編集:** ダブルクリックでインライン再編集
- **編集できるもの:** 内容 · サイズ · 書体のみ（ADR-023）
- **大きさ:** 「－ N ＋」または Ctrl+ホイール
- **書体:** UIは **明朝 / ゴシック** のみ（実フォント名は出さない）
- 内部フォントは可読性優先（ADR-024）。元PDF上の既存文字の編集は行わない

---

## Marker（文字ツール内 · ADR-026）

- UI: 文字ツール選択時に `○ ⬭ ✓ ✔ ✕ △ □ ■` のみ表示（新ツールは増やさない）
- 記号を選んで紙をクリック → `type: marker` を配置
- 操作: 移動 · **サイズ変更**（角ハンドル / －＋ / Ctrl+ホイール）· 削除 · Undo。文字編集なし
- 描画: **Canvas Path 正本**（Unicode 依存禁止）。背景透明 · インクのみ。囲み線は下の文字が残る太さ

---

## Input Strip（内部）· 日時（UI）

**Input Strip** は複数の印字位置（slots）を一つの入力として扱う Object（ADR-025）。

- HTML Form / PDFフォームではない
- 文字オブジェクトの寄せ集めではない
- Undo · Delete · Move は **1単位**
- **リサイズしない**

### 日時（template: `datetime`）

スロット: 年 · 月 · 日 · 時 · 分

- **追加:** 日時ツールで空白クリック → 配置 → 編集開始
- **再編集:** 選択済みをもう一度クリック · Enter · ダブルクリック。貼り付け / Ctrl+D 直後は編集開始
- **まとめて入力:** `2026/07/22` · `令和7年7月22日` などを受け、内部で分解。**年は元号のまま**（西暦にしない。行政書類の既定）
- **Tab / Shift+Tab:** スロット間を紙を書く順に移動（フォーム移動ではない）
- **文字サイズ · 書体:** 文字オブジェクトと同様に「－ N ＋」· 明朝/ゴシック（見切れ防止に必須）
- **複製:** 選択して **Ctrl+C → Ctrl+V**（または Ctrl+D）。Object クリップボードを優先（電子印の画像が残っていても日時を潰さない）。印の貼り付けは「印・画像」タブで Ctrl+V
- **横方向調整（ADR-031）:** 確定後の通常ドラッグは **Object 全体の移動**。slot の横寄せは **下端グリップ**のみ（隣スロットを越えない）。Y・ベースライン固定
- **IME:** 変換中は Tab / Enter で次スロットへ進まない
- 将来の郵便番号・電話番号などは **Template追加のみ**（新 Object 種別を増やさない）

---

## 印鑑 · 画像

- 画像として貼付。移動・リサイズ・削除可
- 回転・トリミング・透明度・色変更はなし
- 差し替えUIなし（消して貼り直し）
- **追加の優先順位（ADR-021 Clipboard First）** — UI案内もこの順
  1. Ctrl+V（クリップボード画像）— 電子印鑑からの最短導線
  2. 紙への Drag & Drop
  3. クリックでのファイル選択
- **リサイズ（ADR-022）:** デフォルトは縦横比固定。Shift ドラッグのみ自由変形。設定UIなし

---

## 黒塗り

- 矩形。角丸なし（墨塗り）
- 配置後、四隅ハンドルでリサイズ可

---

## 白塗り

- 矩形。通常状態は枠・影なし（完全な白）
- Hover時のみ薄い青枠。選択時のみハンドル
- 白塗りの上にさらに白塗り可（重ね）

---

## 選択・移動・削除

- 既存オブジェクトは作成ツール中でも **常時選択・移動**（ADR-016）
- 空白クリック／ドラッグのみ作成ツールが発動
- 通常: ハンドル非表示 / Hover: 薄い青枠 / 選択: ハンドル表示
- 削除: `Delete` / `Backspace` · 選択枠の ×
- 配置の数値入力なし（UXは [ux.md](./ux.md)）

---

## Undo / Redo

- **保存前のみ**
- Ctrl+Z · Ctrl+Y（または Ctrl+Shift+Z）
- 出力後のPDFを「元に戻す」対象にはしない

---

## スナップ

- あり（質感は ux.md）

---

## ガイド

- ドラッグ中かつ吸着中のみ表示（常時・派手禁止）
- 優先: 紙（中央・四辺）→ 他 Object（辺・中央）。Equal Gap なし（ADR-027）
- Alt ドラッグで一時無効。設定UIなし

## 編集ページの定義（正本）

**編集ページ**とは、ユーザーが次のオブジェクトを **1つ以上** 配置しているページである。

- テキスト
- 印鑑
- 画像
- 黒塗り
- 白塗り

機械的定義:

```text
編集ページ ⇔ そのページのオーバーレイオブジェクト数 > 0
```

### 編集と見なさない

| 操作 | 編集？ |
|------|--------|
| ページを開いただけ | いいえ |
| 拡大縮小 | いいえ |
| スクロール | いいえ |
| ページ送り | いいえ |
| オブジェクトを選択しただけ | いいえ |
| 一度置いたが全削除された状態 | いいえ（未編集へ戻る） |

保存で「見たまま固定」されるのは **編集ページのみ**。未編集ページは元PDFをコピーする。

---

## 編集状態のライフサイクル

ページ単位の状態遷移。

```text
未編集
  │
  │ オブジェクトを1つ以上追加
  ▼
編集済み
  │
  │ そのページのオブジェクトをすべて削除
  │ （Undo で空に戻した場合も含む）
  ▼
未編集へ戻る
```

| 状態 | 条件 | 焼き付け時 |
|------|------|------------|
| 未編集 | オーバーレイ数 = 0 | 元ページをコピー |
| 編集済み | オーバーレイ数 > 0 | 見たまま固定（ラスタ） |

選択のみ・閲覧のみでは状態は変わらない。

## 保存（出力モデル）

ボタン名は **「保存」**。セクション見出しは **「提出用PDFを完成」**（文言契約は [ux.md](./ux.md)）。

### 保存対象（固定するページ）

**編集ページだけ** を、次を含めて **見たまま固定**（ページ全体をラスタ化し、新しいPDFページとして生成）する。

- Canvas 描画結果
- テキスト
- 印鑑
- 画像
- 黒塗り
- 白塗り

固定したページでは、既存レイヤー・編集可能オブジェクトは消滅する（そのページに限る）。

### 未編集ページ

- **元PDFのページをそのままコピー**する
- 画像化しない
- 文字検索・コピー可能性は、元ページが持っていた範囲で維持される

### 出力PDFの形

| ページ種別 | 出力 |
|------------|------|
| 編集済み（オーバーレイ > 0） | 画像化した新ページ（見たまま固定） |
| 未編集（オーバーレイ = 0） | 元ページのコピー |

- ページ数・順序は **元PDFと同じ**（提出書類全体を渡す）
- **新しいPDFファイル**を生成する（元ファイルをその場で上書きしない）
- 例: 10ページ中 2・4 のみ編集 → 出力は10ページ。2・4だけ固定、他は元のまま

### 保存方式の数

- **方式は1つだけ**（内部: ラスタ焼き付け。UIボタン: 保存）
- 「編集ページだけを別ファイルに保存」等の派生は **実装しない**（philosophy の将来検討へ）

## ファイル名提案

ダウンロード時に、次を組み合わせたファイル名を生成する。  
**プレビュー表示とダウンロード名は同一関数**（`buildSuggestedFileName`）で、100%一致させる。

| 部品 | UI | 例 |
|------|-----|-----|
| 日付 | （自動 · `YYYY-MM-DD`） | `2026-07-22` |
| 書類名 | ラベル「書類名」· プレースホルダー「例）〇〇申請書」 | `住所変更申請書` |
| 利用者名（任意） | ラベル「利用者名（任意）」· プレースホルダー「例）薫」 | `薫` |

完成例: `2026-07-22_住所変更申請書_薫.pdf`

これは提出書類上の氏名入力ではない。  
**完成PDFをあとで探しやすくするための名前**である。フルネームは要求しない。

SUGUDASU 全体のサンプル人物名は **佐藤薫**。本ツールのファイル名用プレースホルダーは、用途が限定されないため **薫** とする。

## 旧仕様との差分（要約）

| 項目 | 旧（α初期） | 新（本仕様） |
|------|-------------|--------------|
| 固定範囲 | 全ページを画像化 | **編集ページのみ**画像化 |
| 編集ページ | 曖昧 | **オーバーレイ数 > 0**（定義正本） |
| 未編集ページ | 同じく画像化 | **元ページをコピー** |
| オブジェクト操作 | 移動タブ必須 | **常時選択・移動** · 移動タブ廃止 |
| 削除 | Undo／クリアのみ | **個別削除** · 全削除で未編集へ復帰 |
| 再編集 | なし | **文字文言のみ** · 黒白リサイズ |
| UI基準 | （未定義） | **Paper First**（Viewer ではない） |
| CTA | 「提出用PDFを完成」（仮） | 見出し「提出用PDFを完成」+ ボタン「保存」 |
