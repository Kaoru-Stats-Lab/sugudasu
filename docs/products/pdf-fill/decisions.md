## ADR-001: 編集ツールにしない

| 項目 | 内容 |
|------|------|
| **決定** | PDF編集ソフトを目指さない |
| **理由** | 機能追加が Acrobat / Foxit 競争に直結する |
| **帰結** | 既存文字編集・高度なページ編集は MVP 外。今後も「提出」以外の深掘りは原則 Reject |

---

## ADR-002: 提出用ツールに徹する

| 項目 | 内容 |
|------|------|
| **決定** | 提出可能なPDFを完成させることが唯一の成功条件 |
| **理由** | JTBDは「提出できる状態」であり、「編集の快楽」ではない |
| **帰結** | 操作（文字・印・塗り）は提出のためだけに存在する |

---

## ADR-003: 精密配置よりスナップ優先

| 項目 | 内容 |
|------|------|
| **決定** | 精密座標より、揃えやすさ（スナップ）を優先する |
| **理由** | 行政ユーザーに定規・測位の負担は不要。「置けて揃う」を優先 |
| **帰結** | スナップ・ガイドは必須。数値・1pt調整は置かない |

---

## ADR-004: 数値入力なし

| 項目 | 内容 |
|------|------|
| **決定** | 位置・サイズの数値入力UIを設けない |
| **理由** | 迷い・学習・編集ソフト化を招く |
| **帰結** | 配置はドラッグのみ（specification / ux） |

---

## ADR-005: グリッドはドラッグ時のみ表示

| 項目 | 内容 |
|------|------|
| **決定** | グリッドは通常非表示。ドラッグ中のみ表示し、ドロップ後に消す |
| **理由** | 常時グリッドは編集ソフト感を強める |
| **帰結** | ux.md のグリッド節が実装根拠 |

---

## ADR-006: 保存ではなく完成という言語

| 項目 | 内容 |
|------|------|
| **決定** | 結果の本質はクラウド「保存」ではなく、提出物の完成 |
| **理由** | ツールの終点は提出可能なファイルの入手 |
| **帰結** | 完了は「提出用PDFが完成しました」。主ボタンは **保存**（思想語「焼き付け」はUIに出さない）。ボタン直上の安心文と確認ダイアログで固定・不可逆・OCR・黒塗りを伝える。「保存しました」だけの完了は禁止 |

---

## ADR-007: 編集ページの OCR 喪失を受け入れる

| 項目 | 内容 |
|------|------|
| **決定** | **編集済みページ**の固定（ラスタ）により、そのページのテキスト選択・検索喪失を受け入れる |
| **理由** | 提出用の見た目固定を優先。未編集ページまで喪失させる必要はない（ADR-011） |
| **帰結** | 編集ページのみラスタ出力。未編集は元ページコピー。編集ページのテキストレイヤー再構築は非目標 |

---

## ADR-008: ファイル名提案を推奨

| 項目 | 内容 |
|------|------|
| **決定** | ダウンロード時に日付・書類名・利用者名（任意）のファイル名を提案する。プレビューと実ファイル名は同一ロジック |
| **理由** | 完成後に探しやすい名前までが提出体験。提出書類上の氏名入力ではない |
| **帰結** | specification のファイル名節。サンプル人物（全体）は佐藤薫。本UIプレースホルダーは「薫」 |

---

## ADR-009: 競合は Acrobat ではない

| 項目 | 内容 |
|------|------|
| **決定** | 競合定義から Acrobat / Foxit を外す |
| **理由** | 機能比較競争に乗ると負け、かつブランドと不一致 |
| **帰結** | マーケ・LP・優先度議論でも「Acrobat代替」と言わない |

---

## ADR-010: 真の競合は印刷→手書き→スキャン

| 項目 | 内容 |
|------|------|
| **決定** | 置き換える対象は「印刷 → 手書き → スキャン」ワークフロー |
| **理由** | 行政ユーザーの実際の代替手段だから |
| **帰結** | 成功指標も「その回り道をしなくて済んだか」 |

---

## ADR-011: 編集ページのみ固定する

| 項目 | 内容 |
|------|------|
| **決定** | 焼き付け保存時、編集済みページだけ見たまま固定し、未編集ページは元PDFをコピーする |
| **理由** | 提出物は書類全体。固定が必要なのは触ったページだけ。全ページ画像化は迷いに資する |
| **帰結** | specification / technical-design / ux を準拠。全ページラスタは廃止。将来「編集ページだけ切り出し」は別途検討（philosophy） |

---

## ADR-012: 移動タブ廃止 · 作成と選択を分離

| 項目 | 内容 |
|------|------|
| **決定** | 「移動」タブを廃止。タブは作成（文字・印・画像・黒・白）のみ。既存オブジェクトは常時選択・移動可 |
| **理由** | 移動タブ必須はモード税。ドラッグ常在と「迷わない」に資する |
| **帰結** | オブジェクト上は選択優先。空白のみ作成ツール。配置直後は選択状態 |

---

## ADR-013: 再編集は最小のみ

| 項目 | 内容 |
|------|------|
| **決定** | 個別削除必須。文字はダブルクリックで文言差し替え。黒白はリサイズ可。印・画像は差し替えUIなし |
| **理由** | 提出前の手直し（消す・ずらす・直す）は3分提出に必要。フル編集UIは Acrobat 化 |
| **帰結** | レイヤー／回転／インスペクタ等は Reject（却下メモ） |

---

## ADR-014: Paper First Principle

| 項目 | 内容 |
|------|------|
| **決定** | 本プロダクトは PDF Viewer ではなく **Paper Editor** である |
| **理由** | ユーザーは閲覧ではなく提出書類への記入を求めている。「A4用紙へ書く」体験を最優先する |
| **帰結** | UI設計で迷ったら Paper First を優先。PDF領域内スクロールは原則禁止（例外は別ADR）。ToolbarはA4直上。サムネは編集状況可視化でありビューア化しない。編集領域は100vhを最大限利用 |

---

## ADR-015: Calm UX

| 項目 | 内容 |
|------|------|
| **決定** | 上位目標は「処理が速い」ではない。**迷う時間を消す**。Calm UX を採用する |
| **理由** | SUGUDASU は急かす道具ではない。必要なことだけを出し、終われば静かに消える |
| **帰結** | ユーザーを急かさない・待たせない・迷わせない。画面上には「今やるべきこと」だけ。ガイドや確認は必要な瞬間だけ現れ、終われば消える。Paper First（空間）と Calm UX（時間）を両輪とする |

### Calm UX の採用指針

| する | しない |
|------|----------|
| 短いチェックリストで一段伝える | 長文の制約説明を主役にする |
| 進捗は「提出用紙を準備」などの紙の語彙 | 「解析中」「ラスタライズ」などの開発語 |
| 動きは 150〜180ms の短い遷移 | 派手な演出・音・振動 |
| ガイドはドラッグ中のみ · ドロップ後すぐ消す | ガイドの常駐 |

## ADR-016: Object Direct Manipulation

| 項目 | 内容 |
|------|------|
| **決定** | 配置したオブジェクトはモード切替なしで常時選択・移動できる。**移動ツールは存在しない** |
| **理由** | Paper First では「置いた紙を少しずらす」が自然。ツール切替は紙の流れを壊す |
| **帰結** | タブは「次に足すもの」のみ。空白でのみ作成。既存オブジェクト上は常に掴んで動かせる（ADR-012 を強化） |

---

## ADR-017: Inline Text Editing

| 項目 | 内容 |
|------|------|
| **決定** | 文字入力は `prompt()` ではなく、クリック位置で直接入力する |
| **理由** | フォーム入力ではなく紙へ書く体験のため |
| **帰結** | クリック → キャレット → 入力 → Enter 確定 / Esc キャンセル。ダブルクリックで再編集 |

---

## ADR-018: Unified Object Model

| 項目 | 内容 |
|------|------|
| **決定** | 文字・印・画像・黒塗り・白塗りを本質的に **Object** として統一する |
| **理由** | 操作（選択・移動・削除・焼き付け）を共通化し、タイプごとの別UIを増やさない |
| **帰結** | 共通プロパティは `id · type · page · x · y · w · h`。選択は `selectedId`。Type固有（text/font · src）のみ追加 |

---

## ADR-019: Minimal Typography

| 項目 | 内容 |
|------|------|
| **決定** | 文字編集は **内容・位置・大きさ** だけを触る |
| **理由** | 提出書類では大きさ調整で足りる。DTP化を避ける |
| **帰結** | fontSize のみ（± · Ctrl+ホイール）。太字・斜体・字間・行間・色は対象外 |

---

## ADR-020: Minimal Font Selection

| 項目 | 内容 |
|------|------|
| **決定** | 使用できるフォントは **明朝 / ゴシック** の2種のみ |
| **理由** | フォントは表現ではなく可読性の選択。行政・保険・契約書の大半をカバーしつつ UI を増やさない |
| **帰結** | 自由なフォント一覧・アップロード・Webフォント強制は禁止 |

---

## ADR-021: SUGUDASU Ecosystem Clipboard First

| 項目 | 内容 |
|------|------|
| **決定** | ツール間導線はファイル保存ではない。**クリップボードを第一選択** とする |
| **理由** | 電子印鑑 → Ctrl+C → PDF記入 → Ctrl+V が最短。エコシステムとして迷わない |
| **帰結** | 画像追加の優先順位は ①Ctrl+V ②Drag&Drop ③ファイル選択。UI案内もこの順。PNG保存は補助 |

---

## ADR-022: Image Natural Resize

| 項目 | 内容 |
|------|------|
| **決定** | 画像オブジェクトは **デフォルトで縦横比固定**。Shift ドラッグ時のみ自由変形 |
| **理由** | 印影・署名・ロゴは縦横比を崩さない。自由変形は意図のうえでだけ |
| **帰結** | 設定UIは追加しない。黒塗り・白塗りは従来どおり自由矩形 |

---

## ADR-023: Minimal Font UI

| 項目 | 内容 |
|------|------|
| **決定** | 文字UIは **内容・サイズ・書体（明朝/ゴシック）** だけ。実フォント名を出さない |
| **理由** | フォント選択UIを増やすと DTP ソフトへ近づく |
| **帰結** | 「－ N ＋」· Ctrl+Wheel · 明朝/ゴシック横並びのみ。標準明朝・Meiryo・Noto・IPA 等の名称は UI 禁止 |

---

## ADR-024: Readable Default Font

| 項目 | 内容 |
|------|------|
| **決定** | 内部実装は可読性の高い日本語フォントを使う。UI には「明朝」「ゴシック」しか出さない |
| **理由** | ユーザーはフォントを選びたいのではなく、読みやすく自然に見えればよい |
| **帰結** | 明朝 → BIZ UD明朝（フォールバック可）。ゴシック → BIZ UDゴシック（フォールバック可） |

---

## ADR-025: Input Strip

| 項目 | 内容 |
|------|------|
| **決定** | 複数の印字位置を **一つの Object（Input Strip）** として扱う。UI名称は出さず、具体ツール名（いまは「日時」）だけ見せる |
| **理由** | 行政書類では年・月・日などを分けて書く。文字オブジェクトを大量生成すると Paper First が壊れる。一方フォーム認識・OCRは思想外 |
| **帰結** | `type: input-strip` + `template` + `slots[]`。Undo / Delete / Move は1単位。リサイズしない。Template追加で郵便番号等へ拡張可能 |

### Reject

| 案 | 理由 |
|----|------|
| 文字オブジェクトを5個置く | 操作が分裂し、紙へ書く単位にならない |
| PDFフォーム認識 | 編集ソフト化・サーバー依存の入口 |
| OCR / AI欄推定 | 技術自慢・誤認識・思想外 |
| フォームビルダー | 高機能フォームへの逸脱 |

---

## ADR-026: Marker Palette

| 項目 | 内容 |
|------|------|
| **決定** | 文字ツール内に記号パレット（○ ⬭ ✓ ✔ ✕ △ □ ■）を置き、クリックで **Marker Object**（`type: marker`）を紙へ置く。新ツールバー項目は増やさない |
| **理由** | 行政書類のチェック・囲みは「図形を描く」ではなく「紙へ記号を書く」。図形ツール化すると編集ソフトへ逸脱する |
| **帰結** | 移動・**サイズ変更**・削除・Undo。文字編集なし。描画は **Canvas Path 正本**（Unicodeフォント依存禁止）。背景透明・インクのみ。囲み線は下の文字が読める太さ |

### Reject

| 案 | 理由 |
|----|------|
| 図形ツール / 図形一覧 | ツール増殖・編集ソフト化 |
| Unicode 文字として焼く | 環境差・欠字 |
| 線幅・色・回転 | 磨き込み範囲外 |

---

## ADR-027: Adaptive Snap（Paper First）

| 項目 | 内容 |
|------|------|
| **決定** | 紙へ自然に置ける Hard Snap。速度に応じて磁力を変え、ヒステリシスでブルブルを防ぐ。Object より **紙を優先** |
| **理由** | PDF Viewer / PowerPoint の「ガイドに吸う」ではなく、Paper First の「紙に吸い付く」感覚が必要 |
| **帰結** | Priority1=紙（中央・四辺）→ Priority2=他 Object（L/C/R · T/M/B）。Enter≈9px でほぼ100%吸着、Exit≈15px まで維持。ドラッグ中に吸着（pointerup ではしない）。ガイド線は吸着中のみ・控えめ。Equal Gap は Phase2。Alt で一時無効。設定UIなし |

### Reject（本 ADR）

| 案 | 理由 |
|----|------|
| Soft lerp のみ | 「近付いた」止まりで紙に置けない |
| pointerup で初めて吸着 | 離すまで置けていない |
| Equal Gap / Distribution / Align Panel | 行政文書の需要低・編集ソフト化 |
| Grid 吸着 · Snap 設定UI | Paper First は最適解を出す。設定させない |
| Slides 風の派手ガイド | 主役は紙。ガイドは補助 |

## ADR-028: Calm Motion

| 項目 | 内容 |
|------|------|
| **決定** | アニメーションは 150〜180ms · ease-out のみ |
| **理由** | Paper First は静かなUI。演出で急かさない |
| **帰結** | バウンド / Elastic / Overshoot / 回転 / 派手な Fade 禁止。Hover は最小限 |

---

## ADR-029: Unified Focus Ring

| 項目 | 内容 |
|------|------|
| **決定** | 全 Object の選択・hover 枠を統一する |
| **理由** | 種別ごとに枠が違うと「編集ソフトのレイヤー」に見える |
| **帰結** | 選択: 薄い SUGUDASU Blue。hover: さらに薄い Blue。非選択: なし。文字・画像・黒・白・日時・Marker 同一 |

---

## ADR-030: Cursor Language

| 項目 | 内容 |
|------|------|
| **決定** | カーソル語彙を限定する: hover=`grab` · drag=`grabbing` · 文字編集=`text` · リサイズ=`nwse-resize` |
| **理由** | カーソルだけで状態が分かれば、ラベルやモードを増やさなくてよい |
| **帰結** | 上記以外のカーソルを増やさない |

---

## ADR-031: Horizontal Slot Adjustment

| 項目 | 内容 |
|------|------|
| **決定** | Input Strip の各 slot は **横方向(X)のみ** 個別ドラッグできる。Y・ベースライン固定。Strip は最後まで 1 Object |
| **理由** | 行政文書の日付欄は幅だけが書類ごとに違う。レイアウトプリセットや複合スタンプは作らない。「紙で少し横へ寄せる」だけ再現する |
| **帰結** | 確定後は **本体ドラッグ＝Object移動**、**下端グリップ＝slot 横寄せ**（ヒット領域を分離し競合させない）。`slot.dx` + 終了時だけ `reflowInputStripX`。**順序（年→月→日→時→分）は固定**し、隣を越えられない。縦移動・回転・個別リサイズ・個別削除は禁止。Undo は Object 単位。IME変換中は Tab を奪わない |

### Reject

| 案 | 理由 |
|----|------|
| レイアウトプリセット | 書類テンプレ化・思想外 |
| slot 縦移動・個別フォント | 自由レイアウト化 |
| 文字5オブジェクト化 | ADR-025 に戻る |
