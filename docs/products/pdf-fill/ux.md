## 思想（UI層）

迷わせない。測らせない。置かせて、揃えさせて、完成させる。

数値と精密調整で「編集者」にしない。ドラッグとスナップで「提出者」のまま終わらせる。

### PDFビューアを作らない

本プロダクトは **PDFビューアではない。**

ユーザーはPDFを閲覧したいのではなく、提出書類へ記入したい。

編集体験は「PDFを見る」ではなく **「A4用紙へ書く」** を再現する。

以後のUI設計は **Viewer ではなく Paper** を基準とする（[philosophy.md](./philosophy.md) · ADR-014）。

### Calm UX · Calm Motion（時間）

Paper First が空間なら、Calm UX は時間（ADR-015 · ADR-028）。

> 迷う時間を消す。急かさない。待たせない。必要なものだけ出し、終われば静かに消える。

動きは **150〜180ms · ease-out** のみ。バウンド・Elastic・Overshoot・回転・派手な Fade は禁止。

### Marker（紙へ記号を書く）

文字ツール内の記号は図形ツールではない（ADR-026）。  
○ · ⬭ · ✓ などは「紙へ書くチェック／囲み」。

### Input Strip（紙を書く流れ）

行政書類では「令和○年○月○日」のように、一つの日付を複数マスへ書く。

**日時** ツールは、その「書くまとまり」を一回で置く。  
UIに Input Strip とは書かない。フォーム欄の移動ではなく、**紙を書く順番** で Tab が進む。

マス幅が書類ごとに違うときは、各 slot を **左右だけ少し寄せる**（ADR-031）。縦には動かさない。

### Adaptive Snap

紙の中央・端へ強く吸い付く。他 Object にも吸うが紙が優先。  
ゆっくり動かすと吸いやすく、速いと素通り。ガイド線は吸着中だけ薄く出る（ADR-027）。

### Unified Focus Ring · Cursor Language

選択枠は全 Object で同一（薄い SUGUDASU Blue · ADR-029）。  
カーソルは grab / grabbing / text / nwse-resize のみ（ADR-030）。

### UX契約（オブジェクト）

> タブは「足すもの」を選ぶ。足したものはいつでも掴んで動かせる。間違えたら消せる。文字だけ打ち直せる。それ以上の編集ソフトにはしない。

### 固定について伝えること

重要なのは **「全部画像になります」ではない。**

ユーザーが知りたいのは **「何が固定されるのか」** である。

採用する説明:

> 編集したページだけ、見たまま固定されます。

専門用語はユーザー向けUIに出さない。

## 入口（アップロード）— 安心感

制限の羅列を主役にしない。行政文書でも最大の不安は「送られないか」。

```text
提出するPDFを開く

ドラッグ＆ドロップ
またはクリック

✓ このPCだけで処理します
✓ PDFは送信されません
✓ 終わったらそのまま削除できます

提出書類向け · 50ページ程度まで
（40MBまで）
```

「PDFをドロップ / クリック」のような Reader 口調は避ける。

## 読込中

スピナーだけにしない。紙の言葉で進捗を出す。

```text
□□□□□□□□□□
提出用紙を準備しています
1 / 5
```

技術語（ページを解析しています・ラスタライズ）は使わない。

## Paper First

編集領域は常に **A4用紙が主役**。

### レイアウト

```text
Header
Toolbar   ← A4 の直上（視線往復を最短に）
A4 Paper  ← 画面の主役
Footer
```

- Toolbar を画面上部へ遠ざけない（ツール → 紙 → ツール）
- 編集開始後は「紙」が画面の主役になること
- PDFが出た瞬間、約 **150ms** で机に置く（派手でない・ふわっと）
- 不要な余白は持たない（高さ方針は [technical-design.md](./technical-design.md)）

### PDF領域スクロール

**原則禁止。**

理由:

- 紙のメンタルモデルと一致しない
- スクロールバーが二重になる
- 位置感覚を失う
- ドラッグ操作の気持ち良さを損なう

スクロールは **ページ全体（ウィンドウ）のみ** とし、PDFキャンバス内部スクロールは置かない。  
例外が必要になった場合は **ADR必須**。

## 作成ツールと選択（ADR-016）

タブは **次に足すもの** だけ。移動ツールは存在しない。

| タブ | 役割 |
|------|------|
| 文字 | 空白クリック → その場で書く（インライン） |
| 日時 | 年・月・日・時・分を一つのまとまりとして置く |
| 印・画像 | **Ctrl+V** / ドロップ / ファイル選択（この順で案内） |
| 黒塗り | 空白ドラッグで墨の矩形 |
| 白塗り | 空白ドラッグで白い矩形（枠なし） |

- 既存オブジェクト上は、**どの作成タブ中でも** 選択・移動を優先する
- 空白へのクリック／ドラッグだけが、現在の作成ツール
- 配置直後はそのオブジェクトを **選択状態** にする

### 印・画像の案内（Clipboard First）

```text
印・画像を置く

Ctrl+V
または
ドラッグ
またはクリック
```

電子印鑑でコピー → 本ツールで貼り付けが最短。ファイル保存は補助。

## 選択の見え方

| 状態 | 見え方 |
|------|--------|
| 通常 | ハンドルなし。存在感を消す |
| Hover | 薄い青枠のみ |
| 選択 | 青枠 + ハンドル（画像・黒・白）· 文字はサイズUI |

白塗りは通常・非選択時に枠・影を出さない。「ここに何か隠した」印象を消す。

---

## 選択・削除・インライン文字（ADR-017）

| 操作 | 仕様 |
|------|------|
| 選択 | オブジェクトをクリック |
| 移動 | オブジェクト上でドラッグ（モード切替不要） |
| 削除 | `Delete` / `Backspace` · 選択枠の × |
| 文字の追加 | クリック位置にキャレット → **Enterで改行** · 外クリックまたは Ctrl+Enter で確定 · Escキャンセル |
| 文字の再編集 | ダブルクリックでインライン |
| 文字サイズ | 「－ N ＋」または Ctrl+ホイール（大きさのみ） |
| 書体 | 明朝 / ゴシック横並びのみ（実フォント名は出さない） |
| 文字枠 | 四隅ハンドルで伸び縮み可（2行以上の欄に対応） |
| 日時 | 選択済みをクリック / Enter / ダブルクリックで編集。Ctrl+C/V・Ctrl+D で複製（貼った直後も編集可）。Tab は年→月→日→時→分 |
| 黒・白塗り | 選択時に四隅ハンドルで自由リサイズ |
| 画像リサイズ | **デフォルト縦横比固定**。Shift ドラッグのみ自由変形 |

### 改行キー（判断）

紙へ書く体験では **Enter = 改行** が自然。チャットの「Enterで送信」は採用しない。  
確定は「書き終わって紙の外を触る」に相当する外クリック（または Ctrl+Enter）。

### やらない（Reject）

レイヤーパネル、整列パネル、回転、グループ、マルチセレクト変形、数値インスペクタ、フォント一覧、太字・斜体・字間・行間・色、不透明度、トリミング、画像リサイズの設定トグル、UIへの実フォント名表示、「オブジェクト編集モード」への再入場、`prompt()`。

## 左サムネイル（編集状況の可視化）

目的は **ページナビゲーションのビューア化ではない。**

**編集状況の可視化** が目的である。

- 編集ページ（オーバーレイ数 > 0）のみ、青点または鉛筆を表示
- Adobe Reader のようなビューアUIへ発展させない
- ページ切替は副次。主役は常に A4 紙面

未編集ページには編集印を付けない。

## Toolbar 配置

Toolbar は **A4 直上** に置く。

理由: 視線移動の最小化。ツールと紙の往復を最短距離にする。

---

## グリッド

- 通常は **見えない**
- **ドラッグ開始**で表示
- **ドロップ後**に消える

常時グリッドは採用しない。

---

## スナップ（紙の上で位置が決まる）

- カクっと飛んで吸着しない
- **磁石のようにぬるっと**寄せる
- 最後の約 **5px** だけ「スッ」と決まる（一気に飛ばない）

紙の上で位置が決まる感触。吸着＝磁石の暴力ではなく、紙の上の落ち着き。

---

## ガイド

必要最小限のみ。**ドラッグ中だけ**表示。

- 上下左右 · 中央 · オブジェクト揃え
- **ドロップ後 約 0.2秒で消える**（居座らない）

Illustrator / Acrobat のようにガイドが居座るUIは禁止。

---

## 「置けた」の合図

オブジェクトを置いた／ドロップした瞬間、約1フレーム相当の短い青光（✓的な完了感）。

- 音なし · 振動なし
- 派手なトースト不要

---

## 黒塗り（墨塗り）

行政書類で最頻出。

- ドラッグ中は矩形が伸びる
- 離した瞬間は **角丸ゼロの完全な矩形**（墨塗り感）
- ぼかし・角丸マスクはしない
- 配置後も四隅でリサイズ可

---

## 白塗り（修正テープ感）

- 通常は完全な白（枠・影なし）
- Hover のみ薄い青枠、選択時のみハンドル
- 白の上に白を重ねられる
- **ドラッグ中のプレビューだけ**、青の斜線で範囲を見せる（白地の行政書類でも分かるため）。離すと純白に戻る。黒塗りと同じ「墨」にはしない（意味が違うため）

---

## A4 の表示サイズ（ペルソナ判断）

ペルソナは「年に数回、行政・契約の提出書類を埋める人」。開発者のように俯瞰ズームを好むとは限らない。

- 紙は **机（stage）いっぱいに大きく**見せる。メニューは紙の上に置くので、紙を広げてもツールは隠れない
- A4縦は縦長なので、横に余白が残るのは正常（横幅100%に無理に伸ばすと縦がはみ出してスクロールが戻る）
- 小さく見える主因は「高さ不足」と「表示倍率の上限」。stage を高くし、フィット倍率の上限を外す

---

## 数値入力 · 1pt調整

**なし**（仕様である）。文字の大きさだけ ± / Ctrl+ホイールで触る（ADR-019）。

## 完成の3段階（Calm）

「保存しました」ではなく **完成** の物語。

```text
① 見出し: 提出用PDFを完成
② ボタン: 保存 → 確認ダイアログ
③ 完了: 提出用PDFが完成しました
```

確認後、編集ページが約 **0.3秒** でフラットに「固定」される短い演出（技術語なし）のあと、完了メッセージ。

## 保存メニュー

現時点の保存方式は **一つだけ**。

- □ **保存**（ユーザー向けボタン名）

「編集ページだけ保存」「全ページ固定」などの派生選択肢は置かない。  
内部実装はラスタ焼き付け（technical-design）。

---

## ボタン名称とセクション見出し

| 役割 | 文言 | 意味 |
|------|------|------|
| セクション見出し（ゴール） | **提出用PDFを完成** | 何を達成するか |
| 主ボタン | **保存** | 提出用PDFを手元に残す |
| ボタン直上 | 固定・OCR・黒塗りの短い安心文 | 何が起きるか（思想語は載せない） |
| 禁止 | 「保存しました」だけの完了 | クラウド保存・途中保存と誤解される |
| 内部用語 | 焼き付け | ADR · technical-design のみ。UIに出さない |

CTA（ゴール）とボタン名（手段）は役割が違う。混同しない。

## 完成ファイル名（探しやすさ）

紙へ書く体験のあと、**完成PDFをあとで探す体験**も設計対象である。

入力は提出書類の氏名フォームではない。

| UI | 文言 |
|----|------|
| 書類名 | プレースホルダー「例）〇〇申請書」 |
| 利用者名（任意） | プレースホルダー「例）薫」 |
| 完成ファイル名 | ライブプレビュー（ダウンロード名と同一ロジック） |

例: `2026-07-22_住所変更申請書_薫.pdf`

SUGUDASU 全体のサンプル人物名は **佐藤薫**。本画面はファイル名用のためフルネームを要求しない。

## 確認ダイアログ（保存押下後）

専門用語なし。固定されるページを列挙し、**不可逆**を自然に伝える。

```text
保存の確認

編集したページ
2ページ

✓ 2ページ
✓ 4ページ

編集したページは
見たまま固定されます。

あとから内容は変更できません。

編集していないページは
元PDFを保持します。

[やめる] [保存]
```

### 文言ルール

| 使う | 使わない |
|------|----------|
| 見たまま固定されます | ラスタライズ |
| あとから内容は変更できません | 画像化パイプライン / OCR喪失 |
| 元PDFを保持します | 「全ページを変換します」 |

- 編集ページが0のときはダイアログを出さず **「編集内容がありません」** を表示する
- `[保存]` 確定後にダウンロード開始

## 完了メッセージ

「保存しました」は **禁止**。

| 方針 | 文言例 |
|------|--------|
| 第一候補 | **提出用PDFが完成しました** |
| 補足可 | 編集したページだけ固定しました |
