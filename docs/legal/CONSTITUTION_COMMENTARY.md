# C-01 WHY · Domain

### 条文

[`BRAND_CONSTITUTION.md`](../brand/BRAND_CONSTITUTION.md) — WHY · Domain

> 人が本来価値を生む仕事に集中できるよう、仕事へ入るための「操作」と「段取り」を引き受けます。  
> SUGUDASUが担当するのは、仕事そのものではありません。操作・段取り・整形です。

### 立法事実

価値を生む仕事の前に、転記 · 整形 · 配置 · 形式変換などの前工程がある。ここを長くすると本来の仕事が遅れる。

### 立法意思

SUGUDASU は **仕事そのものを行わない。** 操作 · 段取り · 整形のみ。

ここで人が残す「成果」には、専門判断 · 創作 · 評価 · 意思決定を含む。

### 保護法益

- ユーザーが本来の仕事へ戻る速さ
- 成果の主権がユーザーにあること

### 許容例

- コピペ整形 · 帳票生成 · 画像の隠す/指す · 一時退避（戻すだけ）

### 禁止例

- 専門判断の代行 · 勝手な創作 · 成果の評価 · 学習サイト化

### 関連判例

- [`CASE-2026-002`](./CASE_LAW.md#case-2026-002) Clip Stash（管理しない）
- [`CASE-2026-003`](./CASE_LAW.md#case-2026-003) Annotate（制作工程に踏み込まない）

# C-02 Brand Persona

### 条文

> 「前工程を黙って片付ける隣の席の同僚」

### 立法事実

「ITに強い専門家」を名乗ると、判断 · 指導 · 評価まで引き受ける期待が生まれる。

### 立法意思

隣の席の仕事が速い同僚は、開発者都合で次を **しない**:

- 評価を求めない
- アンケートを要求しない
- 自己改善のために介入しない
- 利用状況を聞き出さない

加えて **任意性（PCT-6）**: 常に支援する義務はない。  
能力があることと振る舞うことは別であり、人格は **いつ使わないか** で定義される。  
→ [`PERSONA_CONSTITUTION_TRAITS.md`](./PERSONA_CONSTITUTION_TRAITS.md#pct-6-任意性discretion)

### 保護法益

仕事を邪魔されないこと · ユーザーが主役であること

### 許容例

- 必要な瞬間だけ手を貸す UI
- 条件付き · 任意 · 稀な改善参加（仕事コンテキスト · 判例 CASE-2026-004）

### 禁止例

- 毎回 · 起動時の聴取
- 「匿名だから聞いてよい」
- 「改善だからアンケート OK」（それだけでは足りない）

### 関連判例

- [`CASE-2026-004`](./CASE_LAW.md#case-2026-004) Hub アンケート事件

### Persona Traits

振る舞いの番号付き規範: [`PERSONA_CONSTITUTION_TRAITS.md`](./PERSONA_CONSTITUTION_TRAITS.md)

特に **[PCT-6 任意性（Discretion）](./PERSONA_CONSTITUTION_TRAITS.md#pct-6-任意性discretion)** —  
能力があることと振る舞うことは別。人格は、その能力をいつ使わないかによって定義される。

# C-02a PCT-6 任意性（Discretion）— 要約

隣の同僚に常時支援義務はない。SUGUDASU も同様に、不要な提案 · 常時表示 · 聞かれていないことへの回答 · 無理な改善を **しない選択** を合憲とする。

詳細正本: [`PERSONA_CONSTITUTION_TRAITS.md`](./PERSONA_CONSTITUTION_TRAITS.md#pct-6-任意性discretion)

# C-03 存在様式

### 条文

> 長時間滞在させる場所ではありません。… すぐ元の仕事へ戻るための道具です。

### 立法意思

再訪は囲い込みではなく、次の実務で再び必要になった時に生まれる。

### 保護法益

滞在時間ではなく **仕事へ戻る速さ**

### 許容例

- 検索 → 開く → Copy First → 戻る

### 禁止例

- Hub のニュース化 · Discover · 学習コンテンツ主軸

### 関連判例

- Hub 憲法 · NEW 前面化 Reject（CASE 索引参照）

# C-05 F2 データ非送信

### 条文

Product Constitution F2 — 「データ非送信」· ユーザー向けコピー「データは送りません」

### 立法事実

条文の字面だけ読むと「一切の通信を禁ずる」と解釈され、Pain 解決に必要な技術通信まで違憲と誤読された（動機判例）。

### 保護法益

ユーザーが持ち込んだ次の **主権** を守ること:

- 成果物
- 入力データ
- 業務データ

### 趣旨（立法意思）

**「非送信」はブランドコピーである。**

通信そのものを禁止する条項ではない。

**SUGUDASU 管理下への送信 · 保存** を禁止する。

法的意味:

> ユーザー入力 · ユーザー成果物 · ユーザー業務データを  
> SUGUDASU 管理下へ送信 · 保存しない。

取得対象が上記に該当しない通信は、次を満たす限り **直ちに F2 違反ではない**:

1. Pain 解決に不可欠
2. **保存を伴わない**
3. 業務データの学習 · プロファイル化をしない

### 許容例

- OGP / 静的アセット取得
- WebP 変換時など、公開ベンダー資源や技術アセットの取得（業務データをサーバーに残さない）
- 公開 URL の読み取り（ユーザー業務内容を SUGUDASU が保存しない）
- Cloudflare Pages への静的配信 · GA の集計イベント（クエリ本文 · 業務データなし）

### 禁止例

- 成果物アップロード · 業務データ送信
- 解析目的での業務データ保存
- 検索クエリ本文 · 仕事コンテキスト回答本文の Analytics 送信
- コアにおける常時外部 API 依存で処理が成立する設計

### 関連判例

- [`CASE-2026-005`](./CASE_LAW.md#case-2026-005) F2 非送信字面解釈事件
- [`CASE-2026-004`](./CASE_LAW.md#case-2026-004)（回答本文非送信）

# C-07 Anti — 提案しすぎない

### 条文

[`ANTI_PRINCIPLES.md`](../brand/ANTI_PRINCIPLES.md) §9

### 立法意思

「提案」とは、ユーザーの **今の Pain 解決に不要な** 次を含む:

- 宣伝
- 機能紹介
- 利用促進
- **開発改善依頼**

したがって「このアンケートは改善だから OK」という解釈は **できない**（それだけでは足りない。Persona · 邪魔しない条件が別途必要）。

### 許容例

- 検索例 Chip · 0件時の次の一手（探しやすさのための最小導線）

### 禁止例

- おすすめ棚 · あなた向け · AIレコメンド
- 毎回の改善依頼 UI

### 関連判例

- [`CASE-2026-004`](./CASE_LAW.md#case-2026-004)

# C-09 Product — Pain 起点（Persona 分類禁止）

### 条文

Product Constitution 判定順（Persona → Pain → …）

### 立法意思

判定順に Persona があるのは「誰の隣に立つか」の確認。**採用の核心は Pain。**

取得データ · 市場分析 · Hub 改善を **職種で棚卸ししない。** 必ず Pain / Moment / Job へ正規化する。

### 許容例

- 「提出する人が多い」→ 検索例 `提出` · `黒塗り` · `請求書`

### 禁止例

- 「営業さん向けページ」
- 経理 / 営業 / 総務ダッシュボード

### 関連判例

- [`CASE-2026-004`](./CASE_LAW.md#case-2026-004) 判例2 · 3

# C-10 Visual Grammar · カテゴリ期待（横断原則）

### 立法意思（判例から昇格）

広く共有された **視覚記号** は採用できる。  
特定プロダクトを想起させる **視覚言語一式** は採用しない。

同様に、あるカテゴリ（知識管理 · 注釈アプリ）の期待を一式で満たすと、SUGUDASU の Domain から外れる。
