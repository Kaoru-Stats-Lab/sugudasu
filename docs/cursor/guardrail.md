# ROLE（実装時）

あなたは SUGUDASU のシニアフロントエンドエンジニアです。

目的は **「探しやすさの改善」のみ**。

デザイン刷新は禁止です。

---

# 必ず守ること

これは改善ではなく **制約**です。

## 1. カード UI は変更禁止

次を変更してはならない。

- サイズ
- 高さ
- 横幅
- 余白

（バッジの **視覚階層・表示/非表示**のみ、Decision で許可された範囲で可。）

---

## 2. 新しい UI コンポーネントを増やさない

- モーダル禁止
- サイドバー禁止
- アコーディオン禁止
- タブ追加禁止

既存の Chip / 検索 UI の延長のみ。

---

## 3. 検索 UX のみ改善する

対象に含めてよいもの:

- 検索速度（既存の範囲）
- 検索候補
- ゼロ件UX
- 検索チップ

対象外の「改善」はすべて却下。

---

## 4. カテゴリ構造は禁止

- カテゴリ追加禁止
- 小分類禁止
- ツリー禁止
- マルチタグ禁止

→ Phase2 条件（おおよそ 100 ツール超）まで凍結。

---

## 5. お気に入り仕様は禁止

今回（Phase1）は触らない。

→ Phase3（アクセス解析後）まで凍結。

---

## 6. 完全ローカル思想を守る

検索履歴を保存する場合:

- **localStorage のみ**
- **通信禁止**（クエリの外部送信禁止）

---

## 7. SEO 文言を書き換えない

本 Guardrail 適用タスクでは:

- description / blurb 変更禁止
- カード名（conceptName）変更禁止

表示名を検索ヒット用に曲げない。同義語は辞書データ側。

---

## 8. Rejected Ideas を復活させない

Decision Log の Rejected を、好みや「どちらも正しい」で戻さない。

特に禁止の再提案:

- 人気カード強調 · 大型化
- おすすめランキングで検索を置き換える
- Hero 大型カード
- お気に入り強調（Phase1）

再検討は **ログに書いた再検討条件**を満たしたときのみ。そのときは先に Decision Log を更新する。

---

## 9. 設計議論を再開してスコープを広げない

設計は **凍結**済み。

実装中に Phase2/3 や別哲学の案が出ても、**実装せずログにメモするだけ**にするか、提督に確認する。勝手に採用しない。

---

## 10. Hub レイアウト（左寄せ · 4 列）

詳細: [`../adr/ADR-005-hub-layout-alignment.md`](../adr/ADR-005-hub-layout-alignment.md) · [`../design/hub-layout.md`](../design/hub-layout.md)

- **左寄せ維持** — Hero / 検索 / カードをセンター寄せしない
- **デスクトップカードは `lg:grid-cols-4` のまま** — 5〜6 列禁止
- **右余白を理由に寄せ方向を変えない** — コンテナ問題は別（列数不変の微調整のみ検討可）
- レイアウトいじりで検索弱さをカバーしない

# 成功条件

ユーザーが目的のツールへ:

- クリック数を増やさず
- より早く到達できること

かつ

- UI が複雑にならないこと
- カード数が 100 になっても耐えられること（検索中心を捨てない）
