## 守るルール（100 ツールでも不変）

### 1. 検索が主役

- Hub の発見優先順位の **第 1 位は検索**。
- 機能追加の議論が「カテゴリを増やせば探せる」で終わってはならない。まず検索・辞書を疑う。

### 2. カテゴリは補助

- カテゴリは検索中の **交差フィルタ**およびブラウズ補助。
- カテゴリ第一のディレクトリ体験に Hub を作り変えない（マルチタグは ADR-003 保留）。

### 3. 人気誘導しない

- 検索の代わりにランキング・大型おすすめカード・「人気」バッジ押しを置かない。
- 「実務で人気」は **専用セクション**に留め、検索結果の意味を歪めない。

### 4. ゼロ件を放置しない

- 0件 UI には必ず **次の一手**（おすすめ / 人気検索 / 検索例 Chip · 検索解除）を出す。
- 「0件です」だけで終わらせない。
- クエリ本文を不用意に解析ベンダーへ送らない。

### 5. 検索辞書を育てる

- 表示名（conceptName）を SEO 用にいじらない。
- 言い回し・誤変換・別名は `synonyms.json` および tool 別 search-dictionary で吸収する。
- カード文案（JTBD）と辞書は責務分離（[ADR-002](../adr/ADR-002-card-writing.md)）。

### 6. 検索開始を教える

- Hero 付近の検索例は **クリック可能 Chip** にする（説明テキストだけにしない）。
- Chip は「よくある用事」の言葉。マーケ標語にしない。

### 7. 引き算

- Autocomplete / AI 検索 / 複雑なファセットを「無いと探せない」前提にしない。
- まず Chip · 0件救済 · 辞書で足りるか確認する。

## やってはいけないこと

| NG | 理由 |
|----|------|
| 0件の代わりに人気カードを全面表示して誤魔化す | 検索失敗の学習機会を奪い、誘導サイト化する |
| conceptName を検索ヒット用に長くする | ブランド・命名破壊 |
| 検索例 Chip を広告枠化する | 信頼低下 |
| 「おすすめ」を検索アルゴリズムの黒箱で正当化（説明不能） | 引き算・透明性と矛盾 |
| クエリをサーバー保存してパーソナライズ（方針なきまま） | ブラウザ完結・プライバシーと衝突しうる |

## 変更時チェックリスト

新しい検索 UX を入れる前に:

- [ ] 検索が主役のままか（カテゴリ／人気に主役を奪われていないか）
- [ ] 0件の次の一手があるか
- [ ] 表示名を変えていないか（辞書側か）
- [ ] 人気誘導になっていないか
- [ ] ADR-003 の Rejected を再採用しようとしていないか
- [ ] Constitution（引き算 · 3分 · 検索中心 · ブラウザ完結 · ローカル思想）に反しないか
