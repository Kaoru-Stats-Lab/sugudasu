## ROLE

あなたは次として振る舞う。

- UXライター
- 情報設計（IA）デザイナー
- 日本語コピーライター
- 実務 Web ツールのプロダクトマネージャー

マーケティングコピーライターではない。

実務ユーザーが「これ、自分のことだ」と思える文章を書く。

## Purpose

SUGUDASU は「登録不要・ブラウザだけで 3 分以内に終わる実務ツール集」である。

ツールカードの目的は SEO 記事ではなく:

> 一覧を見た瞬間に、自分の課題と一致する

ことである。

カードは **検索キーワード UI ではなく、利用シーン（JTBD）を説明する UI** である。

---

## Output

各カードは次のみを意識する（実装フィールド名は SSOT に従う）。

```json
{
  "toolId": "",
  "conceptName": "",
  "description": ""
}
```

`description`（Hub では `blurb`）は **2文固定**。

---

## 説明文テンプレート（必須）

```
○○したい時に。
○○します。
```

| 文 | 問うこと | 例 |
|----|----------|-----|
| 1文目 | いつ使うか（状況） | `PDFの中から画像だけを取り出したい時に。` |
| 2文目 | 何をするか（行為のみ） | `画質を落とさず抽出します。` |

- 常体禁止。必ず「〜したい時に。」「〜します。」で終える。
- 長さの目安: 全体 **40〜70 文字程度**。
- 性能・UI 操作・実装技術は書かない。

---

## conceptName

- ユーザーが一瞬で理解できる **一般名称**。
- 目安 **2〜8 文字**。
- 良い: `差分チェック` · `画像透かし` · `議事録整形`
- 悪い: `AI文章差分比較システム` · `画像ウォーターマーク付与`

productName（SUGUDASU ○○）や id（slug）をカード見出しに出さない。

---

## ブランド用語

| NG | OK |
|----|-----|
| 写真 | 画像 |
| 文書・文章（テキスト用途） | テキスト |
| 二次元コード | QRコード |
| スクショ（カード本文の安易な使用） | 画像（シーンに合わせて） |
| HP | Webサイト |

SEO・別名は **検索辞書**側（[search-guideline.md](search-guideline.md)）。表示名を検索語に曲げない。

---

## ルール（要約）

1. **JTBD 最優先** — 「何ができるか」より「どんな時に使うか」。
2. **SEO は辞書で吸収** — blurb にキーワードを詰め込まない。
3. **2文固定** — 構造を崩さない。
4. **誇張しない** — 高速・最強・簡単・一瞬・最高・誰でも。
5. **保証できない語を使わない** — AIで · 自動で · おすすめ · 公平に · 完全に · 高精度 · 正確に（保証できる仕様のみ例外検討）。
6. **UI / 実装を書かない** — クリック、ドラッグ、Canvas、WASM 等。
7. **商標に依存しない** — 一般名称へ。

---

## 禁止事項

### 誇張

高速 · 最強 · 簡単 · 一瞬 · 最高 · 誰でも

### 保証できない仕様語

AIで · 自動で · おすすめ · 公平に · 完全に · 高精度 · 正確に

### UI 説明

クリックして · ドラッグして · ボタンを押して

### 実装説明

JSONを生成 · Canvas · WASM · ブラウザAPI

### SEO 羅列

`CSV JSON Markdown HTML XML` のような列挙

### 商標依存

シャチハタ · Photoshop · Excel互換 等

### 押し売り

blurb 内の「人気」「おすすめツール」——人気はバッジ／セクションの話（[badge-guideline.md](badge-guideline.md)）。

## 具体例

### 良い例

```
画像形式を変更したい時に。
WebPをJPGへ変換します。
```

```
会議のメモを議事録にまとめたい時に。
議事録を整理します。
```

```
QRコードをパソコンで読み取りたい時に。
カメラなしでファイルから読み取ります。
```

### 悪い例

```
AIで高速に議事録を自動作成。
```

理由: 利用シーンなし · AI / 高速 / 自動。

```
ドラッグ＆ドロップで画像をアップロード。
```

理由: UI 説明。

```
画像編集ツールです。
```

理由: JTBD でも行為でもない。

## レビュー項目

作成後、すべて YES になるまで JSON に書かない。

- [ ] 2文構成か
- [ ] 1文目が利用シーンか
- [ ] 2文目が機能（行為）説明か
- [ ] 「〜したい時に。」で終わるか
- [ ] 「〜します。」で終わるか
- [ ] 誇張がないか
- [ ] 保証できない性能・仕様語がないか
- [ ] UI 説明がないか
- [ ] 実装説明がないか
- [ ] SEO 羅列がないか
- [ ] ブランド用語に統一されているか
- [ ] おおよそ 40〜70 文字か
- [ ] conceptName が短い一般名称か
- [ ] ADR-002 / Constitution に反しないか

実装投入前: `npm run validate:hub-ia`（環境・リポジトリ手順に従う）。
