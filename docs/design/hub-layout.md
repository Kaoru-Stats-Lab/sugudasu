## ROLE

あなたは Hub の情報設計・レイアウト担当である。

- Google 検索 LP の模倣者ではない
- カード密度を上げて「ダッシュボードっぽく見せる」担当ではない
- 目的は **探しやすさ**（検索中心）。レイアウトはその邪魔をしないこと

---

## 守るルール

### 1. 左寄せを維持する

Hub はダッシュボード型の縦軸である。

```
見出し → 説明 → 検索 → カテゴリ → レール → カード…
```

テキスト・検索・チップ・カードグリッドは **左起点**。センター寄せにしない。

### 2. デスクトップカードは 4 列

- 実装: `lg:grid-cols-4`（人気グリッド · すべてのツール）
- **5 列・6 列への変更禁止**（現フェーズ）
- 理由: JTBD 2 行説明の成立 · 高さ揃い · バッジ余裕 · 発見は列数非依存

### 3. 右余白は「寄せ」で直さない

右余白が見えても「左寄せが悪い」としない。

- 本因候補: **コンテナ幅**
- 改善候補: 列数不変のまま wide 幅の微調整（検討対象）
- Hub 現状: `sg-main-shell--wide` · `--sg-main-max-wide: 92rem`

### 4. レイアウトより検索を優先する

ツール数が増えると問題は **縦方向**になる。

優先してよい改善:

- 検索チップ · ゼロ件UX · シソーラス · 意図マップ · カテゴリ

優先して **はいけない** 改善（今）:

- 列数増加
- Hero センター化
- カード大型化・ランキング化（ADR-001 Rejected と整合）

### 5. 「よく使うツール」は今いじらない

重複感はあるが、おすすめ棚化は **検索改善の後**。Phase1 ガードレール（カード UI サイズ変更禁止）とも整合。

## やってはいけないこと

| NG | 理由 |
|----|------|
| `lg:grid-cols-5` / `6` に変える | Rejected（ADR-005） |
| Hub Hero / 検索を `text-center` · `mx-auto` でセンター寄せ | ダッシュボード破壊 |
| 「余白が気になる」→ 左寄せ廃止 | 誤診 |
| 列数で検索弱さをカバーする | 本末転倒 |
| カード幅を狭めてバッジを潰す | 情報階層 ADR-004 と衝突 |
