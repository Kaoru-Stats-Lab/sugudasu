## なぜ左寄せか（議論終了）

SUGUDASU Hub は次が一本の縦軸に並ぶ。

- 検索
- カテゴリ
- お気に入り
- 最近使った
- カード一覧

つまり **ダッシュボード**である。Google ではない。

Google:

```
Logo
検索
終了
```

→ センターが自然。

SUGUDASU:

```
見出し
説明
検索
カテゴリ
カード
カード
カード
```

→ **左軸が正解**。

## 右余白は存在するが、左寄せのせいではない

右余白問題は認める。

ただし「左寄せが悪い」ではない。**container 幅の問題**である。完全に別問題。

改善するなら **列数はそのまま**で、コンテナ幅だけ少し広げる派（例: 見かけ 1200 → 1320）。  
現状 Hub は既に `sg-main-shell--wide`（`--sg-main-max-wide: 92rem`）。更なる微調整は検討対象であり、本エントリでは CSS を変えない。

---

## カード 5〜6 列をやらない理由

### ① カード高さが崩れる

4 列なら「タイトル + 説明 2 行」がギリギリ成立。  
5 列 → 幅↓ → 説明 3 行化 → **高さバラバラ**。一覧で痛い。

### ② 探しやすさは列数では決まらない

ユーザーは「画像」「PDF」「請求書」などで検索する。全部は読まない。  
4 列でも 6 列でも発見速度は大差ない（JTBD · 検索中心）。

### ③ 将来カード情報量が増える

NEW · 完全ローカル · お気に入り等。カード幅に余裕がある方が安全。

---

## 本命の優先順位

ツールが 60 / 80 / 100 になると **横ではなく縦**が問題になる。

効くのは:

- 検索 UX
- シソーラス
- カテゴリ
- お気に入り
- 最近使った

→ **検索を強くする**方向が本命。カードを横に増やすより効果がある。

### 最終優先

1. ✅ 左寄せ維持
2. ✅ センター寄せ却下
3. ✅ 右余白＝コンテナ問題
4. 🔜 コンテナ幅微調整は検討
5. ❌ カード 5〜6 列は現時点非採用
6. 🔜 次フェーズはレイアウトより **検索品質**（シソーラス · ゼロ件UX · 検索チップ）

思想: 引き算 · 探す時間を減らす · 3 分で終える。
