## 1. 運用目的とチャネルの役割分担

| チャネル | 役割 |
|----------|------|
| **Search Console（SEO）** | 検索意図（「請求書 無料 インボイス」等）に基づくロングテール流入の土台構築 |
| **X（SNS）** | 課題の言語化 · 口コミの種 · 更新速報 · エゴサによる偶発流入 · 検索経由ユーザー向けの**信頼の担保** |

XはSEOの代替ではない。検索で来た人がプロフィールを見て「怪しくない」と感じる受け皿として機能させる。

## 3. 投稿フォーマットと週次配分

**挨拶単体**（「おはようございます」等）は厳禁。必ず価値または露出の意図を持つ。

| 型 | 割合 | 主な目的 | 発信のトリガー |
|----|------|----------|----------------|
| **① 課題解決型** | 50% | 保存 · リンククリック | コピペ地獄・位置ズレ・事務の手間の言語化 |
| **② FAQユーモア型** | 20% | フォロー · プロフィール遷移 | FAQ由来の一問一答のみ（invoice 等） |
| **③ エゴサリプライ型** | 20% | 非フォロワーへの露出 | **1日3件まで** · 1行テンプレ · 長文禁止 |
| **④ 更新告知（Changelog）** | 10% | 生存証明 | `data/changelog.json` / `updates.html` と連動 |

**トーン:** 実利9割 · ユーモア1割（FAQ由来に限定）。

## 4. 厳守ルール・NGリスト

1. **URLは必ず直リンク**  
   `google.com/search?q=...` 等の迂回URLは禁止。  
   例: `https://sugudasu.com/invoice?utm_source=x&utm_medium=social&utm_campaign=invoice_pain`

2. **誇大表現の禁止**  
   「業界No.1」「最強」等は使わない。便益は「登録不要」「ブラウザ完結」等の**事実**のみ。

3. **カテゴリ比較の徹底（競合名指し厳禁）**  
   freee · マネーフォワード等を名指しで貶さない。  
   比較するなら「クラウド型（アカウント管理）」vs「その場完結型（ローカル処理）」まで。

4. **政治・税制への主観的批判の禁止**  
   インボイス · 源泉等への愚痴は公式から出さない。「どんな制度でも、1秒で終わらせるツールを提供する」職人スタンス。

5. **クラウド保存の誤解を与えない**  
   「サーバーに保存して管理」等は禁止。ローカル完結・非送信と矛盾する表現は使わない。

6. **「データ非送信」の適切な明示**  
   毎投稿には入れない。顧客名 · 印鑑 · 住所を扱う文脈（invoice / receipt / label）では自然に添える。**月1回**、独立した信頼投稿を行う。

7. **リプライでの長文議論の禁止**  
   仕様指摘にはレスバしない。「フィードバックありがとうございます。反映しました」+ changelog 連動で対応。

8. **意味のない挨拶ポストの禁止**（週次配分の「挨拶単体厳禁」と同義）

## 10. スプレッドシート運用（正本）

**運用スプシ（正本）:** [X_Post_Management](https://docs.google.com/spreadsheets/d/1fzztvFFJ2qEhkmrRkwCyEPWdfhwZgcU8gDdNiqWr1EI/edit?usp=sharing)  
**正本 JSON:** `data/x_posts_calendar.json`（v2 · 40本 + エゴサ8件）  
**スプシ貼付用 TSV:** `data/x_posts_calendar.tsv` · `data/x_posts_ego_search.tsv`  
**再生成:** `npm run export:x-posts-tsv`（JSON を編集したあと）

### シート1「投稿カレンダー」（`x_posts_calendar.tsv`）

| 列 | 内容 |
|----|------|
| 投稿ID | W1-D1 等（8月以降は `8月以降（W8-D1）`） |
| 型 | ①②④ |
| ツール | invoice / receipt / … |
| シーズン | 常時 · 再投稿用タグ |
| 文字数 | 参考値 |
| メモ | ピン固定 · changelog 等 |
| 改訂メモ | 編集意図（運用では非表示可） |
| ステータス | 未投稿 → 投稿済 |
| 本文 | **Xにそのままコピペ**（URL含む） |

貼付: ファイルを UTF-8 で開き全選択 → スプシ **シート1の A1** に貼る。列「投稿予定日」を A 列左に足してよい。

### シート2「エゴサリプ」（`x_posts_ego_search.tsv`）

③型はカレンダー外。**1日3件まで** ここから選んでリプ。

### Gemini を使う場面

- **初回ネタ40本の案出し** … 済（v2）
- **追加分・季節入替** … `x_guideline.md` + `changelog.json` を添付して依頼
- **スプシ整形** … **不要**（`npm run export:x-posts-tsv` で TSV 化する）

**禁止事項をプロンプトに明記:** §4 の NG 全項目 · 直リンクのみ · 競合名指し禁止。
