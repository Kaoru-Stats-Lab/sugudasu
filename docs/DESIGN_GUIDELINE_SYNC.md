## 0. 2層の関係

| 層 | ドメイン | トーン | 本ファイル |
|----|----------|--------|------------|
| **コア** | `sugudasu.com` | slate · 中立 · 非送信 · L3 緑＝印刷のみ | 親 `DESIGN_GUIDELINE.md` を正とする |
| **Sync** | `sync.sugudasu.com` | B2B · 稟議 · 組織共有 · 格子密度やや高 | **本ファイル** |

**原則**

1. **主 CTA は両層とも `blue-600`（`--sg-primary`）** — 別ツール遷移時の違和感を防ぐ。
2. **緑はコア同様「印刷・PDF・ローカル完結」専用** — rakumo の「予定を登録する」緑ボタンは採用しない。
3. **violet は Sync 専用・極小面積** — 「設計中」「版更新バナー」以外に使わない。
4. **パステル行背景** — 進行表の **属性識別** のみ（装飾・LP 全面色には使わない）。

### 0.1 デザイン意図 — ユーザーが受け取る印象

実装者・文案担当が「なぜこの色・この余白か」を迷ったときの根拠。**LP コピーにそのまま転載しない**（煽り・心理分析の露呈は避ける）。トークンと挙動の正本は §1 以降。

#### 色彩心理・カラーコーディネーション

**公認の実務インフラとしての安心感**

| 要素 | 狙い | 幹事への効果 |
|------|------|--------------|
| `slate-100` × `slate-800` | 純白・全面ダークを避け、コントラストを抑えたシックな環境 | 本番直前の焦り・視覚疲労を沈静化 |
| `blue-600` 単色（グラデ禁止） | 日本の総務・人事が慣れた B2B / GWS 拡張系の規律を想起 | ツールへの心理的障壁を下げる |

**視線誘導（見たいレーンへ吸い寄せる）**

| 要素 | 狙い | 幹事への効果 |
|------|------|--------------|
| パステル4色（§2 `--sync-row-*`） | 彩度を落とした行背景のみ。文字可読性は `--sg-text` で固定 | 司会・音響・映像・舞台の境界を瞬間認識 |
| `violet` 極小面積 | 基本パレット外の色を **ステータス専用** に限定 | S1（現状）と S2（設計中・将来同期）の鮮度を直感識別 |

#### 情報設計・ウェブディレクション

**海外プロツールの冷たさとエクセルの泥臭さの中間**

- **Notion / Stripe Docs 調の余白** — 枠線・アイコン・広告を削ぎ、進行表データに集中させる。
- **rakumo 流の簡易登録ポップオーバー** — 叩いた位置に最小入力が出るアフォーダンスで、「エクセルより扱いやすい道具」感を出す（組織ツリー等の重厚 UI は借りない）。

**押し売り感の排除 → 稟議に通しやすい業務ツール**

- グラデ・ネオン・満足度バッジ等のマーケ臭を置かない → 職場・上司の前でも開きやすい。
- 印象目標: **洗練された静かな職人気質の相棒**（チャラい SaaS ではない）。

#### 要素融合マトリクス（実装判断用）

| 画面要素 | rakumo → SUGUDASU 翻訳 | 主観的印象 | UI/UX メリット |
|----------|-------------------------|------------|----------------|
| 背景 `slate-100` ＋ 白カード | プレーン入力枠を Stripe Docs 調余白で再構築 | 「洗練・お堅く公的」 | 全体コントラストを下げ、本番のパニック心理を鎮静化 |
| 単色 `blue-600` 主 CTA | グラデ・蛍光の完全排除 | 「実務のための相棒」 | インフラ感・稟議 SS のウケを最大化 |
| パステル役割別行背景 | カレンダー種別色をタイムライン行に移植 | 「自分のレーンが1秒でわかる」 | ＋5分シフト時も担当者が変化を直感検知 |
| `disabled` 見積/料金 | 未実装の正直な開示 | 「誇大広告のない誠実な開発元」 | できる/できない境界をデザインで保証 |
| `violet` 極小バッジ | サブアクセントを設計中ラベルのみに | 「既存機能とは別枠のステータス」 | S2 を実装済みと誤認する本番事故を防止 |
| 180ms スタッガー＋5分デモ | `prefers-reduced-motion` 併記の連動表示 | 「エクセル再計算の恐怖から解放」 | 1箇所のズレ連鎖を視覚で確信させる |

### 0.2 認知科学・UIUX — 3つの体験的印象

ペルソナ: **25〜40代 · 研修・イベント幹事**（日本の一般ビジネスパーソン）。触れた瞬間に目指す同時受容: **使いやすさへの確信** + **心理的安全性**。

| # | 体験軸 | ユーザーの印象（要約） | UI/UX メカニズム | 実装への約束 |
|---|--------|------------------------|------------------|--------------|
| ① | **メンタルモデルの結合** | 「いつも使う業務ツールが、モダンでサクサク動く感じ」 | パステル担当色で **マニュアル不要のアフォーダンス**（どこを見るか本能理解） | 黒背景・ネオン・海外プロツール風は **×** |
| ② | **認知負荷の最小化** | 「画面が静かで目がチカチカしない。情報がスッと入る」 | 低彩度背景 + ノイズ除去 → **分刻みの時刻** に視線集中 | 満足度バッジ・過剰アニメ・バナー乱立 **×** |
| ③ | **シグニファイアの厳格化** | 「今動く範囲と開発中が、触る前に一目でわかる」 | S1=`blue-600` · 未実装=`disabled` · S2=`violet` で **未成熟を隠さない** | 「完全同期」等の過大コピー **×**（`SUGUDASU_SYNC_LINE` と整合） |

> **LP 文案注意:** 競合・他社製品名（rakumo / freee 等）は **ガイド内の比喩に留め、ユーザー向けコピーには出さない**。

### 0.3 二面性 UX — 決裁者と現場の両立

同一画面が **稟議（決裁）** と **本番（操作）** の両方に耐えるよう、装飾を足さずトーンを分ける。

| 誰が見るか | 受ける印象 | デザインの根拠 | 期待される結果 |
|------------|------------|----------------|----------------|
| **上司・決裁者** | Stripe Docs 的な知性。「ギラギラ広告がない。社外秘データを預けても法務的に安心」 | `slate-100` + 白カード · 単色青 CTA · マーケ臭ゼロ | 少額立替・事前稟議が通りやすい |
| **現場スタッフ** | 「エクセルの数式むき出しではない。スマホで自分の担当色だけ追えばいい」 | パステル行 · 簡易登録 · 単一縦タイムライン | 本番の操作ミス・説明コストの削減 |

**ディレクション原則:** 「お堅い顔」と「優しい顔」は **別ページに分けず**、引き算（色・余白・正直な disabled）で1 UI に同居させる。

|------------|-------------|-----------------|-----------|

## 2. Sync 拡張 CSS 変数

`assets/sugudasu.css` の `:root` に追記する想定（**実装はタイムライン UI 着手時**）。

```css
:root {
  /* 親トークン（既存）— 再掲 */
  /* --sg-bg: #f1f5f9; --sg-primary: #2563eb; --sg-print-cta: #059669; */

  /* Sync 専用 */
  --sync-sub-violet: #7c3aed;       /* violet-600 · 設計中バッジ・版バナー枠 */
  --sync-sub-violet-bg: #f5f3ff;    /* violet-50 */

  /* 進行表行属性（役割・担当別パステル）— 文字は常に --sg-text */
  --sync-row-mc: #fdf2f8;           /* 司会アナウンス */
  --sync-row-sound: #f3e5f5;        /* 音響キュー */
  --sync-row-video: #e8f5e9;        /* 映像・配信 */
  --sync-row-stage: #fff7ed;        /* 舞台・登壇者動き */
}
```

**クラス命名案**: `.sync-row--mc` | `--sound` | `--video` | `--stage`（背景のみ · ボーダーは `--sg-border`）。

## 3. レイアウト・パターン

| パターン | rakumo の例 | 借りる | Sync への翻訳 |
|----------|-------------|--------|---------------|
| 組織階層ツリー | 開発本部 ＞ 技術部 ＞ メンバー | **×** | 組織マスター不要。ルーム ID + 共有 URL でフラット化 |
| **簡易登録ポップオーバー** | グリッド空きクリックで最小入力 | **◎** | 「＋行追加」位置にタイトル・時間の軽量ポップアップ |
| 左固定詳細フォーム | 常駐の予定登録パネル | **○** | コア進行表のメモ・設定は **ドロワー**（常駐は LP では使わない） |
| **役割別パステル** | 会議紫・外出緑 | **◎** | 行属性の識別（§2 変数）。複数カレンダー重ねは **×** |
| 週次タイムグリッド | 30分区切り・DnD ブロック | **◎** | 単一縦タイムライン + 必要時 **列追加**（レーン重ねない） |

## 5. 2層対照（衝突回避）

| 要素 | コア（維持） | Sync（追加） | 衝突回避 |
|------|--------------|--------------|----------|
| ベース背景 | `bg-slate-100` | カード `bg-white`（`.sg-card`） | 余白は slate-100 · 作業面は白カード |
| 主 CTA | `bg-blue-600` 単色 | 同左 | グラデ禁止 · オレンジ CTA は **料金・ログインのみ** 検討可（`--sg-brand-orange` は小面積） |
| サブアクセント | なし | `violet-50` / `violet-600` 極小 | 設計中・版バナー以外に紫を混ぜない |
| 入力 | Notion 風プレーン input | 簡易登録ポップオーバー | 行クリックでページ遷移しない |
| 信頼バッジ | emerald「データを送らない」 | **クラウド保存** を別文言で（混同しない） | コア LP に Sync バッジを置かない |

## 6. タイムライン UI — DnD 仕様（MVP）

| 操作 | 挙動 | 視覚 | 後続連動 | a11y |
|------|------|------|----------|------|
| **行 DnD** | 左グリップで上下移動 | `bg-slate-50` + `shadow-sm` | 以降行の開始・終了を再計算（既存エンジン） | `prefers-reduced-motion` 時は即時並べ替え |
| **＋5分** | 該当行延長 or 全体シフト（モードは製品設定） | `bg-blue-50` ボタン 180ms 沈み込み | 後続時刻をスタッガー更新（デモは LP でも可） | `aria-live="polite"` で「◯分シフトしました」 |
| **版更新バナー** | サーバー新版検知 | 上部 `border-violet-200` 細バナー | **手動反映のみ**（自動上書き禁止） | バナー内ボタンへフォーカス移動可能 |

**エンジン SSOT**: `docs/notes/TIMELINE_TOOL_SPEC.md` · `assets/timeline-engine.js`（コアと共有）。
