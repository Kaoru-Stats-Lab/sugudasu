## note 公開用本文（コピペ）

### タイトル

ハッカソン直前、班分けが崩れた瞬間の胃の痛み

### リード

ハッカソンや研修の幹事で一番胃が痛くなる瞬間は、開始直前の「班の組み直し」です。事前に職能や属性を丁寧に配分しても、欠席連絡が来たら表は一瞬で崩れます。IT営業やマーケの現場で調整役をやってきたカオルが、名簿をどこにも預けない **SUGUDASU 班分け** を作った理由——派手な機能より、幹事の「ちょっとした余白」を取り戻す話です。

### 本文

## アウトプットを出すための、チーム編成という制約

納会みたいに「楽しく座ればいい」場面なら、エクセルでランダムに並べるだけで済むこともあります。でも成果物をちゃんと出すハッカソンやワークショップは話が別です。

「この班にはエンジニアを最低1人」「この2人は絶対に別班に」「ファシリテーターは分散させて」——条件が重なると、途端に手が止まります。結局、ホワイトボードに名前を書き出しては消し、消しては書き直し。周りからは「もう決まった？」と聞かれ、時間だけが過ぎていく。幹事経験がある人なら、あの焦燥感を思い出すはずです。

事業開発の現場でも何度か似た調整をしてきました。表計算で綺麗に管理できていたはずが、現実の制約が入るとたちまち破綻する。班分けはアウトプットの土台なのに、そこに一番時間と神経をすり減らす——それが現場です。

## 開始直前のドタキャンは、強制リセット

本当に怖いのは、受付が始まってから来る「体調不良で欠席」の連絡です。何時間もかけた班表が、一瞬で意味を失います。「3班、エンジニアゼロになりました」——初対面ばかりの中で、手作業で名簿を組み直すしかない状況は、幹事の心臓に悪い。

過去に「もう時間がないからこのままで……」と妥協したことがあります。成果に差が出て、後から悔しかった。運営側が本当に欲しいのは、ルーレット演出ではなく、例外が起きたときにさっと対応できる余裕です。

## スイッチャーが詰まる、名簿とルームの境界線

オンライン開催では、班が決まっても仕事は終わっていません。Zoom のブレイクアウトルームに人を入れていく作業で、班番号と部屋名の対応表がすぐ作れないと、スイッチャーが詰まります。

「名簿はできたけど、誰をどの部屋に……？」裏チャットが慌ただしくなり、コピペと手作業で成形する時間が食われていく。班分けツールに必要だったのは結果一覧だけでなく、Slack や Teams、Zoom にそのまま貼れる**成形済みテキスト**の出力でした。

## あえて Zoom やクラウドと自動連携しない理由

開発中、AI 相棒と何度も相談しました。「Zoom API でボタン一つに連携できたら」と。でもやりませんでした。

API 連携を入れると、アカウント登録や名簿の一時的な外部預かりが発生します。セキュリティが厳しい現場では「今日、今すぐ、ここで」使えなくなる。**SUGUDASU** は入力した名簿をサーバーに送らない設計を最優先にしました。便利さのために現場の安心を犠牲にしない——それが引き算です。

## 泥臭い幹事に、5分のコーヒータイムを

こうしてできたのが **SUGUDASU 班分け** です。コードは書けないけれど、効率化の痛みはわかる総務・研修幹事・コミュニティ運営者向けに作りました。

事前に PC で職能付き名簿を用意しておけば、当日はスマホだけで欠席者を外して班を組み直せます。出力はチャットに貼りやすいテキストと、後で再現できる JSON だけ。トラブル続きで胃がキリキリする幹事に、開始前にコーヒーを一口飲む余裕を届けたい。説明できる公平さの、地味なインフラで十分です。
