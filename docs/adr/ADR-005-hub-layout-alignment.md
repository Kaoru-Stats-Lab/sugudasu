## Background

なぜ議論したか

Hub のスクリーンショットで「右に余白が多い」「Google のようにセンター寄せにした方がよいのでは」「カードを 5〜6 列にして密度を上げては」という提案が出た。

SUGUDASU Hub は検索一発の LP ではなく、**検索 · カテゴリ · お気に入り · 最近 · カード一覧**が一本の縦軸に並ぶ **ダッシュボード型**である。レイアウト好みの議論は無限に続くため、思想に沿って凍結する。

---

## Problem

何が問題だったか

1. **右余白**が目立ち、「左寄せが悪い」と誤診されやすい。
2. **センター寄せ**は Google 型検索 LP の慣習だが、Hub の情報構造と合わない。
3. **5〜6 列**は一見密度が上がるが、カードの JTBD 2 行・バッジ余裕と衝突する。
4. レイアウト議論が、本命の **検索品質改善**（シソーラス · ゼロ件 · チップ）の時間を奪う。

---

## Options

| ID | 案 |
|----|-----|
| A | **左寄せ維持 · 4 列維持**。右余白はコンテナ問題として分離。列数は増やさない |
| B | Hero / 検索をセンター寄せ（Google 型） |
| C | カードだけ 5〜6 列 |
| D | コンテナ幅を少し広げる（例: 見かけ 1200 → 1320 相当）— 列数は不変 |

---

## Decision

採用案

**案 A を採用。案 D は検討対象（実装は別エントリ）。案 B · C は却下。**

### 採用

| 項目 | 判定 | コメント |
|------|------|----------|
| 左寄せを維持 | ✅ | 議論終了 |
| センター寄せにしない | ✅ | LP ではなく Hub / ダッシュボード |
| 日本語可読性（左軸の説明） | ✅ | 縦に積む UI は左揃えが自然 |
| ヘッダーとの整合性 | ✅ | サイトクロムも左起点 |
| 右余白＝コンテナ問題として分離 | ✅ | 「左寄せが悪い」ではない |
| デスクトップカード **4 列**（`lg:grid-cols-4`） | ✅ | 現状維持 |

### 検討対象（今は実装しない）

| 項目 | 判定 | コメント |
|------|------|----------|
| コンテナ幅を少し広げる | 🔜 検討 | 列数はそのまま。Hub は既に `sg-main-shell--wide`（`--sg-main-max-wide: 92rem`）。更なる微調整は計測後 |

### 将来（今すぐではない）

| 項目 | 判定 | コメント |
|------|------|----------|
| 「実務で人気 / よく使うツール」の棚化 | 🔜 | 1 枚だけ目立つ重複感。人気 3 件 · 横スクロール · コンパクトカード等。**検索・シソーラス改善の後** |
| 60〜100 ツール時の縦スクロール対策 | 🔜 | 横列増ではなく検索 UX · カテゴリ · お気に入り · 最近 |

---

## Rejected

| 案 | 理由 | 再検討条件 |
|----|------|------------|
| **センター寄せ** | Hub は「見出し→説明→検索→カテゴリ→カード…」の縦軸。Google（Logo→検索→終了）ではない | 原則なし。Hub を LP に作り変える意思決定が出たときのみ |
| **カード 5〜6 列** | (1) カード幅↓で説明が 3 行化し高さバラバラ (2) 発見速度は列数より検索で決まる (3) バッジ・人気等で情報量が増える余地が必要 | カード blurb を 1 行に縮める別 ADR が通り、かつ解析で 4 列の横スクロール／余白が害と判明したときのみ |
| **左寄せをやめて余白を消す** | 右余白の本因はコンテナ幅。寄せ方向の問題ではない | — |

---

## Future Review

1. ビューポート別の右余白が **体感で害**（解析 or 提督確認）→ コンテナ幅のみ微調整（列数不変）
2. ツール数がおおよそ **60〜100** → 縦方向発見の再設計（検索優先は維持）
3. 「よく使うツール」重複感の苦情 → おすすめ棚の別 ADR

再検討しても壊さないもの → **Constitution**。

---

## Constitution

| 思想 | Hub レイアウトでの意味 |
|------|------------------------|
| **引き算** | 列数を増やして「詰め込んだ感」を出さない。センター寄せの装飾をしない |
| **3分で終わる** | 探す時間を減らす本命は **検索品質**。レイアウトいじりで代替しない |
| **検索中心** | 4 列でも 6 列でも発見速度は大差ない。シソーラス · ゼロ件 · チップを優先 |
| **ブラウザ完結 / ローカル** | 本 ADR の直接対象外。カード幅の余裕はバッジ（完全ローカル等）の表示余地として間接的に守る |
