## Background

なぜ議論したか

Hub の発見優先順位は **検索が 1 位**（`HUB_IA_REFRESH_V2`）。しかし検索欄だけでは、初見は「何と打てばよいか」が分からない。

一方で、検索が弱いからといって **人気カード誘導・ランキング**で埋めると、サイトが Discover 化し、SUGUDASU の引き算・目的検索と衝突する。

Top IA Phase1 で、検索 UX の採用・保留・非採用を分離して記録する。

---

## Problem

何が問題だったか

1. **検索開始の摩擦** — 例がテキストだとタップ／クリック行動にならない。
2. **0件の死** — 言い回し違いで 0件になり、離脱する。
3. **表示名と検索語のギャップ** — 「ハンコ」「OCR」「スクショ」等。表示を変えるとブランドが壊れる。
4. **人気誘導の誘惑** — 0件や初見を「おすすめカード大型化」で誤魔化したくなる。

---

## Options

検討案

| ID | 案 | 概要 |
|----|-----|------|
| A | **検索チップ + 0件救済 + 同義語辞書** | 表示名不変。辞書と UX で吸収 |
| B | **人気カード / ランキングで入口を埋める** | 検索は副次 |
| C | **Autocomplete / AI 検索を先に入れる** | コスト大 · プライバシー・複雑性 |
| D | **マルチタグを先に実装** | カテゴリ交差を主探索にする |

---

## Decision

採用案

**案 A**。

### 採用

#### 1. 検索チップ

- Hero 下の検索例を **クリック可能 Chip** にする。
- 設定: `hub-config.searchExampleChips`。
- 新規コンポーネント体系は増やさない（既存 Chip 様式に乗せる）。
- **理由:** 初見が「検索という行為」を始めやすい。カテゴリや人気より先に検索行動を教える。

#### 2. ゼロ件UX

- 0件時に放置しない。
- **おすすめ / 人気検索 / 検索例**の Chip + **検索解除**を維持する。
- キーワード本文は GA 等に送らない（プライバシー）。
- **理由:** 言い回し失敗を「次の一手」に変換し離脱を減らす。誘導ランキングではなく、**再検索・解除**を主とする。

#### 3. 同義語辞書

- `data/synonyms.json` のみで吸収（必要に応じて tool 別 `search-dictionary`）。
- **conceptName / 表示名は変えない。**
- 例: ハンコ→印鑑系、OCR→QR読取、スクショ→マスク、写真→画像系。
- **理由:** ブランド用語を守りつつ、検索ヒット率を上げる。SEO と表示の責務分離（[ADR-002](ADR-002-card-writing.md)）。

### 保留

#### マルチタグ / マルチカテゴリ

- **実装しない。** TODO・disclaimer のみ可。
- **理由:** 100 ツール超で必要になり得るが、現規模では単一 `categoryId` + 検索辞書の方が費用対効果が高い。カテゴリは **補助フィルタ**のまま。

---

## Rejected

却下案 · 理由

| 案 | 理由 |
|----|------|
| **人気カード誘導**（結果代わりに人気を前面化・強調） | 検索主導と矛盾。押し売り。0件の本質（言い回し・辞書不足）を隠す |
| **おすすめランキングを検索の代替にする** | 目的検索より誘導優先。Constitution 違反 |
| **案 C（AI / Autocomplete 先行）** | 複雑性・コスト・期待値管理が過大。まず辞書とチップで足りる |
| **表示名を検索語に寄せる** | 命名・ブランド破壊。辞書の仕事を UI に押し付けるな |
| **0件で何も出さない** | 離脱確定。SUGUDASU の「終わらせる」体験に反する |

---

## Future Review

再検討条件

1. **100 ツール到達** — マルチタグ / ファセットの再評価（保留解除候補）
2. **検索ログ蓄積** — チップ文言・同義語・0件サジェストの根拠更新
3. **アクセス解析** — 検索開始率・0件率・解除後の成功率が悪化したら UX 再設計
4. **辞書メンテコストが爆発** — Embedding / 外部検索の是非を別 ADR で検討（その場合もプライバシー Constitution は維持）

## Constitution

SUGUDASUが絶対守る思想

| 思想 | 検索 UX での意味 |
|------|------------------|
| **引き算** | Autocomplete や AI を「無いと探せない」前提にしない。まず Chip と辞書 |
| **3分で終わる** | 探す時間を短くする。0件で迷子にしない |
| **検索中心** | カテゴリ・人気は補助。ランキングで検索を置き換えない |
| **ブラウザ完結** | 検索クエリを不用意に外部送信しない |
| **完全ローカル思想** | ツール本体の非送信と矛盾する「クラウドおすすめ検索」を Hub に持ち込まない |
