## 実装前（必須 · 報告してから着手）

1. `git status` — クリーンでなければ提督に確認
2. `MULTI_AI_CODER_PLAYBOOK.md` §3 自己レビュー
   - 許可ファイルのみか
   - 変更前コードが現行リポジトリと一致するか（不一致 → **BLOCKED**）
   - 呼び出し元が設計図にあるか
3. 1項目でも FAIL → **BLOCKED 報告して停止**（§6.2 フォーマット）

## 実装中の厳守

- 許可ファイル以外は触らない
- 設計図の「変更後関数全文」をそのまま反映（ついでリファクタ禁止）
- リネーム · 共通化 · 依存追加 · 無関係フォーマット禁止
- パッチが当たらない → 推測で埋めない → **BLOCKED**
- テスト失敗 → 期待値を勝手に変えない → **BLOCKED**（INV 古い可能性）

## 実装後（必須）

### 1. ゲート実行（パケット §4 + 変更種別）

| 種別 | 最低限 |
|------|--------|
| A | `npm run test:{tool}` |
| B | test + `npm run preview:pages` で対象ツール簡易確認 |
| C | + `validate:tool-naming` · `validate:ogp` · `build:pages` |
| D | `validate:tool-naming` · `build:pages` |
| E | `npm run test:all` · `build:pages` |
| F | `release:pages:*` + DEPLOY_LOG（提督依頼時のみ） |

### 2. 全 INV を順に検証し pass/fail を記録

### 3. 完了報告（§6.1 フォーマット）

```markdown

- **ゲート:** {コマンド} → exit {code}
- **スコープ外変更:** なし
- **BLOCKED 事項:** なし
```

### 4. 提督向け1行サマリ（日本語）

「{変えなかったこと}を維持したまま、{変えたこと}を実施しました。」
