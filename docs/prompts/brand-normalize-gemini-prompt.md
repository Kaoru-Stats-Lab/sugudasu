# SUGUDASUとは

SUGUDASUは

- ブラウザだけで完結
- 登録不要
- 入力データを送信しない
- 実務向けWebツール集

です。

ユーザーは

「請求書を作りたい」
「PDFから画像を取り出したい」
「画像を切り抜きたい」

など、

**"やりたいこと"**

で検索します。

一方、SUGUDASU内部では

ブランドとして使う正式用語を統一しています。

例

写真 → 画像

スクショ → 画像

PDFファイル → PDF

文章 → テキスト

などです。

検索では

ユーザーの自由な言葉を受け入れながら、

最終的にはブランド語彙へ正規化します。

---

# 今回作るもの

今回は

**ブランド語彙への正規化辞書**

だけ作ります。

これは

ユーザー検索語
↓
ブランド語彙

へ変換する辞書です。

---

# 目的

例えば

スクショ → 画像

写真 → 画像

PDFファイル → PDF

フォト → 画像

記事 → テキスト

文章 → テキスト

このように

**SUGUDASU内部で使う正式語へ吸収する辞書**

を作ってください。

---

# 重要

これは

検索ヒット率を上げるためではなく

**ブランド語彙を統一するため**

の辞書です。

つまり

表示文言まで統一できる品質が必要です。

（誤変換・タイポ・広い俗称は Layer2 search-thesaurus の仕事。toolId 推薦は Layer3。）

---

# 正規化対象

## 1. 表記ゆれ

## 2. 日本語の言い換え

## 3. 略称

## 4. 英語（ブランド語への直結のみ）

## 5. 俗称（商標・通称でブランド語が明確なもの）

# ブランド語彙

可能な限り以下へ寄せてください。

画像 · PDF · テキスト · 表データ · QRコード · 印鑑 · CSV · JSON · Markdown · Webサイト · 動画 · フォント · 議事録 · 請求書 · 領収書 · シフト · タイムライン · 班分け · 割り勘 · ギフト · 企画書 · 検索 · Excel · Word · PowerPoint · 画像切り出し · 透かし · マスク · 時給計算 · 差分チェック

必要なら追加して構いません。

---

# やってはいけないこと

・検索アルゴリズムを提案する

・Fuse.jsなど実装を書く

・検索順位を考える

・SEO施策を書く

・ツール推薦を書く（toolId を書かない）

・JTBDを書く

・シソーラスを書く（Layer2）

・近い意味語を書く（曖昧な関連語）

今回は

**normalize辞書**

だけです。

---

# 出力形式

JSONのみ。説明不要。

[
  {
    "from": "スクショ",
    "to": "画像",
    "category": "略称"
  }
]

---

# 品質要求

最低100件以上（厳密なブランド正規化。無理に300件に水増ししない）。

重複禁止。

日本人が実際に検索しそうな語を優先。

ブランド語彙を汚さないこと。

「写真→画像」のように、**1つの正式語へ収束させる設計**を徹底してください。
