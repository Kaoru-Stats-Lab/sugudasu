## Gemini への依頼文（コピペ用）

```text
あなたは日本語の業務効率化・人事研修・イベント幹事UXに詳しい調査アシスタントです。
礼賛・前置き・記事本文は不要。指定フォーマットの表と箇条書きのみ出力してください。

【背景（調査の前提）】
- ユーザー痛点: 人事・研修担当・ハッカソン運営が「ブレイクアウト班・チーム分け」を公平に決めたい
- 均等に分けたいが、固定3人班・各組メンター必須等の **制約で目標定員を超える組** がありうる
- 既存解: Excel RAND() / 手作業 / Online Stopwatch の Random Name Pickers・Group Generators（演出・教室向け）
- 自作ツール方向（SUGUDASU group-split）:
  - ブラウザ内のみ（非送信）· 名簿1行1名 · シード再現
  - **2次利用前提** — 結果を Slack / Excel / LINE / 社内Wiki に **コピペで流す**
  - ルーレット演出はしない · サーバー同期・日程調整は対象外

【依頼1】グループ分け・班分けの「ユーザーあるある」表
次の列で **最大15行**（各セル100字以内）。

列: シーン | 困りごと | よく使う回避策 | 回避策の弱点 | ローカル非送信ツールが効く条件

シーン例: 新入社員研修ブレイクアウト / ハッカソンチーム / 異動者研修 / 店舗ロールプレイ / 忘年会卓（副次） / 学校席替え（参考）

【依頼2】既存手段の比較表
**特定サービス名は最大4つまで**（例: Online Stopwatch Group Generators · Excel · Googleスプレッドシート · 調整さん等から **グループ分けに関係するもののみ**）。

列: 観点 | 既存手段A | 既存手段B | 向いている人

観点例: データの行き先 / 制約（固定班・離すペア）/ 2次利用（Excel・Slack）/ 公平性の説明 / 演出 / オフライン / 名簿上限

【依頼3】2次利用 — 幹事が結果を「どこにどう貼るか」表
人事・研修ファシリが班割結果を **実際に使う先** を調査。**8〜12行**。

列: 貼り先 | 典型フォーマット | 必須列・体裁 | 失敗例（貼れない・崩れる）

貼り先例: Excel / Googleスプレッドシート / Slack / Teams / LINE / メール / 社内Wiki / 投影スライド / 名札印刷（label連携）

【依頼4】SUGUDASU group-split の位置づけ — 1段落以内×3項目（箇条書き）
- Online Stopwatch 等で足りるケース（正直に）
- group-split を選ぶ合理的理由（非送信 · TSV · 制約 · シード、など）
- group-split で **やってはいけない** 約束（ルーレット演出 · 日程調整 · 名簿API取得 · 100%最適分配 等）

【依頼5】検索意図・キーワード候補
日本語ロングテール **10〜15件** · 形式: キーワード | 検索意図（1行）| 記事で触れるべき注意（1行）

例: 研修 グループ分け ランダム / ブレイクアウト 班分け / ハッカソン チーム分け 公平 / 席替え ツール 等

【禁止】
- 記事本文・Zenn下書き・Markdown記事
- 未確認の統計数字（「〇%の人事が」等）。不明は「要確認」
- SUGUDASU の礼賛文
- 「100%公平」「完全ランダム保証」等の断定
- 実在しない機能（AI最適編成 · Zoom連携 · 自動リマインド 等）を group-split に帰属させないこと
- 学校向けルーレット演出を group-split の主戦場にしないこと（人事・研修が主）

【出力順】
§1 あるある表 → §2 比較表 → §3 2次利用表 → §4 位置づけ箇条書き → §5 キーワード表
```
