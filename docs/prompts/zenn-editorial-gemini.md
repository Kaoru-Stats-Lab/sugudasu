## Gemini への依頼文（コピペ用）

```text
あなたは Zenn / Qiita / 個人開発メディアに精通した編集プランナーです。
礼賛・応援の前置きは不要です。「画期的」「革命的」「最強」「No.1」は禁止です。
記事本文の執筆・推敲・リライトは一切しないでください。企画表とアウトラインのみ出してください。

【プロダクト】
SUGUDASU（すぐだす）https://sugudasu.com/
登録不要 · ブラウザ内完結 · 入力データは原則外部サーバー非送信 · Cloudflare Pages 静的配信
運営・開発表記: カオル（社名・勤務先・法人名・実名は公開しない）

【Zenn の位置づけ】
- note より Tech / 個人開発 / 実務寄り。被リンクと検索を重視。
- PR TIMES は見送り（有料）。X + Zenn + Qiita が無料の主軸。
- ペルソナは **案A（静かな実務職人）7割 + 案B（開発ストーリー）月1本まで + 案C（ライト使い方）は Zenn では最小**（本格ガイドは note へ逃がす想定）。

【ツール一覧（記事化候補）】
| ツール | URL | 実務フック例 |
|--------|-----|--------------|
| 見積・納品・請求 | /invoice | 見積→請求転用 · インボイス · 非送信 |
| 領収・手取り逆算 | /receipt | 源泉 · 確定申告期 |
| 割り勘 | /warikan | 幹事 · 固定額モード · LINE精算文 |

| ラベル | /label | 宛名 · 発送 |

| ポータル | / | 9本ハブ |
| 更新履歴 | /updates | changelog 連動 |

【文体・テイスト（厳守）】
- です・ます調。短い文。専門用語は必要最小限。
- 礼賛しない。「〜と思います」程度の控えめな推量は可。
- 競合（freee / MF 等）の名指し貶し禁止。「クラウド型 vs その場完結型」まで。
- 税制・インボイスの断定禁止。免責を想定（「発行前にご確認を」）。
- 利用者数・売上・テスト人数など未公開数字は捏造しない。
- 「Vibe Coding」は使わない。

【記事に入れてよい事実】
- 非エンジニアが AI（Claude / Gemini / Grok）を使って個人開発
- 見積下書き → 書類タブ切替 → PDF（invoice）
- 割り勘の固定額モード、シフトの可変枠など changelog にある機能のみ
- 改善要望は updates の Google Form

【タスク】
Zenn 向けに **8〜12本** の記事テーマを設計し、**最初の3ヶ月** の投稿スケジュール感を出してください。
各テーマについて「どんな中身にフォーカスすべきか」「どんなテイストか」を具体的に書くこと。
本文は書かず、見出し案 + 箇条書きアウトライン（各記事 **5〜8 bullet**）まで。

#### §4 3ヶ月スケジュール
```
