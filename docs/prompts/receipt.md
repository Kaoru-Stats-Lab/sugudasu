# 要件定義書：手取り逆引き電卓 ＆ 領収書一括生成（`receipt.html`）

> **SUGUDASU 実装時の上書きルール（正本）**
> - 単体HTML完結ではなく、`tools/receipt.html` + `assets/sugudasu.css` + `assets/sugudasu-shell.js` で実装する（他ツールと同型）。
> - チャット共有は `invoice.html` Phase 1 と同じ「文面コピー + 送信先URL起動」（Webhook直叩きはしない）。
> - URLパラメータ共有は **任意機能**。宛名・金額をURLに載せる旨をUIで明示する。

## 1. 概要・思想

本ツールは「ログイン不要・1秒解決・手数最小」を掲げる Web ツール集 **SUGUDASU** の新規アセット（`receipt.html`）である。

取っ払いの単発アルバイト現場や個人間取引で、発注者が「手取りジャスト〇〇円を支払いたい」となった際の「本体価格・消費税・源泉徴収税」の逆算を1秒で解決し、その場でスマホから PDF/URL 共有するか、事前に A4 縦で複数人分をまとめて印刷できるハイブリッドツールを目指す。

---

## 2. 画面構成・UI

完全クライアントサイド。レスポンシブ（スマホ・PC）。大きく2エリア。

### ① 入力・設定（画面上部・印刷時非表示）

- **金額入力:** 希望「手取り金額」
- **計算条件:**
  - 源泉徴収税 10.21% 有無（デフォルト ON）
  - 消費税率 10% / 8% / 0%
  - 内税 / 外税（デフォルト 外税）
  - 端数処理 四捨五入 / 切捨 / 切上（デフォルト 四捨五入）
- **出力モード:**
  - 1枚出力（スマホ・単発）
  - A4マルチ印刷（3〜4名・切り取り線付き。一括反映ボタンあり）
- **マナー:** 受領印（赤丸）表示 ON/OFF
- **但し書きサジェスト:** アルバイト代 / 作業手間賃 / WEB制作費 / お車代
- **アクション:**
  1. PDF保存・印刷（`window.print()`）
  2. Slack / Chatwork 等にテキスト報告（Phase 1 方式）
  3. URLで領収書共有（クエリパラメータ）

### ② 領収書プレビュー（画面下部・印刷メイン）

- 白黒ベースのビジネス向けデザイン
- 1枚モード: 中央1枚
- マルチモード: A4縦に3〜4枠・点線カット

## 3. 計算仕様

手取り $Y$、税率 $T$、源泉 $W$（例 0.1021）のとき、外税・源泉ありの本体価格 $X$:

$$X = \frac{Y}{1 + T - W}$$

内訳:

- 本体価格: $X$（端数処理適用）
- 消費税: $X \times T$
- 源泉徴収: $X \times W$
- 差引支給 = 本体 + 税 − 源泉 = $Y$（ジャスト整合）

**実装注意:** 内税モードは別式。MVP は外税のみでも可（Backlog参照）。

## 4. 現場特化機能

1. **印紙税注記:** 逆算後の支払総額が5万円超で「※電子データにつき印紙税不適用」を薄グレー表示（税務助言ではない旨を免責と整合）
2. **受領印:** CSS/SVG の赤丸「受領」「済」（トグルで非表示可）
3. **印刷:** `@media print` で入力・ボタン・chrome 非表示、領収書のみ
