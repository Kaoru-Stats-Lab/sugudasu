```text

【背景】
新興の無料ブラウザツール「SUGUDASU テストデータ」を、閉域・セキュリティ厳しい現場向けに伸ばしたい。
ただし業種が広すぎて楔（最初の1市場）が決まらない。

提督の方針:
「閉域でテストデータが欲しいところ」ではなく、
**SUGUDASUのような新興サービス（無料・登録不要・ブラウザ単体・非送信）を受け入れられそうな楔** から入る。

【SUGUDASUの現実】
- 知名度ゼロに近い · エンタープライズ営業なし · 情シス向け資料も薄い

- 競合: Handy Dev Tools（ブラウザ・日本語・1万件）/ SnowFakery（CLI・YAML）/ Mockaroo（クラウド）

【依頼 — Grokらしい切り口で】

## A. 「新興ツールを使わない」本当の理由（業種別）
以下のセグメントごとに、**表向きの理由** vs **本音** を分けて書け:
- 金融受託SI（テスター・マニュアル）
- 人材・給与ベンダー
- 医療機器・医療SIベンダー
- 官公庁ベンダー
- 製造・物流の社内情シス
- 小規模会計事務所・社労士事務所
- 建設・リフォームの受託

各セグメント:
| 表向き | 本音 | SUGUDASUが入る隙間 | 隙間なしならOUT |

## B. 現場の「最後の手段」ランキング
閉域でダミーデータが欲しいとき、実際に何をするか — 頻度順に:
1. Excel手打ち
2. 社内の過去CSV再利用
3. ネットからダミーCSV DL（規程違反気味）
4. 本番の一部コピー（禁止気味）
5. エンジニアがスクリプト
6. SnowFakery / 社内TDM
7. Handy Dev Tools 等のブラウザツール
8. 諦めてデータ少ないままテスト

→ 各セグメントで **2番目に多い手段** は何か（1位は業界で共通かもしれない）

## C. Handy Dev Tools に勝てる「瞬間」
https://handy-dev.tools/test-data-generator と https://qiita.com/chiyoyo/items/d0ee467a0229a465efcd を前提に、
SUGUDASUが **Handyでは代替できない瞬間** を5つ。
（「非送信」だけでは弱い — もっと具体的に: 誰が · どの承認プロセスで · 何に困って）

## D. 楔候補の辛口スコアリング
以下6候補を **新興SUGUDASU受け入れ度** で0〜10点（辛口 · 平均7点超えない想定で）:
1. 金融受託・マニュアル
2. 人材・給与（受託）
3. 医療機器ベンダー
4. 小規模社労士・会計事務所
5. 建設・リフォーム受託
6. スタートアップ・個人開発（比較用 · 閉域ではない）

| 候補 | 受け入れ度 /10 | 最初の100ユーザーの取り方 | 失敗パターン |

**1位を1つだけ推奨** — 理由は3 bullet

## E. 最初の100人の取り方（GTM）
```
