## 記事ネタカタログ（§記事ネタ — 提督が EPISODE_BULLETS に転記）

### NOTE-01 ハッカソンで班決定≠完了

- **文脈:** 班分け後に Zoom BR · Slack 案内が本番。スイッチャーが班表を見ながら手動移動
- **Pain:** エンジニア0人の組が出たときの再編 · 班番号とBR名の対応がないと詰む
- **引き算:** Zoom API 連携をあえてやらない（OAuth · 名簿送信）
- **ツール:** group-split · Zenn #14 へ link
- **カオル角度:** マーケ/事業開発で「現場調整役」をやってきたから、ツールは「誰をどの組に」まで

### NOTE-02 納会の円卓とハッカソンは別物

- **文脈:** 同じ「班分け」でも納会（バッファあり）とハッカソン（ゼロバッファ）で必要機能が違う
- **Pain:** 各組必須をアイディアソンにかけるとオーバーキル
- **引き算:** 緩和モード · 各組必須を幹事が選べる設計
- **ツール:** group-split

### NOTE-03 名簿をクラウドに上げたくない抽選

- **文脈:** キャンペーン幹事は「公平」と「説明責任」が欲しいが、統制システムは重い
- **Pain:** Excel RAND() は再計算で説明が崩れる
- **引き算:** 懸賞管理SaaS · AI最適班 · 投影ルーレット演出
- **ツール:** fair-draw · statements リンク

### NOTE-04 社外秘スクショを変換サイトに載せられない

- **文脈:** WebP が Excel に貼れない · 未公開素材を iLoveIMG 等に送りたくない
- **Pain:** 「ブラウザ完結」と謳うサイトでも向きが違う（WebP→JPG vs PNG→WebP）
- **引き算:** 一括ZIP · 数百枚 · サーバー送信
- **ツール:** webp-to-jpg

### NOTE-05 Excel列コピーで行数がズレた夜

- **文脈:** 500行 cap · 行数チェック · コピーゲート（normalize 設計の原点）
- **Pain:** 全角半角直したつもりが1行消えて貼り戻し失敗
- **引き算:** 自動クリップボード上書き · 全自動変換
- **ツール:** normalize · Zenn #12 link

### NOTE-06 請求書ツールにログインを付けなかった理由

- **文脈:** 副業フリーランスが「今この1枚だけ」欲しい · クラウド請求の月額疲れ
- **Pain:** 下書きJSONは自分のPCに残す · PDFは取引先へ
- **引き算:** アカウント · チーム同期 · 会計ソフト直結の約束
- **ツール:** invoice · statements

### NOTE-07 割り勘で「飲まない人」を数値にした話

- **文脈:** 幹事の透明精算 · 上司/後輩比率 · 端数過不足
- **Pain:** 電卓とメモの揉め事 · LINEで説明しづらい
- **引き算:** 割り勘アプリのSNS連携 · 過剰な断定（合法宣言）
- **ツール:** warikan

### NOTE-08 AI3名と役割分担（Zenn #2 の note 版）

- **文脈:** 非エンジニアが企画・仕様・監査を人間側で持つ · 実装はAIに任せすぎない
- **Pain:** プロンプト1発では品質が安定しない
- **引き算:** 「AIが全部作った」ナラティブ
- **ツール:** hub · Zenn #2 link（技術はZennへ逃がす）

### NOTE-09 投影ルーレットをあえて作らない

- **文脈:** 教室向け演出 vs 幹事のTSV/Slack
- **Pain:** 会場映えと事務記録は別Pain
- **引き算:** ドラムロール演出 · 統制システム
- **ツール:** group-split · fair-draw
