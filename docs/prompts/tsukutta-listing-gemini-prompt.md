# Tsukutta 掲載文案 — Gemini プロンプト（再利用）

**更新:** 2026-07-18  
**目的:** Cursor トークンを使わず、[Tsukutta](https://tsukutta.app) の「アプリを編集」欄を埋める  
**原則:** Gemini に **製品仕様を発明させない**。提督が貼る **事実ブロックだけ** を言い換える

---

## 0. 結論（どうする？）

| やること | 誰 | なぜ |
|----------|-----|------|
| 事実・制約・禁止文句 | **提督（または Cursor で SSOT からコピペ）** | Gemini は抽選・景表法・非送信の境界を知らない |
| キャッチ・タグ・動作環境の文案 | **Gemini** | マーケ文案だけで足りる |
| スクリーンショット撮影 | **提督** | UI の正は本番。AI に捏造させない |
| コード変更・仕様判断 | **しない** | Tsukutta は露出チャネル。製品は sugudasu.com |

**今後も同じ:** エッジ製品（§1 の撒き餌一覧）を Tsukutta に載せるたび、該当の事実ブロックを1つ貼って Gemini に投げる。

---

## 1. 撒き餌（エッジ）だけ — 事実ブロック集

**Tsukutta に載せるのは「差別化1文が即言える」ものだけ。**  
請求書・見積会議・SNS・ギフト・割り勘など、便利だがコモディティ寄りのものは **ここに置かない**（必要になったら別途）。

| 優先 | id | プロダクト | エッジ（撒き餌の芯） | 推奨カテゴリ |
|:----:|-----|------------|----------------------|----------------|
| 1 | `fair-draw` | 抽選 | 景表法一次チェック × 名簿非送信 × 証跡PDF | 業務効率化ツール |
| 2 | `normalize` | 全角半角整え | 変換サイトNGでも名簿を貼れる · 非送信 | データ処理・変換 |

- 案A / 案B / 案C（各1行・全角28文字以内目安）

- 1〜2行。ブラウザ名・登録不要・スマホ可/推奨PC など、事実ブロックの範囲だけ

## 3. 検収の目安（エッジ撒き餌共通）

| ○ | × |
|---|-----|
| 差別化1文が仕事場面で通じる | 「合法」「監査合格」「世界初」「完全非送信」 |
| カテゴリが画面6択のラベルそのまま | 英語IDや独自カテゴリ名 |
| 禁止表現を事実ブロックどおり守る | 事実にない連携・同期・AI自動判断を追加 |

抽選（1-A）のキャッチ検収例:

- ○ 「名簿を預けずに、抽選の説明材料まで残す」

- × 「これで景表法クリア」「監査合格ツール」

## 4. Cursor を使うとき（例外）

次だけ Cursor / Agent:

- 本番スクショの置き場整理
