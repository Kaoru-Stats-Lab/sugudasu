# 製品の絶対制約（違反案は即捨て）
1. コア sugudasu.com: 登録不要・ブラウザ内完結・データ非送信・無料。この約束を壊す案は不可。
2. Sync sync.sugudasu.com: アカウント必須・クラウド・有料。コアの「非送信」を Sync に流用して「非送信のまま同期/API」と書いてはいけない。
3. Sync 旗艦は Schedule（工事・制作工程）と timeline-sync（イベント当日同期）。旗艦を捨てた新規5本並列は不可。新規案は旗艦の楔か、明確な第2柱か、旗艦クラスター拡張のいずれか。
4. Sync は無期限クラウド保存を前提にしない（ルーム/イベント単位の保持期限・エクスポート）。
5. 電帳法・税務・労務は「対応完了」と断定しない。
6. パスワードマネージャ領域は OUT。
7. AIチャット1発で足りる定型文・方言変換は PARK。
8. コア UI を必須ログインで汚さない。Sync 導線はオプトインのみ。

# すでに採択済み（重複禁止・焼き直し禁止）
- T-SYNC-INV-ARCHIVE: 請求・領収の有期限履歴ルーム（条件付きGO・第2柱候補）
- T-SYNC-EVENT-ROOM: fair-draw/timeline の当日ステータス Room（縮小GO・旗艦拡張）
- T-SYNC-SECURE-BIN: 有期限共有ビン（HOLD）
- 旗艦: Schedule / timeline-sync（P0は製品E2E。新ドメインより先）

# 探索方法（必須）
海外マイクロSaaS名や「日本版〇〇」から始めない。

次のタイプの変数から **2〜4軸** を自分で選び、格子（組み合わせ表）を先に作れ。軸の数は3でなくてよい。跳ねそうな軸を優先せよ。

## 変数プール（ここから選ぶ・足してもよい）

**タイプα — 既存アセット（A）**
invoice, receipt, fair-draw, group-split, timeline, Schedule DB, normalize, table-conv, lottery, stamp, label, report, sns, PDF/印刷, CSV列ピック, Room/Push-Pull, 手動反映バナー, retain/purge

**タイプβ — 制約・摩擦（B）** ※制度に限らない。日本固有でなくてよい
版ズレ（誰かが古い時刻で動く）, 提出様式が相手ごとに分岐, 「名簿を外に上げない」, 日本語取引先名・表記ゆれ, Shift-JIS/Excel文字化け, 有期限で消したい機密, IT低リテ現場監督, 紙と口頭の当日オペ, 証跡は残したいがクラウド常駐は嫌, 都度イベント vs 毎月現場, 一人親方の履歴迷子, 幹事と受付とステージの役割分裂

**タイプγ — 相互作用の型（C）**
1人ブラウザ完結, オプトインクラウド保存, 当日多端末Room, 有期限共有URL, Export一発提出, 都度課金イベント, 月額現場, 閲覧のみリンク, 編集1・閲覧多, 終了後自動失効, コアPDF＋Syncは進行だけ

**タイプδ — 任意の第4軸（使うなら）**
ペルソナ（幹事/監督/一人親方/情シス）, 時間スケール（当日3時間/四半期/年）, 成果物（紙・PDF・CSV・画面同期）, 課金の自然さ（都度/月額/席）

# タスク

## Part 0 — 軸の宣言
選んだ軸（2〜4）を名前付きで宣言し、各軸の値を5〜12個列挙。なぜその軸にしたかを2行で。

## Part 1 — 格子から仮説（最低12セル）
軸の積からセルを選び、各セルを次の1行で書け:
`[軸値の組] → 一文ピッチ`

ルール:
- 文中に海外SaaS製品名が必要ならそのセルは捨てる（Mass上澄み）。
- 採択済み3IDの言い換えは捨てる。
- 「保存場所が欲しい」「テンプレ共有したい」だけのセルは捨てる。

## Part 2 — 跳ね度フィルタ
Part1の各案に付ける:
| 案ID | 既存カテゴリ名で説明しきれるか(Y/N) | 噛み合うアセット数 | Bが「保存場所」以外か | 跳ね度(高/中/低) | 残す/捨てる |

跳ね度「中」以上かつ「残す」だけを Part3 へ。最低6案。足りなければ Part1 に戻ってセルを追加。

## Part 3 — プロダクト化（残した案のみ・最大8）
各案:
| 項目 | 内容 |
| プロダクト名（日本語・短く） | |
| 一文ピッチ | |

## Part 4 — Top3 と反証
跳ね度×実現可能性で Top3。各案について失敗シナリオ2つと予防策1行。
うち「今は旗艦E2E以外に1つだけ楔を置くなら」どれか、理由付き。

# 出力形式
