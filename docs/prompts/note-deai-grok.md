# Grok依頼用: note 原稿 — AIっぽさ除去（第2パス）

**用途:** Gemini たたき台 or 提督初稿の**人間味トーン調整**（事実追加・構造変更はしない）  
**位置づけ:** note 執筆パイプライン **第2パス**（第1: `note-editorial-gemini.md` · 第3: 提督事実追記）  
**更新:** 2026-06-21

---

## パイプライン

```text
note-editorial-gemini.md（Gemini 下書き）
  → note-deai-grok.md（本ファイル · Grok で AI 味除去）
  → 提督が実体験1行追記 + operator-profile / statements 突合
  → note エディタに貼付
```

## Grok への依頼文（コピペ用）

```text
あなたは日本語の個人開発マーケター兼編集者です。
以下の条件で、添付原稿から「AIっぽさ」を消してください。

【やること】
- 口語のリズム · 幹事あるあるの具体 · 1人称（カオル）の自然さを上げる
- 同語尾・三段論法・「〜ですよね」連打・締めの定型（「使ってもらえたら嬉しいです」等）を削る
- 冗長な前置き・礼賛・「はじめに」を削る

【やらないこと】
- 事実 · 数字 · 機能 · URL の追加・変更
- 社名 · 実名 · 競合の名指し批判の追加
- 「100%安全」「業界No.1」等の断定追加
- 見出し構成（H2 の数・順）の変更
- 文字数を大幅に増やす（各 H2 は +10% 以内）

【トーン】
- note「引き算の記録」— エモ寄りだがチープくない
- IT営業・マーケ経験の非エンジニア「カオル」
- 製品名: SUGUDASU 班分け（グループ分けと書かない）

【出力フォーマット — この順のみ】
### noteタイトル案（3つ · 各32字以内）
### リード（120〜180字 · 1段落）
### 本文（H2見出し5つ + 各本文 · note に貼れる散文）
### 変更メモ（bullet 3つ以内 · 何を AI っぽくしていたか）

【添付原稿】
（ここに Gemini 出力 or 前稿を貼る）
```
