## 1. 調査の目的（我々の問い）

**現場監督**が抱える「スケジュール管理」「提出物」「検査書類」の**二重入力・版ずれ・再送**を減らし、**省力化**したい。

特に次の仮説を検証したい:

| 仮説 | 内容 |
|------|------|
| **H1** | 監督は最低でも **提出用工程表** と **現場運用用スケジュール** の2系統を持っている（または持たざるを得ない） |
| **H2** | 運用情報（重機入場・資材納入・打設 m³ 等）を提出用ガントに直接書くと **提出物を汚す**ため、別管理になる |
| **H3** | **1つの工程表DB + ビュー/プロパティで表示出し分け**（Notion のプロパティ目アイコン方式）なら、入力は1回で済む可能性がある |
| **H4** | 検査書類の一部は工程表・施工計画と **同じマスタデータ** から自動生成/転記でき、省力化の余地がある |
| **H5** | マス市場（小規模ゼネコン・サブコン現場監督・¥200/月級ツール）では、フル ERP ではなく **軽量工程表 + 提出 Export** が現実的 |

**あなたの仕事:** 上記を肯定/否定/条件付きで評価し、**プロダクト設計に使える具体像**（何を1つにまとめ、何を分けるか）を返す。

## 2. 我々が検討しているプロダクト（比較の軸 · 誤解禁止）

- **NOT:** 大規模 ERP（工事奉行・SAP）· 施工体制台帳 SaaS 全体 · 電子納品の完全代替 · 法規保証ツール
- **NOT:** Microsoft Project / 本格ガント SaaS との正面競争
- **IS:** **Notion タイムライン（左表+右棒）に近い** 日ベース工程表 · 組織 ¥200/月 · 監督10名 · 現場10
  - 1現場 = 1 工程表（工期がチャート範囲）
  - サブアイテム（工区・階・フェーズ等）· 依存矢印 · プロパティ自由追加
  - **ビュー preset**（例: 提出用 / 現場用）で行・列の表示オンオフ
  - xlsx Export · A3 印刷（v1）· Excel Import なし
  - ゲスト（下請け）招待で工区入力を分離可

ペルソナ: **現場監督**（メール+パスワード各自ログイン）· 役所・施主・協力会社へ工程表を出す · 日々の納入・打設・検査調整を現場で回す。
