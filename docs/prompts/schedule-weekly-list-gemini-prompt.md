## Gemini への依頼文（コピペ用）

```text
あなたは日本の建設業（土木・建築・設備）における現場管理と協力会社連携の調査アシスタントです。
礼賛・前置き・長い挨拶・LP コピー・実装コードは不要。**指定フォーマットの表と箇条書きのみ**出力してください。
数値・法令はソース（官公庁 · 業界団体 · 書籍名等）を明記。不明は「要確認」。推測は「仮説」と明記し確定事項と混ぜないこと。

## 1. 調査の目的（我々の問い）

**現場監督**が週次で下請協力会社に配る「週間工程・週間予定」について、
**受け手（下請職長・現場代理人）** と **送り手（元請監督）** の両視点で実態を調べ、
SUGUDASU Schedule の **週間リスト Export（v1.1 候補）** の列・フォーマット・秘匿・FB を確定したい。

**チャート（ガント）をそのまま見せれば足りるか？** を含め、次を検証する:

| 仮説 | 内容 |
|------|------|
| **W1** | 下請はガントより **縦表・箇条書き** を LINE で欲しい |
| **W2** | 必須情報は **top5 列** 程度に集約できる |
| **W3** | **協力会社別1ファイル** が主流 |
| **W4** | 元請のみの情報があり、下請週間から **除外** すべき |
| **W5** | FB は **行コメント・遅延申告** で足り、チャットアプリ化は不要 |
| **W6** | ゲスト **編集** より **コメント＋監督承認** が現場で受け入れられやすい |

**あなたの仕事:** 上記を肯定/否定/条件付きで評価し、**週間リストの列セット・Export 形式・権限モデル** に落とせる具体像を返す。

---

## 2. 我々が検討しているプロダクト（比較の軸 · 誤解禁止）

- **NOT:** フル施工管理 SaaS · 日報アプリ · LINE 代替 · チャット機能の内蔵
- **NOT:** 月間提出用工程表（監理向け）の代替 — 週間配布は **別 Export**
- **IS:** 1現場1工程表DB · 監督が **site ビュー** で編集 · 下請には **週間表 PDF/PNG** または **リスト URL** を配布
  - 行フィルタ: 協力会社（`prop_subcontractor`）単位
  - preset 案: `weekly_sub` — 週間必須列のみ · 人工数等は除外
  - ゲスト招待（v1）· 行コメント・遅延申告（v1.1 候補）
  - 正本は紙/PDF（提出 preset）— 週間は **運用配布** レイヤ

ペルソナ:
- **送り手:** 元請現場監督（週会後に LINE 配布）
- **受け手:** 下請職長 · 現場代理人 · 職方の班長（スマホ · LINE 中心）

## 3. 依頼タスク

### 【依頼1】週間配布の実態マップ（表 · 最低10行）

列:

| 列名 | 内容 |
|------|------|
| 配布物の種類 | 週間工程表 · 重機入場表 · 資材納入表 · 朝礼看板 等 |
| 誰が作る | 元請監督 · 下請自社 · 事務 |
| 誰に出す | 下請職長 · 職方 · 門番 · 生コン工場 等 |
| 更新頻度 | 週次 / 日次 / 変更時のみ |
| 典型チャネル | LINE · 紙 · FAX · メール · 施工管理アプリ · 口頭 |
| 典型フォーマット | Excel · PDF · 画像 · ホワイトボード写真 · URL |
| ガント全体を見せるか | はい / いいえ / 自社分のみ |
| 二重管理パターン | 週間Excel → LINE再送 等 |

**土木・建築を主**に、設備・内装があれば2行追加。

### 【依頼2】下請職長が LINE で欲しい情報 — **top10 列**（表 · 10行）

我々の仮説列（日付 · 工種 · 工区 · 重機 · 資材 · 監督連絡 · 立会 · 道路規制 · 備考 · ステータス）を **優先度1–10** で並べ替えよ。

列:

| 列名 | 内容 |
|------|------|
| 順位 | 1–10 |
| 項目名 | 例: 日付 · 工区 · 重機 |
| 必須度 | 必須 / あると助かる / 不要 |
| 典型の記載例 | 1行 |
| 載せないと起きる事故 | 具体1行（なければ「—」） |
| 我々仮説との差分 | 一致 / 仮説に無い / 優先度ずれ |

表の下に **top5 確定案** を箇条書き5行。
```
