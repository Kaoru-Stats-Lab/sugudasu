| 項目 | 内容 |

## 3. リサーチ依頼

### 3-1. 比較調査（各 5〜10 行で要点）

以下の **「ログイン後 · 個人アカウント設定」** に何を載せているか調査（β 相当 · 請求タブ除くでも可）:

- **Notion** — My account / Security
- **Linear** — Profile / Security
- **Figma** — Account settings
- **Airtable** — Account
- **Basecamp** — My profile（シンプル志向）
- **Slack**（反面教師）— 情報過多の例として

さらに **インフラ・逼迫・制限** の見せ方:

- **GitHub** — Storage / rate limit 警告
- **Dropbox** — 容量バー
- **Notion** — ワークスペース制限時の UI
- **statuspage.io 型** — 障害はアカウント外の Status ページ

### 3-2. 答えてほしい問い（必須）

1. **アカウントページに「利用統計」（アクセス数 · ログイン回数 · 最終ログイン）は載せる製品は多いか？** Sync 幹事に価値があるか？
2. **ストレージ逼迫・quota 上限は「アカウント設定」か「使用中のリソース画面」か「グローバルバナー」か？** ベストプラクティスは？
3. **サービス全体の障害・メンテ（Supabase 停止 · CF 障害）はアカウントページに書くか？** それとも `/status` リンク · バナーのみか？
4. **「あなたは trial · 残り○ルーム枠」はアカウントに出すか、ルーム作成時だけ出すか？**
5. **インフラ逼迫をユーザーに見せることのデメリット**（不安 · 解約促進 · サポート問い合わせ増）と **メリット**（透明性 · データ消失前のエクスポート）を权衡。
6. **個人開発・小規模 SaaS** でアカウントページをミニマルに保った成功例はあるか？
