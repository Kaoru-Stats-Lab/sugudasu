# 要件定義書：フリマ送料・手数料比較（`calc.html`）

> **SUGUDASU 実装時の上書きルール（正本）**
> - 単体HTML完結ではなく、`tools/calc.html` + `assets/sugudasu.css` + `assets/sugudasu-shell.js`。
> - Chart.js 等は使わず CSS のみで棒グラフ（積み上げ帯）。
> - 料金は **JS 内定数モジュール** + **`asOf`（基準日）を画面常時表示**。巨大マスタDBは作らない。
> - コピーは断定弱め（「この前提では〇〇の手取りが最大」）。免責は `<details>` + ツールチップ。

**優先度:** P2（Backlog §10）  
**状態:** 提案採用・未実装

## 1. 概要・思想

メルカリ・ヤフオク・ラクマに同一価格で出品したとき、「どこで手取りが最大か」を **出品前の3秒スピード診断** で比較する。

全運送会社・全便種のタリフを網羅しない。**個人出品者の超頻出4パターン** に送料を固定し、if 分岐で軽量維持する（タリフ地獄回避）。

## 2. 入力UI（3項目のみ）

1. **販売想定価格**（数値）
2. **荷物サイズ（4択ラジオ）**
   - 最薄（定形郵便等）— 各社一律 **100円**（目安）
   - 薄型（A4・厚3cm以内）— メルカリ **210** / ヤフオク **210** / ラクマ **180** 円
   - 小箱（コンパクト等）— 各社一律 **520円**（箱代込み目安）
   - 大型（60サイズ〜）— 各社一律 **750円**（目安）
3. **ラクマ手数料率** — 4.5 / 6.0 / 7.0 / 8.5 / **10%**（デフォルト10%）

**追加（MVP推奨）**

- ヤフオク **Yahoo!プレミアム会員** ON/OFF（ON: 8.8% / OFF: 10%）
- 画面常時: `料金目安の基準日: YYYY-MM`

## 3. 計算ロジック

| プラットフォーム | 手数料 | 手取り |
|------------------|--------|--------|
| メルカリ | 販売価格 × 10% | 価格 − 手数料 − メルカリ送料 |
| ヤフオク | 価格 ×（8.8% or 10%） | 価格 − 手数料 − ヤフオク送料 |
| ラクマ | 価格 × 選択率 | 価格 − 手数料 − ラクマ送料 |

## 4. 出力UI

- 3本の **横向き積み上げバー**（手取り=緑系 / 手数料=赤系 / 送料=グレー）
- 結論行: 「この前提では **〇〇** の手取りが最大（2位より +N 円）」
- `?` ツールチップで各社前提を補足
- 下部 `<details>`: 計算の前提条件と免責（離島・サイズ超過・料金改定非反映）
