## §1 Executive Summary

Notion（2024–2026）の認証 UI は **中央カード · 広い余白 · Progressive Ingestion（段階入力）** と **設定モーダルの左ナビ（Account / Workspace 分離）** が核。幹事ペルソナとのシナジーは高い。

Sync β では **見た目と情報設計だけ借りる**。**マジックリンク主導線 · ワークスペース切替 · 請求タブ · UUID 表示 · Notion 色そのまま** は採用しない。Primary は `--sg-primary`（blue-600）· 退会のみ赤。

最小アカウント設定は Notion の個人設定から **4 要素** — メール · パスワード · ログアウト · 退会（危険ゾーン）。

## §2 Notion — 画面別 UI 要素カタログ

### 1. 新規登録 / ログイン入口（Progressive · メール先行）

* レイアウト: 画面中央の白カード · 周囲余白
* 要素（上→下）:
  1. ロゴ（中央 · モノクロ）
  2. タイトル：「Notionへようこそ」
  3. サブ：「ログインまたはアカウント作成」
  4. OAuth：「Googleで続行」「Appleで続行」（枠線ボタン · 左ロゴ）
  5. 区切り：「または」+ ヘアライン
  6. メール入力（ラベルなしプレースホルダ）
  7. Primary：「メールアドレスで続行」（**Notion は黒ソリッド**）
  8. フッター：続行＝規約同意（**チェックボックスなし** · 文言のみ）

### 2. パスワード入力（メール送信後）

1. ロゴ
2. 「パスワードを入力」
3. サブ：「{email} としてログイン中」
4. パスワード欄（**表示/非表示アイコン**）
5. 「パスワードを忘れた場合」（フィールド近傍）
6. Primary：「パスワードでログイン」
7. Secondary：**「代わりにログインコードをメールで送信」** ← Sync **不採用**

### 3. メール確認 / ログインコード

OTP 貼り付け画面 — Sync の **パスワード主経路** では初回確認メールのみ（ログインコード UI は不要）。

### 4. パスワードリセット（Forgot）

1. 「パスワードの再設定」
2. 説明文
3. メール入力
4. Primary：「再設定リンクを送信」
5. 「ログイン画面に戻る」

### 5. パスワード再設定（メール着地）

1. 「新しいパスワードの設定」
2. 新パスワード · 確認
3. 強度インジケータ（任意）
4. Primary：「パスワードを更新してログイン」

### 6. 設定 IA（Settings & members）

* 全画面オーバーレイモーダル · 左ナビ + 右パネル
* **Account:** My account · Notifications · Connected apps · Language
* **Workspace:** Settings · Members · Upgrade（Sync β では **Workspace 節全体なし**）

### 7. My account（個人）

* 右パネル：アバター · 表示名（インライン）· メール行 + **「Change email」** ボタン
* メール変更：モーダル → **現パスワード** → 新アドレス確認コード

### 8. Security

リスト行 + ヘアライン区切り · 右に chevron または Change / Log out  
パスワード変更 · 2FA · セッション一覧 · 全端末ログアウト

### 9. 退会

My account 最下部 **Danger zone** · 赤「Delete account」  
確認：ダイアログ → **登録メールをタイプ入力** で削除ボタン活性化（要実機確認）

### 10. エラー・列挙対策

* エラー：フィールド直下 or カード上部（赤）
* 存在有無を伏せる：**同一の成功トーン**（「案内を送信しました」）

## §3 Notion — アカウント ID の見せ方

| データ種別 | UI に出すか | Sync β |
|------------|-------------|--------|
| ユーザー UUID | **出さない** | **同左** |
| ワークスペース ID | 設定内の一部 | **ルーム概念のみ** · `event_public_id` は共有用（別論） |
| 表示名 | 出す | **β 任意**（メールで足りる · P2） |
| ログインメール | 出す | **必須** |

**ワークスペース切替 UI:** Sync 1幹事1ルームモデルでは **借りない**。

**個人設定の最小構成（Notion から抜粋）:** メール · パスワード · ログアウト · 退会。

| 製品 | 登録 | 設定 IA | ID 露出 | Sync 適合 |

```text
┌─────────────────────────────────────┐
│         [SUGUDASU Sync ロゴ]         │
│                                     │
│   [ ログイン ]  [ 新規登録 ]  ←タブ   │
│                                     │
│   （将来） [ Googleで続行 ]          │

│   [ ログインする ]  ← blue-600 全幅   │
│                                     │
```

## §7 明示的に借りないもの（Notion から）

1. **ログインコード / マジックリンク主導線**
2. **ワークスペース切替 · Members · Upgrade ナビ**
3. **Primary ボタン黒（#000）** — `blue-600` / `--sg-primary`
4. **β の請求・プラン・カード UI**（S3 · Stripe Portal）
5. **エンドユーザーへの UUID / 内部 ID 表示**
6. **Notion HEX そのまま**（`#0075de` 等）
7. **登録をメールのみ1段階にする Progressive** — Sync は **パスワード同画面必須**
