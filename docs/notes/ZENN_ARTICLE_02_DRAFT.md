### 本文

プログラミングは専門外です。営業や事務の現場で「また同じ書類を打ち直す」と感じたのが、実務ツール [SUGUDASU](https://sugudasu.com/?utm_source=zenn&utm_medium=social&utm_campaign=article_02_hub_devstory) を個人開発したきっかけです。非エンジニアが AI だけで一夜にサイトを完成させた、という話ではありません。**誰が何を書くか** を分けた方が、バグも誇大コピーも減りました。

前回の記事では、見積の下書きを請求書に転用する手順（invoice）を書きました。今回はその裏側 — ツール群の配信と、改善の回し方 — です。

## AI の使い分け — 企画と実装を混ぜない

私が使っているのは、ざっくり次の分担です。

| 役割 | 向く作業 | 向かない作業 |
|------|----------|--------------|
| **Gemini（企画セッション）** | Zenn テーマ表 · 3ヶ月スケジュール · 比較表 | 記事本文 · HTML/JS の実装 |
| **Cursor / Claude（実装セッション）** | コード · changelog 突合 · ドラフト執筆 | 礼賛リライト · 未実装機能の宣伝 |
| **別ツール（監査）** | タイトル短文化 · 冗長削除 | 事実の追加 |

Zenn 連載のテーマ10本は、Gemini に **「表と見出し案だけ」** 出させ、本文は Cursor 側で書いています。企画プロンプトには「記事本文の執筆は禁止」と明記しています。一度、企画と実装を同じチャットに混ぜたとき、存在しない機能まで記事に載りかけたので、役割を分けました。

「魔法のプロンプト1本」より、**出力フォーマット（§1〜§7 の表だけ、など）** と **禁止リスト（礼賛 · 競合名指し · 未公開数字）** の方が効きました。

## Cloudflare Pages を選んだ理由 — サーバーより「預けない」

SUGUDASU は **静的サイト** です。`build-pages.mjs` が HTML・`sitemap.xml`・`robots.txt` を `dist/` に生成し、Cloudflare Pages が配信します。見積ツールの入力は **ブラウザ内の JavaScript** が処理し、下書きは `.json` ファイルとして手元に保存します（前回記事参照）。

### 補足: 静的サイト（SSG）とは？

ページを **アクセスのたびにサーバーで組み立てる** のではなく、**公開前に HTML を焼いておく** 方式です。個人開発では、サーバー代を抑える以前に、**ユーザーの会社名や金額を自社 DB に預けない** 設計を優先しました。現状、Pages Functions（サーバーレス API）は使っていません。機能追加はまず「ブラウザで完結するか」で判断しています。

## Form → changelog — 要望を製品に戻す

改善の窓口は [更新履歴ページ](https://sugudasu.com/updates) の Google Form です。届いた要望を `data/changelog.json` に載せ、ビルドでサイトへ反映します。

たとえば 2026-06-17 には、見積で保存した下書きを納品書・請求書に転用できる旨を、画面上・FAQ・読み込み後の案内に追記しました。ユーザーから「転用のやり方が分かりにくい」という声があれば、まず changelog に事実として残し、必要なら Zenn や X で補足します。

有料のプレスリリース配信は、収益化前の段階では見送りました。代わりに **Zenn（設計・実務の話）** と **X（短い課題フック）** で認知を積み上げています。派手さより、**changelog が動いていること** が信頼になると考えています。

```json
{
  "date": "2026-06-17",
  "type": "improve",
  "title": "見積・請求書：下書きの納品・請求への転用を案内",
  "body": "見積で保存した下書きデータを読み込み、書類種別タブで納品書・請求書に切り替えて転用できる旨を追記しました。",
  "tools": ["invoice.html"]
}
```

## まとめ

非エンジニアが AI と並走するコツは、AI に全部任せないことです。**Gemini には企画表、Cursor には実装と記事、監査は別セッション** — この分担で SUGUDASU は育っています。完成品ではなく、実務で使いながら直しているポータルです。
