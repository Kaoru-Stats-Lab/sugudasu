## 2. 設計原則（サービス設計）

### 2-1. Outcome-first（Rule-first 禁止）

ユーザーが決めるのは **ロジックではなく貼り先（用途）**。変換パイプラインの順序はプロダクト固定。

| 避ける | 採用 |
|--------|------|
| NFKC/半角/ハイフンを10チェックで組み合わせ | **用途プリセット 1 択** |
| 正規表現入力 | 詳細 **最大3トグル**（折りたたみ） |
| 無制限貼付 | **500行 cap** |

### 2-2. 軽量

- 処理は **ブラウザ内1文字列** · サーバー送信なし
- **500行上限**（改行 `\n` でカウント · 空行も1行）
- タブ区切り（スプシ列コピー）は **列数自由** · 行数のみ cap
- 超過: 変換ボタン無効 · 「501行以上は2回に分けて貼ってください」

コピー文言例:

> **500行まで。** Excel から1列コピーした量なら、だいたいそのまま使えます。

### 2-3. 行数一致チェック（Before / After）— **提督FIX · 2026-06-19**

**背景（経験者談）:** おっちょこちょいな人は、中身を見ずコピーすると **行数ズレ＝行ズレ・欠行** で事故る。行数だけでも先に確認させる。

| 要件 | 内容 |
|------|------|
| 表示 | 変換後、目立つ位置に **`入力 N 行 → 出力 M 行`** |
| 一致（N === M） | 緑系 · ✓ 「行数は同じです」 |
| 不一致（N !== M） | 黄/赤 · 「行数が変わっています。意図した変換か確認してください」 |
| コピー | 不一致時は **コピーボタン前に確認**（チェック1つ or 強調ダイアログ · 実装時にどちらか1つ） |
| デフォルト変換 | **行を増減しない**（空行も保持 · 行内変換のみ） |

行数が変わり得る操作（将来トグル）を ON にした場合:

- 事前に「この設定では行数が変わる場合があります」とトグル横に注記
- 不一致時の警告は同じ

**プレビュー:** 先頭 **5行** Before / After 並列（全文 diff は v2 · メモリ節約）

**変更サマリ（任意 · 軽量）:** 「124箇所変更（英数48 · 空白76）」— 行数チェックの補助。v1 は行数必須 · 箇所数は余力あれば。

## 4. 変換ロジック（実装正本 · v1）

**方針:** 決定的 · 辞書なし · AIなし · **行境界 `\n` は原則維持**

### 4-1. 英数字

- 半角: 全角 `Ａ-Ｚａ-ｚ０-９` → `- 0xFEE0`（カナは v1 ではプリセット外 unless 明示拡張）
- 全角: 逆
- NFKC: v1 では **英数半角プリセット内のみ部分適用**可。全体 NFKC は非推奨

### 4-2. 空白

- `U+3000` → `U+0020`
- 行頭行末 trim（`csv_roster` ON 時）
- 行内連続スペース: v1 では **潰さない**（行数・列位置事故防止）。v2 でオプション可

### 4-3. ハイフン統一（トグル ON 時）

対象を半角 `-` に: `‐` `‑` `–` `—` `－` 等（固定リスト）

**長音保護（B02 · Gemini A02）:** 直前文字がカタカナ（`[\u30A0-\u30FF]`）の `ー`（U+30FC）は **置換しない**。

### 4-4. 不変条件

- タブ `\t` は **列境界として維持**
- 改行 `\n` の **個数は原則不変**（§2-3）
- **先頭ゼロ（B04）:** 一切数値化しない · 出力も string のまま
- 意味・正誤・社名表記の「正しさ」は **保証しない**（FAQ）

### 4-5. 貼付時スキャン（Phase A · `SUGUDASU_OOPS_GUARDRAILS.md`）

| 検知 | UI |
|------|-----|
| セル内改行疑い（A01） | 1行にタブなし改行が複数 → 黄バナー |
| 置換文字 `�` · 疑わしい `?` 連続（E02） | 赤バナー · 該当行をプレビューで強調 |
| 先頭ゼロ行（B04） | 情報バッジ「0始まりコードを維持します」 |

## 5. UI フロー

```
[入力 textarea]     247 / 500 行
[貼付警告バナー]    （該当時のみ）
[用途] ○ EC  ○ CSV名簿  ○ 全角英数
[▶ 変換]            ← プレビュー更新用（コピーは下）
[行数]  入力 247 行 → 出力 247 行  ✓
[変更]  23 箇所（任意 · v1.1）
[詳細 ▼] 3トグル
[Before | After プレビュー 各5行 · 変更箇所ハイライト v1.1]
[コピー（変換してから clipboard）]  [入出力入れ替え]
[コピー成功] 緑フラッシュ · Copied! · 2sロック · フィルター注意1行
```

- 入力のたびに行カウンタ更新（変換前）
- 501行以上: カウンタ赤 · 変換不可
- **C01対策:** コピーボタン = **最新設定で再変換 → clipboard**（入力欄の生テキストをコピーしない）
- **競合UX回避:** 変換ボタン押下だけでは clipboard を触らない

## 9. FAQ（うっかり防止 · v1 掲載分）

1. **Excelに貼り戻したら行がズレた** — セル内改行・フィルター/非表示行を確認。本ツールは行数を変えない設計。
2. **長音がハイフンになった** — カタカナ直後の `ー` は保護。型番のハイフンのみ統一。
3. **先頭ゼロは消える？** — 消えない。数値化しない。
4. **下の方の文字化けが不安** — 貼付時に `�` 等を検知したら上部に警告。
5. **行数が1行ズレ** — ヘッダー行を含めてコピーしていないか、Excel側の行数とバッジを突合。
6. **変換前をコピーしてしまった** — 「コピー」は常に最新変換結果を clipboard に入れる。
