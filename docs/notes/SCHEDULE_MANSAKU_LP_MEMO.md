## 1. 比喩の芯

SHIROBAKO で「万策尽きた」に近いのは、だいたい次の **表と版の泥沼のあと** です。

- 話数・工程の連鎖崩れ · 外注への板引き組み直し
- どの進行表が正か分からない · 一部スタジオだけ古い版
- 放送日は動かない · そのうえ交渉・別ルート・人に頼る

**本田さんに相談する · 仲間で何とかする** — ここは製品の置き換え対象ではない。  
SUGUDASU が置けるのは **その直前**:

| 万策尽きたの「前」 | 万策尽きたの「本番」（人） |
|--------------------|----------------------------|
| 昨夜ずらしたのに古いExcelが回っている | 外注との交渉 |
| 手動リスケで2時間溶けた | 本田さん・プロデューサーへの相談 |
| 正本が分からず確認地獄 | クリエイティブ判断 · 品質トレードオフ |
| 紙掲示とデータのズレ | 朝礼で危機感を共有する |

**LPの一文:**  
> 万策尽きた、の前に。表の手直しと版の食い違いは、道具で減らす。相談する体力を残す。

## 2. LPコンセプト評価

### ◎ 強い点

- **Motivation と一致** — 「便利になりたい」ではなく「また同じところで詰まりたくない」
- **Excel卒業と差別化** — Excelの延長 · 正本1本 · 印刷+URL併用と矛盾しない
- **SHIROBAKO層に刺さる** — アニメ進行 · 長期多下請 · 映像の Schedule 寄り枝
- **紙媒体とも接続** — 刷り直し前の正本1本（[`design-paper-lp-waitlist-spec.md`](design-paper-lp-waitlist-spec.md)）と同型の「事故の前」

### △ 注意点

| 論点 | 扱い |
|------|------|
| **建設 Sync LP の主 H1 にしない** | 現場監督ペルソナは SHIROBAKO 文脈を共有しない · 正本は建設語彙 |
| **作品・キャラ名の商標** | 製品LPの主見出しに「あおい」「本田」は避ける · 比喩は本文・note/Zenn向け |
| **過剰約束** | 「万策尽きなくなる」禁止 · **一部の原因を減らす** のみ |
| **検索SEO** | `万策尽きた` は意図が広すぎ · LPは比喩 · KWは `進行表` `工程表` 系 |

### 使い分け（MECE）

## 3. LP文案案（映像・アニメ枝 · Hold）

### ヒーロー

- **H1:** 万策尽きた、の前に。
- **サブ:** 板引きの手直しと、版の食い違いだけ。道具に任せる。
- **リード:** 放送日は動きません。だから、進行表の正本とリスケ連動を先に整える。

### ペイン4（箇条書き）

1. 話数がずれたのに、外注だけ古い進行表  
2. 板引き組み直しで夜が潰れる（手動リスケ）  
3. どのExcelが正本か、口頭確認地獄  
4. そこで体力を使い、本当に相談すべきときに「万策尽きた」

### やること / やらないこと

| やる（検討中） | やらない |
|----------------|----------|
| Excelから取込 · 1行変更で後続連動 | クリエイティブ判断の代行 |
| 閲覧 URL1本 · 外注への最新版 | 交渉 · 発注 · 品質管理 |
| 印刷出力（進行表の手触り） | 香盤表・分刻み当日表（→ Timeline 別） |
| 初校/話数/工程の **現場語** | チケット · WBS · IT語 |

### 誠実線（必須）

- 未リリース · 需要調査または Sync 準備中と明記  
- 「相談不要になる」ではなく **相談に使う時間を残す**  
- SHIROBAKO は **インスピレーション** · 公式コラボではない

### CTA 案

- 制作進行の方へ — 3分アンケート（フォーム or 将来）  
- または guides へ — `制作進行表をExcelで回すときの落とし穴`（未着手）
