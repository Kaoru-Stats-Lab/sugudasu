### リード（メモ）

- スプシから1列コピー → どこかの **全角半角変換** に貼る → Excel に戻す、は日常業務。
- 高リテラシーほど **中身を見ずコピー** する（経験者談 · OOPS_GUARDRAILS 前提）。
- Gemini に「うっちょこ事故カタログ」を洗わせたら、**行ズレ・データ変質・クリップボード先祖返り** の3型に集約できた → それを潰す normalize を作った、という流れ。

### H2 案（Gemini リサーチ × 自社解消の対応表が記事の芯）

1. **三大事故タイプ**（表1枚 · 読者が「あるある」と認識する部分）
   - ① 行・行対応のズレ（空行 · セル内改行 · 分割貼付）
   - ② 意図しない変質（NFKC · 長音→ハイフン · 先頭ゼロ · 姓名スペース）
   - ③ クリップボード先祖返り（変換前コピー · 変換忘れ · 履歴ソフト）

2. **一般的なWeb変換ツールで起きがちなこと**（Gemini調査の「他サービス側」· 固有名なし）
   - テキストを **サーバーに送る**（機密列・会員ID列の心理障壁）
   - **NFKC一括** · ルールをユーザーに任せる（丸数字 · 長音 · 型番）
   - 変換と同時の **自動クリップボード上書き**（確認チャンスなし）
   - **行数が変わっても気づけない**（空行削除 · trim しすぎ）
   - 500行超を **無制限**（ブラウザ固まり · 分割漏れ）

3. **SUGUDASU で採った解消**（事例ID ↔ 実装 · 1表 + 短い段落）
   | 事例 | 他でよくある | normalize の解消 |
   |------|-------------|------------------|
   | A01 | セル内改行で行が増える | 貼付スキャン警告 · 行数バッジ |
   | A02 | 空行削除で貼戻しズレ | **行数不変**設計 · N→M 表示 |
   | B01 | NFKCで記号が壊れる | NFKC全体 **禁止** · 英数帯のみ |
   | B02 | レーザーの「ー」が `-` | カタカナ直後長音 **保護** |
   | B04 | 00123 → 123 | **string固定** · 数値化しない |
   | C01 | 変換前をコピー | コピー = **最新設定で再変換** |
   | E02 | Mac↔Win 化け | `�` 検知バナー |
   | — | 501行一括 | **500行 cap** + 2回に分ける案内 |

4. **Outcome-first — ルール10個より用途3択**
   - EC登録 / CSV名簿 / 全角英数 — 裏のパイプラインは固定（NORMALIZE spec §3）
   - 「正規表現いらない」が記事の実務パート

5. **横展開 — normalize から全ツールへ**
   - `sg-copy-feedback.js` · `DESIGN_GUIDELINE.md` §3.8
   - warikan / invoice / fair-draw へ同型コピー契約（1段落 · #2 開発記事への内部リンク）

6. **やらないこと · Phase B**
   - 全文 diff · AI判定 · CSV列操作（T06 は別）
