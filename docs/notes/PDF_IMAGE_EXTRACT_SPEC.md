## 1. 製品境界（Job）

| 向く | 向かない |
|------|----------|
| 仕様書・提案書から **図・スクショ・写真** を抜いてスライド / チャットへ | OCR・テキスト抽出 |
| 社外秘PDFを **アップロードせず** 手元で抜く | PDF圧縮・結合・分割・墨消し |
| **埋め込み画像がある** PDF | ベクターのみ・文字のみ（**抽出0件になりうる**） |

**重要（境界）:**  
PDFの種類によっては画像が埋め込まれておらず、**1枚も抽出できない**ことがあります。また、見た目が画像でも内部が図形描画だけの場合は対象外です。

**ユーザー向け一言:**  
「埋め込まれた画像を端末内で取り出します。ページ全体の見た目コピーではありません。PDFファイル自体はサーバーに送りません。」

**導線:**

```text
PDF → 本ツールで画像抽出 →（任意）透かし → 保存
```

## 2. v0.1 Must / OUT

### Must

| 項目 | 内容 |
|------|------|
| 入力 | PDF **1ファイル** · ドロップ / クリック |
| ページ幅 | 一度に処理できる幅は TECH の `MAX_PAGES`。**自動で部分処理はしない**。総ページが幅を超えるときは **開始ページのみ** ユーザー指定。終了は `min(開始 + MAX_PAGES - 1, 総ページ)` |
| 開始ページ | **1〜総ページ数** の整数のみ入力可。範囲外・空・非数は **即時エラー** とし、**抽出ボタンは無効**（黙ってクランプしない） |
| 抽出対象 | PDFに **埋め込まれた画像**（ページ全体ラスタ化ではない） |
| 出力形式 | **自動選択**（元データをJPEGのまま維持できる場合は JPEG。それ以外は PNG）。UIで形式バッジを表示 |
| ファイル名 | `{元ベース名}_p{ページゼロ埋め2桁}_img{連番ゼロ埋め2桁}.{png\|jpg}` 例: `sample_p03_img02.png` |
| ZIP名 | `{元ベース名}_p{開始ゼロ埋め3桁}-{終了ゼロ埋め3桁}_{件数}img_{HHmmss}.zip` 例: `sample_p040-089_7img_095012.zip`（同一範囲の再抽出でも衝突しにくくする） |
| 保存 | 単枚DL · **ZIP一括**（watermark の ZIP 実装を流用可） |
| UI | サムネ一覧 · **「N件抽出」を常時表示** · 各サムネに **出現ページ**（例: p.3 · p.3, p.5）· **JPEG/PNG バッジ** |
| Dedup | **ダウンロード用バイトの重複を防ぐ**。一覧では同一画像でも出現ページを併記して「消えた」誤解を防ぐ |
| 小画像 | 既定で **短辺 16px未満 かつ 面積 256px²未満** を除外。トグルは v0.1 OUT。FAQで説明 |
| 非送信 | `data-sg-privacy-badge` · `data-subject="PDF"` |
| PC | Desktop-first バナー必須 |
| エンジン読込 | **当該ツールのみ**動的読込（サイト全体に載せない） |
| 依存 | 公式 PDF エンジン（Mozilla pdf.js / `pdfjs-dist`）のみ。追加ラッパーパッケージは v0.1 に入れない |

### OUT（v0.1）

- ページ全体を画像化（**v0.2 候補**）  
- PDF圧縮・結合・分割・墨消し・OCR  
- 複数PDF一括 · フォルダ階層  
- 抽出と同時の透かし焼き込み（完了後に `/watermark` へ誘導）  
- 小画像フィルタのUIトグル  
- CMYK / ICC / ソフトマスク / Form内ネストの完全対応（ベストエフォート · 期待値は §4）  
- 終了ページの手動入力 · 自動で次の幅へ連続処理

## 3. UI（必須体験）

1. PDFを選ぶ  
2. **総ページ ≤ `MAX_PAGES`:** 従来どおり抽出へ進む（範囲UIは出さない）  
3. **総ページ > `MAX_PAGES`:** 開始ページ入力（既定 **1**）· 処理範囲のリアルタイム表示 · **「{開始}〜{終了}を抽出」** ボタン（単なる「抽出」は禁止）。終了は自動計算  
4. 処理後、見出し級で **「18件抽出」** のように件数を出す（0件も明示）  
5. サムネ一覧：プレビュー · ファイル名候補 · **出現ページ**  
6. 単枚保存 / ZIP  
7. 空・範囲・色の注意は画面上または FAQ で読める  

### 範囲表示（総ページ > `MAX_PAGES` 時）

```text
全186ページ

開始ページ
[1]
────────────
抽出範囲
1〜50（50ページ）
────────────
[1〜50を抽出]
```

開始を変更すると終了・件数・ボタン文言を即時更新する。文書末尾では幅が短くなる（例: 総186 · 開始170 → `170〜186（17ページ）` · `[170〜186を抽出]`）。

開始が **1〜総ページ数** の外（0 · 負数 · 総ページ超 · 空 · 非数）のときはエラー表示し、抽出ボタンを無効化する。有効値へ黙って書き換えない。

## 4. 失敗・注意のユーザー文言

| 状況 | 文言（要旨） |
|------|----------------|
| 画像0件 | 「埋め込み画像は見つかりませんでした。文字・図形だけのPDFや、画像が別の持ち方の場合があります。」 |
| 小画像除外 | 「ごく小さい画像（短辺16px未満かつ面積がごく小さいもの）は除外しています。」 |
| 開始ページ不正 | 「開始ページは 1〜総ページ数 の整数で指定してください。」 |
| ページ幅超過（UI） | 範囲選択を表示。自動では処理しない |
| ファイルサイズ超過 | 「このPDFは目安の上限を超えているため処理できません。（部分的な抽出はしません）」 |
| 読込失敗 | 「このPDFは開けませんでした（暗号化・破損の可能性）。」 |
| 色空間 | 「一部の画像は色が正しく出ないことがあります（印刷向けの色空間など）。」 |
| メモリ等 | 「処理中に止まった場合は、ページの少ないPDFや画像の少ないPDFで再度お試しください。」 |
| メモリ（FAQ） | 「50ページ以内なのに途中で止まりました」— 画像枚数・解像度で完了しないことがある。範囲を絞って試す |
| ページ数（FAQ） | 「ページ数が多いPDFはどうなりますか？」— 目安50ページ幅 · 開始指定 · 複数回に分ける |
| ZIP（FAQ） | 「範囲を分けて抽出すると、ZIPファイルはどうなりますか？」— 範囲ごとに別ZIP · ファイル名にページ範囲 |
| 見た目はあるが0件 | 「ページに図が見えても、埋め込み画像ではなく図形として描かれている場合があります。その場合は抽出できません。」 |

## 6. 受け入れ（DoD）

1. 埋め込み画像付きサンプルから、目視で分かる図が1枚以上出る  
2. 一覧の件数表示が結果と一致する。同一画像の出現ページが併記される  
3. **自動で部分処理はしない。ユーザーが指定した範囲だけ処理する**（総ページ ≤ 幅のときは全ページ＝指定相当）  
4. 出力ファイル名・ZIP名が §2 の形式に従う（同一範囲の再抽出でも ZIP 名が衝突しにくい）  
4b. 開始ページが不正なとき抽出できず、エラーが出る（黙ってクランプしない）  
5. PDFの内容がサーバーへ送信されない（ページ読込・解析等の通信は約束ページ方針どおり別）  
6. Desktop バナー · privacy-badge · FAQ（0件 · ページ数範囲 · メモリ · 非送信 · 小画像 · ZIP · 透かし）  
7. TECH 記載の単体テストが exit 0
