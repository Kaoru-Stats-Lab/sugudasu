### 本文

新入社員研修のブレイクアウト、社内ハッカソン、納会の円卓替え — どれも「班分け」ですが、**現場で困るポイントは全然違います**。私は納会では事業部と職種をシャッフルして円卓を決め、ハッカソンでは「各チームにエンジニアを1人以上」を守りながら組み直し、アイディアソンでは班分けそのものは大きな問題にならなかった一方、エンジニアリングのイメージがない参加者への役割説明に苦労しました。

この記事では、**名簿を外部に送らずに班を決め、Slack や Excel にそのまま貼れる** 静的ツール [SUGUDASU group-split](https://sugudasu.com/group-split?utm_source=zenn&utm_medium=social&utm_campaign=article_14_group_split) の設計と、幹事が押さえるべき線引きを書きます。進行表の同期設計で言う **「混ざっても読める」** に近い発想を、班分けでは **「後から説明できる公平」** と呼んでいます。

## 班分けあるある — 新入社員 · ハッカソン · 共有

| シーン | 幹事の本音 | Excel 手作業のつらさ |
|--------|------------|----------------------|
| 新入社員研修のブレイクアウト | 同じ部署ばかり固まらないでほしい | `RAND()` は F9 で班が変わる · 制約はマクロ |
| ハッカソン | 各チームにエンジニアがいないと動かない | 職能列の転記ミス · 欠席で即崩れる |
| 当日の共有 | Slack に一覧を貼って終わらせたい | 表の整形に15分 |

「ランダムに分ける」だけなら、[Online Stopwatch](https://www.online-stopwatch.com/random-name-pickers/) のようなグループ生成サイトでも足ります。**教室の演出**には向いています。人事名簿を貼る用途では、**名簿の行き先** と **再現説明** と **2次利用（TSV/Slack）** が別問題になります。

## 同じ班分けでも、納会・アイディアソン・ハッカソンで重さが違う

| | 納会・円卓 | アイディアソン | ハッカソン（オンライン） |
|---|------------|----------------|--------------------------|
| 班分けの難度 | 低 | 低 | **高** |
| 制約 | 所属を分散 | 職能は hard にしない | **各組にエンジニア必須** |
| 欠席 | 料理・グラスで吸収 | ある程度吸収 | エンジニア0人 → **即再編** |
| 当日の主役 | 幹事 | ファシリ | **スイッチャー**（BR · Slack） |

納会では欠席しても卓の再編はほとんど問題になりません。ハッカソン（特にオンライン）では、班を決めたあとも **Zoom のブレイクアウトルーム（BR）への案内** と **Slack の部屋割当** が残ります。Zoom の画面には所属や職能が出ないので、**班番号と BR 名の対応表** を持った「スイッチャー」が、ホスト画面で人を動かす役割になります。班分けツールはここまで自動化しませんが、**幹事が渡す正本（TSV + シード）** がないとスイッチャーも迷子になります。

アイディアソンは班分けより **「班に入ったあと何をするか」** の方が難しいことが多いです。ツールが解くのは **「誰をどの組に」まで** です。

## Excel RAND() とシード付きシャッフル — 何が違うのか

「Excel でできないの？」は半分 Yes です。補助列に `RAND()` を入れ、並べ替えれば、一見ランダムに班を作れます。問題は **正本が残らない** ことです。

| | Excel `RAND()` | シード付きシャッフル（group-split） |
|---|----------------|-------------------------------------|
| 再実行 | F9 · 再計算で **班が変わる** | 同じ名簿 + 同じシード → **同じ班** |
| 説明 | 「たまたまこうなった」になりやすい | シード値 + 名簿指紋で **ルールの結果** と説明 |
| 制約 | VBA · 手作業が増える | 固定班 · 離すペア · 属性分散/各組必須 |
| 共有 | 表をそのまま渡す | **TSV · Slack · 告知文** を1クリックコピー |

`RAND()` は **ランダムさ** は出せます。幹事が本当に欲しいのは、クレームが来たときに **「なぜこの班か」を説明できること** です。Napster と iTunes の比喩ほど遠くなく、**Excel RAND とシード付きシャッフル** の違いが、現場ではこの差です。

group-split では名簿をシード付きでシャッフルし、画面にシード値と名簿の指紋（SHA256 の先頭）を出します。**100%公平の魔法**ではなく、**説明できる公平**を狙っています。

### 補足: シード値とは？

**シード（seed）** は、乱数の出発点になる数です。同じシードなら、同じ入力からは同じ並びが再現されます。班分けでは「幹事が決めたルール + このシード」の結果として共有する、という使い方です。中身を手で計算する必要はありません。

## ルーレットサイトと役割が違う

| | Excel RAND | ルーレット系サイト | group-split |
|---|------------|-------------------|-------------|
| 名簿の行き先 | ローカル | **サーバーへ送信されることが多い** | **ブラウザ内のみ** |
| 2次利用 | 手動 | 演出中心 | **TSV · Slack · 告知文を1クリックコピー** |
| 制約 | マクロ | 均等分割が中心 | 固定班 · 離すペア · **属性の分散/各組必須** |

研修名簿を海外のサーバーに送りたくない場合、F12 のネットワークタブで **名簿貼付時に POST が出ないか** を確認するのが手っ取り早いです。SUGUDASU は静的 HTML + ローカル JS のみで、名簿データの POST は出ません（競合を「使うな」とは言いません。**用途で選ぶ**話です）。

## ハッカソン名簿の職能列 — ツールの前にやること

ハッカソンで「各組にエンジニア1名」をかけるなら、名簿の職能列は幹事が事前に確定してください。

| 避ける | 代わりに |
|--------|----------|
| 「ITに詳しい」「なんとなく触れる」 | **エンジニア / UI / 企画** など幹事が正規化 |
| エントリー自由記述だけ | Excel 1列にまとめる（[文字列正規化](https://sugudasu.com/normalize) 経由でも可） |

エンジニアの人数が組数より少ないときは、数学的に全組を満たせません。group-split v1.2.3 では **「可能な限り実行」** モードを選べ、満たせない組だけ結果に表示します。厳格に止めるか、1組空けてでも当日の班を出すかは **幹事の判断** です。

アイディアソンでは各組必須は不要です。所属の **分散** だけで十分なことが多いです。

## 2次利用 — TSV と Slack が主役

結果は画面で見せて終わりではなく、**Excel 用 TSV** と **Slack 用テキスト** を1クリックコピーする設計です。200名超はチャットより TSV 推奨のソフト警告を出します（上限 250名）。

欠席が出たら、結果画面で名前をタップして除外し、**同じシードで再構成** できます（M02 Resilience）。名簿テキストも同期更新されるので、会場のスマホからでも組み直しやすくしています。

## オンラインならスイッチャーが持つもの（Zoom API は使わない）

group-split は Zoom / Teams の BR を自動作成しません。代わりに、幹事が事前に揃えると当日が楽になるものがあります。

1. **班 ↔ BR 対応表**（例: グループ1 = BR-A）
2. **班 ↔ Slack**（例: `#team-01`）
3. **班分け正本**（TSV + シード）
4. 欠席時は幹事が再実行 → スイッチャーへ **差分1行** を DM
5. **2画面** — Zoom + 班表（スマホ可）

製品側では今後、BR 名・Slack 名を列に足した **スイッチャー対応表** のコピー出力（M13）を検討しています。

## やらないこと

- Zoom ブレイクアウトの API 連携（スコープ外）
- 出欠管理 · 座席のリアルタイム共同編集
- ルーレット演出（Online Stopwatch の得意分野は奪わない）
- 「AI が相性のいい班を100%保証」

## まとめ

班分けは **イベントの種類で難度が変わります**。納会は軽く、ハッカソンは職能制約と欠席再編とスイッチャーまで含めて重い。`RAND()` のたまたまではなく、**説明できる公平** — 名簿を預けず、制約付きで割り、Slack に貼れる形で渡す。その事務作業に特化したのが group-split です。
