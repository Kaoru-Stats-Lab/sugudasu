## 1. ペルソナから逆算

### 1-1. 主ペルソナ

| ID | 本機能での姿 |
|----|----------------|
| **P-B** | 契約文・規程・社外秘メモを AI に直させたあと Diff で確認。片方だけ Excel 経由で全角スペースが混ざり、**中身は同じなのに差分だらけ**になる |

### 1-2. Job → 仕様

| # | 声 | 仕様 | 満たさない |
|---|-----|------|------------|
| J1 | 「全角スペースだけの差分で見づらい」 | 任意 ON で比較前に空白系を揃える | 常時強制 |
| J2 | 「入力欄の原文は残したい」 | **textarea は書き換えない** | blur 置換 |
| J3 | 「数字・日付差は落としたくない」 | 空白系のみ | NFKC · normalize 移植 |
| J4 | 「設定を読みたくない」 | チェック **1つ** · 既定 **OFF** | トグルの山 |
| J5 | 「社外秘を送りたくない」 | ブラウザ内 · 非送信 | サーバー正規化 |
| J6 | 「いま無視モードか分かりたい」（レビュー追加） | ON 時ステータス注記 **Must** | 黙って結果だけ変える |

### 1-3. 成功 / 失敗

**成功:** チェック ON →「危険な変更を確認」→ 空白ゆれだけの差分が減り、数値・日付等の要確認が残る。ON 中である旨が画面に出る。  
**失敗:** 原文が書き換わる / 全角英数まで消える / normalize と画面が混ざる / ON に気づかず「差分なし＝完全一致」と誤認する。

## 3. Must（v0.1）

### 3-1. UI

配置: 「危険な変更を確認」ボタン行（既存操作列）にチェック1つ。大レイアウト変更なし。

| 要素 | 内容 |
|------|------|
| id | `diff-cleanse-ws` |
| ラベル | **全角・行末スペースを無視して比較** |
| 補足（ラベル横 or 下 · 1行） | `入力欄の文字は変わりません` |
| 既定 | **OFF** |
| LocalStorage | **しない** |

### 3-2. 適用タイミング

1. 「危険な変更を確認」クリック  
2. 既存 `normalizeInput`（CRLF → LF）の後  
3. チェック ON なら before / after に `normalizeWhitespaceForDiff` を適用  
4. その結果だけを `computeChanges` へ  

**textarea の `.value` は更新しない。**

### 3-3. `normalizeWhitespaceForDiff`（比較専用 · 旧名 cleanseWhitespaceForCompare）

**対象は実務で頻出する空白のみ。全 Unicode 空白クラスは対象外。**

行を維持したまま:

| # | 処理 | 理由 |
|---|------|------|
| 1 | `U+3000` → `U+0020` | Excel 全角スペース |
| 2 | `U+00A0` → `U+0020` | Word / Web NBSP |
| 3 | 文中・行末を含む `\t` → `U+0020` | Excel セル由来タブ（Gemini） |
| 4 | 各行末尾の半角スペースを trimEnd | Word/Outlook 行末ノイズ（多数決 Must） |
| 5 | 連続半角スペースは畳まない | 字下げ・表の意図 |

**行数は変えない**（空行は残す）。Figure Space 等は触らない。

### 3-4. ON 時ステータス注記（Must）

`#diff-status`（または同等の既存ステータス行）に、比較実行後かつチェック ON のとき:

> 空白のゆれを無視して比較しています。入力欄の原文には全角スペース等が残っている場合があります。

差分 0 件でもこの注記を出す（「動いていない」誤認防止 · Gemini）。

### 3-5. FAQ（Must · 同リリース · 1問）

**Q:** 「全角・行末スペースを無視」すると何が消えますか？  
**A:** 比較のときだけ、全角スペース・一部の特殊空白・タブ・行末の余分な半角スペースの違いを揃えます。入力欄の文字は変わりません。数字・日付・文言の違いは残ります。貼り戻す前に本文自体を整えたい場合は「全角半角整え」を使ってください。

### 3-6. 非送信

既存 Diff と同じ。

## 4. Should / OUT

### Should（やらなくてよい）

なし（v0.1 は Must のみで閉じる）。

### OUT

| やらない | 理由 |
|----------|------|
| LocalStorage でチェック記憶 | 常時 ON の見落としが危険（4者一致） |
| NFKC · 英数全角半角 · ハイフン · 日付統一 | normalize の領域 · 防波堤 |
| `text-normalize.js` の import | 数十行の局所関数で足りる |
| 全 Unicode 空白の網羅 | 論争・肥大化防止 |
| 文字数カウンタ · 差分行ツールチップ · 初回トースト | 薄い拡張の範囲外 |
| textarea 書き換え · 入力都度自動解析 | 既存 DECISION 維持 |
| 新規 HTML / id · LLM | — |

## 5. 実装スケッチ

```js
function normalizeWhitespaceForDiff(text) {
  return String(text ?? '')
    .replace(/\u3000/g, ' ')
    .replace(/\u00a0/g, ' ')
    .replace(/\t/g, ' ')
    .split('\n')
    .map((line) => line.replace(/ +$/g, ''))
    .join('\n');
}
```

単体テスト:

1. `A　B` vs `A B` → OFF 差分あり · ON 差分なし  
2. `金額 100` vs `金額 200` → ON でも数値差分あり  
3. `foo\tbar` vs `foo bar` → ON で揃えられる  
4. ON 後も textarea 文字列不変
