# SUGUDASU Sync 共通管理画面（仕様・UIモックアップ）

## 0. デザイン原則・設計思想

* **ストイック・ミニマリズムの徹底：** アバター画像、本名（ユーザー名）、部署名などの個人情報は1ミリも取得・表示しない。情シス監査をノーパスで突破する「無機質な暗号制御室」に徹する。
* **2層分離構造（疎結合）：** デバイス・決済を司る「不変のインフラ」を固定し、生存中の暗号化データをプロダクト別に「可変のセッション」として動的プラグインする。
* **ログイン・フリー（JIT性）：** IndexedDB内のマスター鍵の有無を静的JSが自動判定し、アクセスした瞬間にログインボタンなしで0.1秒でせり出すUIとする。

## 2. 各コンポーネントの機能・UIコピー仕様

### ① 画面ヘッダー

* **Master ID（識別子）：**
IndexedDB内の128bitマスター鍵（2SKDの片親）のハッシュ値の冒頭・末尾のみを無機質に表示。文字列クリックでクリップボードにコピー可能とする。
* **接続ステータス：**
SupabaseのRealtime（またはRPC通信）が生存している場合、`[ 🟢 接続中 ]` と小さく常時明示する。

### ② クラウド・セッション管理（可変レイヤー）

* **プロダクト別のグルーピング：**
Supabaseの共通テーブル `[ userId, productId, sessionId, encryptedBlob, expiresAt ]` からデータをフェッチし、`productId` ごとに自動でグループ分けしてネスト表示する。
* **自動消滅カウントダウンタイマー：**
`expiresAt`（TTL時刻）までの残り時間を、フロントエンドのJSで秒単位でリアルタイムに減算カウントダウン表示。
* **[ ⚠️ サーバーから完全強制削除 ] ボタン（爆破装置）：**
クリック時、確認ダイアログなし（または1クリックのシンプルトグル）でサーバー上の該当レコードを即座に物理削除（`DELETE`）する。「タブを閉じ忘れた」というユーザーの恐怖をいつでも一撃で消滅させる特権を与える。

### ③ 認証済みデバイス・暗号鍵の管理（不変インフラ）

* **端末名の自動判別：**
個人を特定する名前は入力させず、ブラウザの `navigator.userAgent` から「OS名 / ブラウザ名」を自動抽出して表示。
* **[ ❌ 鍵を剥奪 ] ボタン（遠隔キック）：**
会議室のPCなど、出先でログアウトし忘れたブラウザのアクセス権（セッションキー）を、手元の端末から一発で無効化する。
* **[ 📱 新しい端末を安全に同期する ] ボタン：**
クリックするとモーダルが開き、1Password式の「128bitのSecret Keyを含んだ使い捨てQRコード」を表示。新端末のカメラで読み取らせることで、インターネット通信を1ミリも介さずに「光（カメラ）」で鍵の搬送を3秒で完結させる。

### ④ プラン・決済管理

* **1ビットの生存証明：**
SUGUDASU側はクレジットカード番号や住所を一切保持しない。Stripeから返ってくる「サブスクリプション有効（Active）」のステータスフラグだけをフロントエンドで受け取り、契約期限のみをテキストで静的に1行表示する。
* **[ 領収書・決済情報の管理 ↗ ] リンク：**
クリック時、Stripeが公式に提供する「Customer Portal（顧客ポータル）」の外部URLへ直接リダイレクトさせる（F3：重いサーバー機能は自前で持たない）。解約手続きやカード変更、インボイス領収書のダウンロードはすべてStripe側で安全に行わせる。
