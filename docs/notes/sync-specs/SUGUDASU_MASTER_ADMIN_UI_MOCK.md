# SUGUDASU マスター管理画面（仕様・UIモックアップ）

**対象：運営者（提督）専用コントロールパネル**

## 0. 設計原則・開発思想

* **ゼロデータ・テレメトリー（不覗の統治）：** ユーザーの生データや個人情報（メールアドレス等）を覗く機能を技術的・構造的に完全排除。運営者自身のリスクと管理コストを $O(0)$ にする。
* **インフラレス認証（URLハッシュ駆動）：** サーバー側に「管理者アカウント」や「ログイン画面」の仕組みを一切作らない。URLの `#master_key=` に埋め込まれた秘密鍵を用いて、静的HTML（ブラウザ）側でSupabase/Stripeの読み取り専用APIキーを一瞬で復号・フェッチする。
* **クライアントサイド・アグリゲーション：** 管理用の重いバックエンドサーバーは用意しない。提督のブラウザが、直接各SaaSのAPIから数字を引っ張ってきて画面上でガッチャンコする「静的1枚絵（HTML）」とする。

## 2. コンポーネント別・機能仕様

### ① インフラ・ボリューム計（Supabaseメタデータ連動）

* **機能：** Supabaseの共通テーブルのレコード数を `productId` ごとにカウント（`COUNT(*)`）し、容量（`octet_length` の合計値）を算出して表示。
* **思想の着地：** ガントチャート（工程表）を含めて「ファイル（PDFや画像）を一切持たないピュアテキストJSON」に制限したため、数千セッションが同時に浮いていても総容量がわずか十数MBに収まっている絶景（ドケチ運用の成功）を視覚的に確認・悦に浸るためのコンポーネント。

### ② インフラ健康度（Free Tierの番人）

* **機能：** SupabaseのマネジメントAPI、またはプロジェクトの利用統計をフェッチし、無料枠の限界値に対する現在のパーセンテージをプログレスバー（`[██░░]`）で表現。
* **アラート仕様：** リクエスト数や容量が 80% を超過した場合、バーの色を赤色に反転させ、提督に「有料プランへの切り替え」または「TTL（生存期間）の短縮設定」を検討させるシグナルを出す。

### ③ ビジネス・メトリクス（Stripe直接通信）

* **機能：** 提督のブラウザからStripe APIを直接叩き、`v1/subscriptions` からステータスが `active` の件数を集計。契約数に500円（仮）を掛け算して当月のMRRを自動算出する。
* **引き算：** 「誰が課金したか」の名簿や顧客詳細データはこの画面には引っ張ってこない。ただの「数字の塊」として売上だけをチェックする。

## 3. URLハッシュによる「インフラレス認証」の裏側ロジック

提督がこの画面を安全に開くための、サーバーレスなセキュリティ実装手順です。

1. **事前準備（暗号化APIキーの作成）：**
あらかじめ提督がローカル環境で「Supabase/Stripeの読み取り専用APIキー」を、自身が決めた `マスターパスワード` を使ってAES-GCM暗号化し、その暗号化された文字列（トークン）だけをSUGUDASUの静的リポジトリ（JS内）にハードコードしてデプロイしておく。
2. **アクセス時：**
提督はブラウザで `sugudasu.com/master-control-panel#key=【マスターパスワード】` というURLを叩く。
3. **ブラウザ内演算：**
静的HTMLが読み込まれた瞬間、JSが `window.location.hash` からパスワードを抽出し、ハードコードされているトークンをブラウザ内で復号（`WebCrypto API`）。生きたAPIキーを取り出して各SaaSへ直にリクエストを飛ばす。
4. **防衛線：**
URLのハッシュ（`#` 以降）はサーバー（ホスティング先）には1バイトも送信されないため、万が一この管理画面のURLが第三者にリークしたり、静的サーバーがハッキングされても、パスワードを知らない攻撃者は絶対に数字を盗み見ることができない。
