# SUGUDASU Sync マーケティング改善指示（レビュー反映版）

以下の改善内容を既存のSUGUDASU Syncマーケティング案へ反映してください。

---

# 前提

SUGUDASUのブランドは

- 登録不要
- ブラウザ完結
- すぐ使える
- 個人開発
- シンプル

である。

Syncはこの思想を崩さず、

**「便利だから使う。その結果として安心。」**

という順番で伝えること。

優先順位は

> 利便性
> ↓
> 安心
> ↓
> 技術

を維持する。

---

# ① セキュリティ表現を見直す

以下のような断定表現は避ける。

NG例

- 運営でも読めません
- あなたしか開けません
- 絶対安全
- 完全に秘密

代わりに、

**仕組みを説明する表現**

へ変更する。

例

- データはブラウザで暗号化されてから同期されます。
- サーバーには暗号化されたデータだけが保存されます。
- Syncは、サーバー側で暗号化前のデータを扱わない設計です。
- 復号に必要な情報は、あなた自身が管理します。

技術的事実を誇張せず、
説明責任を果たせる表現を優先する。

# ④ 「なぜ有料なのか」を説明する

FAQまたは比較表を追加する。

想定質問

「Google Driveで十分では？」

「iCloudで同期できますよね？」

「無料で同期できませんか？」

これに対して

比較ではなく

設計思想の違いを説明する。

例

- Syncはブラウザ側で暗号化してから同期します。
- サーバーは暗号化されたデータを保存する役割に徹します。
- シンプルな同期体験とプライバシーを維持するためのサービスです。

競合批判はしない。

「設計思想の違い」

として説明する。

# ⑧ コピー評価基準

コピーは以下の条件で評価する。

- 一読で意味が分かるか
- 専門知識不要で理解できるか
- SUGUDASUらしいシンプルさがあるか
- 誇張表現になっていないか
- 技術的事実と一致しているか
- 長期運用しても説明責任を果たせるか
