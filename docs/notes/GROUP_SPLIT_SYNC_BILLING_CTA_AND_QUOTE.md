**ステータス:** ブレストのみ · 実装・Go/No-Go 未確定

#### 「国内ではこういう製品が少ない」ニュアンス — 文言ライブラリ（ブレスト 2026-06-26）

**意図:** 決裁者に「なぜ新規ツール費用か」を伝える。**日本に同類がゼロ**とは言わない（誇大・リスク）。**現場のデファクト（Excel+連絡アプリ）との差** と **国内で探しにくいニッチ** を述べる。

**事実の置き方（概要 PDF 用 · 1段落以内）**

| いま国内で広いやり方 | 本製品が足すもの |
|----------------------|------------------|
| Excel / スプレッドシートで進行表 | 1箇所の時刻変更で **後続コマが連動再計算** |
| LINE・メールで PDF/スクショ再送 | **共有 URL** で版の提示（S2 で多端末同期を設計中） |
| Google / 組織カレンダー（rakumo 等） | **当日のランニングオーダー**（分刻みの進行表）に特化 |

海外の放送・イベント向けプロダクション用ツールはあるが、**社内研修・勉強会の幹事が稟議で少額導入する** 文脈とは要件・価格帯が異なる — という書き方は可（**競合名の名指しはしない** · Backlog 方針）。

**採用候補（トーン別 · そのまま貼れる案）**

| トーン | 文案 |
|--------|------|
| **A · 稟議向け（推奨）** | 社内研修・イベント運営において、進行表の作成・当日の時刻修正・スタッフへの共有は、多くの場合 **表計算ソフトとメッセージアプリの併用** で行われています。時刻の連動再計算と共有 URL による版の提示に特化した **小規模向けの国内クラウドサービスは、一般的な業務ツールの中では見つけにくい** 領域です。 |
| **B · やや短い** | 国内の現場では、進行表の「当日±数分のずれ」と「全員への周知」が分断されがちです。**進行表の連動再計算とチーム共有に絞った** ツールは、汎用カレンダーや表計算の代替として位置づけられます。 |
| **C · 幹事の体感** | 「Excel で組んだ進行表を、当日5分押したら後ろを全部手で直す」「PDF を何度も再送する」——この作業に特化した国内向けサービスは、幹事が自力で探しても **すぐには見つからない** ことが多いです。 |
| **D · 1行（表の脚注用）** | ※汎用グループウェアや表計算では代替しにくい、**イベント進行表の当日運用** に特化した用途です。 |

**避ける表現（稟議・法務リスク）**

| NG | 理由 |
|----|------|
| 日本初 · 国内唯一 · 類を見ない | 検証負荷 · 誇大広告 |
| 他社製品名での劣化比較 | Backlog「競合名指し禁止」 |
| 完全同期 · 100%安全 | S2 設計中 · 誠実開示と矛盾 |
| 大手と同等の機能 | 過大表示 |

**配置案:** 概要 PDF の **§2 解く課題** の直後に **「国内における一般的な運用」** 小見出しで A または B を **5〜7行** 挿入。LP 本文には短く D のみでも可。

## 3. 動線早見（個人 vs 法人）

```text
【個人・コミュニティ幹事】
  Freeで班分け → 前日までにチケット購入（500円）
  → 当日「チケット消費」1タップ → 全端末同期
  →（チケット切れ・初回のみ）救済 → 事後領収書

【法人・研修担当】
  Freeで下書き・制約テスト → 3日前に見積PDF（2,000円コミコミ）
  → 稟議・法人カードで前払い → 当日は追加課金なし
  → 終了後：領収書DL（経費精算）
```
