## 記事の芯（1段落）

「オフライン」と言っても **HTML の初回取得** · **JS の実行** · **ユーザー入力の処理** · **ファイルの保存** は別問題。  
SUGUDASU が言う「非送信」は **あなたの入力データ（請求書・名簿・画像）を運営サーバーへ POST しない** こと。  
初回アクセスにネットが要ること · 広告タグ · Sync の同期 — とは切り分ける。

---

## H2 案

1. **3つの「オフライン」** — 飛行機モード / タブを閉じたあと / サーバーに載せない
2. ブラウザが **オンラインでしかできないこと** — 初回ロード · CDN · 外部 API · ログイン同期
3. ブラウザが **オフラインでも（またはローカルだけで）できること** — Canvas · FileReader · localStorage · 印刷 · クリップボード
4. **「ブラウザ内処理」と「非送信」は同義ではない** — アップロード型も UI はブラウザ
5. **DevTools で自分で確かめる** — 画像 POST の有無 · Application タブの限界
6. **SUGUDASU ツール当てはめ表**（例）

| ツール | 主なローカル処理 | ネットが要る典型 |
|--------|------------------|------------------|
| 請求書 / 見積 | 入力 · 印刷 · localStorage 下書き | 初回 HTML/JS · 広告 |
| 全角半角整え | テキスト変換 · コピー | 同上 |
| WebP変換 | デコード · エンコード | 同上 |
| **マスク** | Canvas 黒塗り · PNG 出力 | 同上 |
| Sync 進行表 | 編集バッファ · CRDT | **同期 API**（別ライン） |

7. **選び方のチェックリスト** — 社内規程 · 何を隠すか · 何枚か · 説明責任
8. CTA → `statements` · ツール一覧
