## リード（案 · 本質版）

同時編集で本当に困るのは「保存できない」ことではない。**読めなくなる** ことだ。イベント本番では、受付が足した3行と会場が足した3行が交互に混ざるだけで進行が止まる。この記事は `CRDT` を流行語として売る話ではなく、**混ざっても読めるようにする設計**と、**現場で止められる運用**をどう組み合わせるかの実務メモ。

## H2 アウトライン（本質版）

### 1. まず痛みから — 「速く編集できる」のに現場が止まる理由

- **具体例:** 受付担当「誘導A/B/C」を追記、会場担当「音響A/B/C」を同じ位置に追記。保存は成功するが、行が交互に並ぶと役割の塊が崩れて読む人が迷う。
- **オマージュ:** `"HI mdoamd!"`（mom/dad の interleaving 例）を 1枚図で紹介し、**文字の混在問題を行の混在問題に翻訳**する。
- **結論:** 現場で必要なのは「同時編集そのもの」ではなく、**意味の塊を守ること**。

### 2. CRDT は名前ではなく手段 — 1段落で十分

- **定義（最小）:** Conflict-free Replicated Data Type。離れた編集操作を、あとで同じ結果に収束させるための型。
- **言い換え:** 読者に伝える主語は CRDT ではなく、**混ざらない同期**・**壊れない合流**。
- **立ち位置:** 「最先端を追う」ではなく、**幹事の失敗コストが高い場面を先に救う**。

### 3. なぜ「全員フル編集」より「現場向けマージ」か

| | Google 表 / フル共同編集 | SUGUDASU Sync（方針） |
|--|--------------------------|------------------------|
| 編集者 | 全員フル編集 | **編集者少数 · 閲覧者多** |
| 反映 | 自動上書き | **新版バナー + 手動反映**（本番事故防止 · §3-3） |
| データ | 永久クラウド | **短期バッファ** · export 前提 |
| 衝突 | セル単位 | **行序列** — NonInterleaving LSeq 系を検討 |

- **本質:** 自動で全部解決するより、**人が判断すべき境界を残す**ほうが現場事故が減る。
- **F1 整合:** コア `sugudasu.com` は **非送信のまま**。この設計思想は export 合流にも転用できる。

### 4. 技術核はここだけ — NonInterleaving LSeq の意味

- **要点:** 同じ編集者の連続挿入を、後から見ても **塊として保つ**。
- **見せ方:** 数式は最小限。`peer + seq_no` の直感図を先に出し、コードは補助に回す。
- **境界:** 文字単位 CRDT の全面導入はしない。今回は **行並び**に集中する。

### 5. 今どこまでできて、どこから先が S4 か

| フェーズ | 同期モデル |
|----------|------------|
| **S1–S2（今）** | `payload` + `revision` 楽観ロック · 閲覧者手動反映 |
| **S4** | 編集者2人 · **行 insert/reorder の CRDT マージ** |
| **公開後 LP** | 脚注「衝突フリー設計」— **S4 ゲート後** |

- **誠実さ:** 実装前は「予定」と書く。できていないことを盛らない。
- **製品クラスター:** timeline+group（当日）と schedule（工程）は横断シナジー薄。CRDT もクラスター内で語る。

### 6. なぜ刺さる人には刺さるのか

- **刺さる対象:** 本番で「誰の更新が消えた？」を説明する責任がある人。
- **刺さる理由:** 流行語ではなく、**事故を1件減らす設計**として読めるから。
- **必要条件:** Before/After の具体例、守る単位の明示（文字ではなく行）、運用ガード（手動反映）をセットで示す。

### 6.5 昔を知る読者向けの比喩（短く）

- **Git vs SVN:** どちらも正しい。違うのは「各自で進めて後で合流」か「中央で順序を握る」か。
- **Napster/Gnutella vs iTunes:** どちらも音楽を届ける。違うのは分散で広がる設計か、中央管理で品質を担保する設計か。
- **本文の締め:** **違うのは優劣ではなく、失敗の仕方と運用コストの置き場所。**
- **注意:** 比喩は2つまで。懐古で脱線せず、必ず「進行表の行が混ざる痛み」に戻す。

### 7. やらないこと・限界（ここを先に明言）

- 本番中 Checkout · 永久保存 · Zoom 連携
- 「CRDT だから競合ゼロ」とは言わない — **意図的に手動反映**する閲覧者 UX を残す
- Schedule / 請求書 / 抽選への CRDT 拡張（ **リスト型ツールのみ** ）
