# SUGUDASU JSON構造（json-view）— V1 仕様

**更新:** 2026-07-21  
**id:** `json-view`  
**状態:** V1（UX 1.1 微調整込み · 実機3ケースのユーザーテスト待ち）  
**思想:** 高機能 JSON IDE ではなく、貼った JSON から欲しい1値を30秒で持ち帰る工具（JSON構造ナビゲーター）

---

## 1. JTBD

> AI / API が吐いた JSON をブラウザだけで覗き、必要な値または場所を1個だけ最速でコピーする。

優先順位: **貼る → 見る → 探す → コピーする**

---

## 2. 憲法（絶対）

| 禁止 | 理由 |
|------|------|
| サーバー通信 / 外部 API / AI | 非送信 |
| LocalStorage / IndexedDB / Cookie / 履歴保存 | タブ寿命のみ |
| ログイン / アカウント | 不要 |
| 自動修復・推測補完 | 「見る」と「直す」の境界を守る |
| JSONPath クエリエディタ | 開発者ツール化を避ける |
| Schema / Fetch / JWT / 変換 / Pretty大量ボタン | 探索以外はスコープ外 |
| React / 仮想スクロールライブラリ / 状態管理ライブラリ | 過剰。V1は遅延展開のみ |

---

## 3. V1 機能（Must）

### 3.1 入力

- 大きな textarea。**貼り付けで即 `JSON.parse`**
- 「構造を見る」ボタン・Ctrl+Enter も可
- パース失敗: **自動修復しない**。行番号（取れる場合）＋「この付近を確認してください」のみ
- Hero は短く（安心バッジ + 1行リード）。PathBar を本体直上の主役に置く

### 3.2 ツリー

- 型: object / array / string / number / boolean / null
- **初期展開はルートのみ**
- すべて閉じる（必須）・すべて開く（任意）
- 再帰 DOM 禁止 → `flattenVisible` で可視行だけ通常 DOM 生成
- **仮想スクロールは V2**（入れない）

### 3.3 巨大配列ガード

- 展開時は先頭 **`ARRAY_CHILD_CAP`（200）件**まで
- 注意文: `大きな配列です（N件）。先頭200件を表示しています。`
- 検索ジャンプで 200 件目以降を開いた場合は **例外表示**し、文末に  
  **`検索対象のみ一時表示しています。`** を付ける
- 「すべて開く」も配列は先頭 200 件まで（固まり防止）

### 3.4 PathBar（場所）

| | |
|--|--|
| **表示（人間向け）** | `data > departments[0] > email` |
| **コピー（機械向け）** | `$.data.departments[0].email` |
| **更新** | 行ホバー / キークリック / 検索ジャンプ |
| **操作** | PathBar クリックで JSONPath をコピー |
| **見た目** | 通常 input ではなく、枠・背景を強めたコピー用ボタン |

表記切替（JS / Python 等）は **しない**。

### 3.5 検索（ターゲット・エクスパンション）

- **1欄**でキー・値の部分一致（大小無視）
- placeholder 例: `email / user_id / city`
- 入力中: 件数表示（例: `0 / 15`）。**この時点では自動展開しない**
- Enter / 「次へ」: 次ヒットへジャンプ
  - ヒットの**親ノードだけ**展開
  - 無関係な枝は閉じたまま
  - 黄色フラッシュ + `scrollIntoView`
- 「すべて閉じる / 開く」は **Tree 直上**（検索の下）
- JSONPath クエリ入力は **しない**

### 3.6 コピー（Copy-first）

| 対象 | 操作 | 内容 |
|------|------|------|
| 値 | **値テキストをダブルクリック** | string は生値（`"` なし）。number/boolean はそのまま。object/array は整形 JSON |
| 場所 | PathBar クリック | `$.…` |
| キー単体 | **しない**（V1削除） | 発見性が低く利用頻度も低い |

- 行全体・キーへのダブルクリックでは値コピーしない（誤爆防止）
- 成功時: トースト約 0.8 秒（`✓ 値をコピーしました` / `✓ 場所をコピーしました`）。場所コピー時は PathBar を短く緑フラッシュ

### 3.7 レイアウト · FAQ

- デスクトップ列比おおよそ **40% / 60%**（入力 : Tree）
- Key / Value コントラストを明確に（Key 濃紫 · Value ダークグレー）
- FAQ 先頭: **JSONPath（場所の表示）とは？**（`[0]` = 配列先頭も一言）

## 4. Non-Goal（V1 禁止・誘惑）

- JSONPath 入力欄 / フィルタクエリ
- Path 表記切替 UI
- Pretty / Minify / Sort / Convert / Download の羅列
- Schema 検証、AI 要約、編集モード
- React 化、仮想リスト、高度な JSON ライブラリ

迷ったら **追加せず削る**。性能問題が出てからだけ拡張する。

## 6. 実機テスト（V1 合格条件）

各ケースで **30秒以内に欲しい値をコピーできること**。

1. **ChatGPT API 系** — 深い nested から `content` 等を抜く  
2. **巨大配列** — 1万件級で検索ジャンプし、注意文と一時表示が分かること  
3. **設定 JSON** — PathBar で場所を理解し、クリックでパスを持ち帰れること
