## 1. 結論（3行）

| 問い | 答え |
|------|------|
| **土木建築特化は他業界に転用できるか？** | **エンジンと UI パターンは転用できる。** ただし「転用」は **別ガント SaaS** ではなく **同じコア + 垂直パック（preset・Export・文言）** が現実的。 |
| **リサーチは必要か？** | **境界の切り分けは今の調査で足りる。** 第二垂直を **製品として出す** 判断だけ、軽い検証（1業種 dogfood or Gemini 1ラウンド）が任意。 |
| **土木サブ垂直（堤体・ビル・工場）は別製品か？** | **いいえ。** 同一 Schedule · **Export テンプレ / 既定プロパティ / SEO LP 段落** の差。製品を裂かない。 |

## 2. 四層モデル（作り込みの境界）

**下ほど汎用 · 変更コスト低。上ほど特化 · 市場訴求・転記削減の芯。**

```
L3 サブ垂直パック     堤体 · ビル · ほ場 · 工場建築 …（様式・用語・初期列）
L2 建設ドメイン       監理提出 · 週間下請 · 重機 · 打設届 · fleet/site 連携
L1 現場運用パターン   提出/現場 preset · visibility · 週間 Export · ゲスト
L0 工程表コア         左表+右棒 · 日次 · 依存(opt-in) · sub-items · xlsx/PDF Export
```

### 2-1. 各層の定義と転用

| 層 | 中身（例） | 作り込み | 他業界への転用 |
|----|------------|----------|----------------|
| **L0 コア** | `schedule-engine` · 分割ペイン · payload schema · 組織/現場 | **1回作って凍結** | ◎ WEB制作 · 映像 · 印刷 · 展示会 — Gemini 15垂直で **同型ペイン** |
| **L1 運用パターン** | `submit`/`site` preset · `visibility` · 週間リスト · 版付き PDF | **パラメータ化**（名前だけ差し替え） | ◎ 「クライアント提出 vs 社内運用」の **二重管理** は建設以外にも共通 |
| **L2 建設** | `prop_machine` · 工区 · 監理 A3 · 下請フィルタ · 打設届 v2 | **建設パック**として実装 | △ 製造試作・設備は **一部プロパティ** が似るが様式は別 |
| **L3 サブ垂直** | 土木共通仕様書トーン · 建築検査日 · 堤体セル語彙 | **テンプレ・seed データ** | × 他業界には載せない · **SEO 用 LP 差分** 程度 |

### 2-2. 判断ルール（Agent · 提督共通）

新機能の提案時:

1. **L0 に入るか？** — 全垂直で「工程表の表+棒」に必要か。No なら入れない。
2. **L1 に昇格できるか？** — 「提出/現場」以外の **汎用ペア名** に言い換えられるか（例: `client` / `internal`）。
3. **L2 か？** — 法令・現場用語・建設様式に依存するか。Yes なら **建設パック** フォルダ・preset 名に `construction_` 等。
4. **L3 か？** — 堤体だけの列か。Yes なら **新コードを書かず** Export テンプレ or 初期 `properties[]` seed。

**禁止:** L3 の要件を L0 に入れる（全ユーザーに堤体用語を見せる）· L0 を L2 の名前でマーケする（ガント SaaS 比較に乗る）。

## 3. 他業界転用マップ（既存リサーチの読み替え）

[`excel-gantt-verticals-gemini-RESULT.md`](excel-gantt-verticals-gemini-RESULT.md) §1–2 より。**勝ち筋は同じ:** 連動再計算 · 版混乱 · 低 IT リテラシー · Excel 正本のコピペ。

| 他業界 | L0 | L1 の言い換え | L2 相当（別パック） |
|--------|-----|---------------|-------------------|
| **広告・WEB制作** | ◎ | クライアント提出 / 制作内部 | 納品物 · 外注 · 素材納入 |
| **映像制作** | ◎ | クライアント版 / 現場進行 | 撮影日 · 機材 · 外注 |
| **印刷・DTP** | ◎ | 顧客確認 / 工場内部 | 入稿 · 色校正 · 下版 |
| **展示会設営** | ◎ | 主催者提示 / バックヤード | 搬入 · 電源 · 撤去 |
| **建設（土木建築）** | ◎ | 監理提出 / 現場 site | 重機 · 検査 · 占用 · 週間下請 |
| **受託IT小規模** | ◎ | 顧客マイルストーン / スプリント内部 | チケット連携は **別製品** |

**転用の形:** `sync.sugudasu.com/schedule` をそのまま売るのではなく、将来 **同一 L0+L1 · 別 L2 パック名**（例: `schedule-production` ブランド or 同一アプリ内 `industryPack: construction`）— **v1 では建設のみ出荷**。

## 4. 建設サブ垂直 — 頭打ちへの対処

**問題:** ほ場整備 · ビル · 堤体 · 工場建築に **製品を割る** と、それぞれ市場が小さく **開発が先に頭打ち**。

**方針:**

| やる | やらない |
|------|----------|
| **1 アプリ「現場工程表」** — SEO は「工程表」「土木」「建築」で広く取る | サブ垂直ごとに別 repo / 別 Supabase |
| **L3 = テンプレ市場** — 初期行サンプル · A3 枠 · 用語プリセット | 堤体専用エンジン · ビル専用依存ルール |
| **dogfood で L3 優先順** — 提督の現場（土木）→ 需用の多い Export から | 4サブ垂直を並行で深掘り |
| **fleet/site は建設 L2 エコシステム** — 他業界にはパックごと外す | 製造 ERP まで一体化 |

**サブ垂直の差が出るのは主に:**

- Export 枠（監理者・発注者の好み）
- 既定プロパティ（工区 vs 階 vs 桩号）
- LP・SEO 段落（検索意図の違い）

**エンジン・Sync · preset 構造は共通。**

## 5. マーケット規模の見方（メタ）

| 層 | TAM の考え方 |
|----|--------------|
| **L0 のみ** | 汎用ガント红海 — **勝てない**（§0-4）。名乗りは「工程表」でも比較表には乗らない。 |
| **L0+L1** | 「Excel 二重管理の転記削減」— **横断ニッチ** · Gemini 15垂直。¥200/月 × 小規模組織。 |
| **L0+L1+L2 建設** | 日本の小規模ゼネコン・サブコン監督 — **提督第一ターゲット** · fleet/site で **ARPU 上乗せ**（別課金）。 |
| **L3 サブ垂直** | **獲得効率**（SEO・口コミ）の話。TAM を増やすというより **CV を上げる**。 |

**頭打ち感は L3 で製品を割ったときに起きる。L2 まで一本化 + エコシステムで伸ばす。**

第二垂直（WEB制作等）は **L0+L1 が検証済み後** — 新 L2 パックとして **边际コストが低い** ときだけ。

## 6. リサーチ要否

| 論点 | 状態 | 次アクション |
|------|------|--------------|
| **汎用 vs 特化の境界** | **本ファイルで第一回答** | 機能追加時に §2-2 を適用 |
| **建設以外への転用可能性** | Gemini 15垂直 **済** | 第二垂直 GO 時のみ 1 業種に絞って再調査 |
| **L1 パターンの他業界名称** | 未検証 | 任意: `client`/`internal` で WEB 制作に刺さるか **1 問アンケ** |
| **土木サブ垂直の Export 優先** | 未 | dogfood · 提督の監理様式 1 種を **テンプレ第1号** |
| **堤体 vs ビル vs 工場の TAM 比較** | **不要（初期）** | SEO キーワードボリュームだけ見れば足りる · 製品分岐には使わない |

**結論:** **境界探りのリサーチは不要。** 既存 Gemini + 提督動機で四層は引ける。**実装優先度** と **第二垂直 GO/NO** だけ、出荷後に判断。

## 7. v1〜v2 への落とし込み（固定）

| フェーズ | 層 | 出荷 |
|----------|-----|------|
| **v1** | L0 + L1 + L2（最小） | 分割ペイン · submit/site · 月間提出 Export · ゲスト |
| **v1.1** | L1 + L2 | 週間下請 · `weekly_sub` |
| **v2** | L2 | 打設届 · 占用 · fleet `vehicle_key` 参照 |
| **v2+** | L3 | 土木/建築 **2 テンプレ** まで。堤体専用は **文言 seed のみ** |
| **将来** | 別 L2 パック | WEB制作等 — **新規リサーチ 1 ラウンド後** |

**コード上の示唆（将来）:**

```text
payload.industryPack: "construction"   // 既定
viewState.presets.*                   // L1 — パック共通
properties[]                          // L2 — パック seed
exportTemplates[]                     // L2–L3
```

v1 では `industryPack` 省略可（暗黙 construction）。

## 10. 付録 — 備考の Path・リンク（提督案 · 2026-06-29）

**問い:** 備考に `C:\` · `\\NAS\` · 外付け · Cloud URL を書いたらリンクにしたい — **ニーズはあるか · 共通か？**

### 10-1. 結論

| 問い | 答え |
|------|------|
| **ニーズ** | **ある。** Excel 工程表のセルに図面フォルダ・PDF・写真パスを書くのは日常。転記削減の延長で「工程行から資料へ飛びたい」。 |
| **層** | **L0（共通）** — 備考・テキストプロパティの **URL 自動リンク** は建設以外（制作素材フォルダ · 入稿 PDF）にもそのまま効く。 |
| **リサーチ** | **不要。** fleet が既に **Drive https リンク** を正としている（下記）。 |

### 10-2. ストレージ種別と Web の現実

| 種別 | 例 | ブラウザからクリックで開く | 製品の扱い（案） |
|------|-----|---------------------------|------------------|
| **Cloud https** | Google Drive · SharePoint · Dropbox 共有 URL | ◎ 新タブで開ける | **v1.1** — 備考内を自動 `<a>` · アイコンで種別表示 |
| **UNC / NAS** | `\\fileserver\現場\図面` | **× 基本不可**（セキュリティ） | **コピーボタン** · クリックでクリップボード · エクスプローラはユーザー操作 |
| **ローカル / 外付け** | `D:\現場\` · `Z:\`（NASマップ） | **× 同上** `file://` は Web からブロック | 同上 · 印刷 PDF には **パス文字列** として残す |
| **相対・社内略称** | `現場フォルダ/配筋/` | × | テキストのまま · v2 で現場マスタに **既定ベース Path** を足せば展開候補 |

**提督向け一言:** 「リンクできるようにしたい」は正しいが、**https だけが本当のリンク**。NAS/PC Path は **コピー支援** が現実解 — Excel でもセルは文字列で、ダブルクリックはエクスプローラ任せ。

### 10-3. 既存の前例（エコシステム）

| 製品 | パターン |
|------|----------|
| **fleet**（machine-dashboard） | 領収書・特自検 PDF = **Google Drive 共有 URL**（`fleet-cost-requirements.md`） |
| **Schedule** | `prop_note` 備考 = text · 未実装のリンク化 |

**方針の整合:** 新規ファイル保存は **Cloud URL を推奨**（STATEMENTS / オンボーディング1行）。NAS Path は **既存資産の参照** としてコピー可能に留める。

### 10-4. 実装スコープ案

| 段階 | 内容 | 層 |
|------|------|-----|
| **v1** | 備考はプレーンテキスト（現状） | — |
| **v1.1** | `https?://` 自動リンク · 行 Edit プレビュー | **L0** |
| **v1.1** | Path っぽい文字列（`\\` · `X:\`）に **コピー** アイコン | **L0** |
| **v2** | 任意列 `prop_link`（単一 URL）· 週間 Export には出さない可 | L0 |
| **やらない** | ブラウザから NAS を直接マウント · ファイル同期 · 添付ストレージ SSOT 化（Sync Storage 方針と別論点） |

**下請週間リスト:** Cloud URL を載せるかは **visibility** — 社外秘フォルダは `internal` 行のみに Path を書く運用。
