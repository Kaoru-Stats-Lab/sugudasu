## 1. Pain · ポジション

外部CSVダウンロード禁止 · Excel手打ち · 本番コピー禁止 — の現場で、**テスト・マニュアル用の整合済みダミー**をブラウザ内だけで作りたい。

**NOT:** 本番DBマスキング · 大規模TDM · 実在口座/マイナンバー  
**IS:** 顧客マスタ/取引明細風CSV · シード再現 · 社内ルール（ID/ドメイン）· 地雷混入

## 6. データセット依存あるある — 取り込み判断

人事・給与系開発で「**テストの成否がデータの作り方に依存する**」パターン。  
SUGUDASU は **50型ビルダー全面競合はしない** — 楔（労務SMB）で **よく事故る・説明しやすい・単一HTMLで届く** ものだけ入れる。

**判断軸（4つ）**

| 軸 | 入れる |
|----|--------|
| **頻度** | 受託SI・社労士・情シス兼務が月1回以上踏む |
| **事故率** | 本番コピー禁止・Excel手打ち・インポート失敗で定時が溶ける |
| **再現** | シード＋地雷で「同じバグを説明できる」（C5） |
| **実装** | 単一HTML · コンプラ安全 · 過度なドメイン専門知識不要 |

**凡例:** ✅ 済 · 🔜 次 · 📋 ドキュメントのみ · ❌ 意図的に入れない

### 6.1 入れる（済 or 次）— これが肝

| あるある | 典型事故 | 状態 | 実装 |
|----------|----------|------|------|
| **フリガナ全角/半角・ソート** | インポートエラー · あいうえお順崩れ | ✅＋🔜 | 全角カタカナ生成 · 地雷で半角カナ · `normalize` 導線 |
| **日付形式バラバラ** | `YYYY/MM/DD` vs `YYYY-MM-DD` vs `YYYYMMDD` · Excel化け | ✅地雷＋🔜 | 生年月日`/` · 入社`-` · 地雷でゆらぎ · **形式選択はv0.3** |
| **郵便番号↔住所不整合** | 住所自動入力・配送マスタが壊れる | ✅ | 57件整合マスタ · 地雷でハイフンなし7桁 |
| **基準日依存（年齢・勤続・入社上限）** | ハードコード年で翌年テストが嘘になる | ✅ | 基準年＝今年 · 入社開始年指定 |
| **雇用形態×給与レンジ** | パートに正社員給与 · 社保分岐未テスト | ✅ | 形態別レンジ · **再雇用**約7% |
| **高齢・定年後（再雇用）** | 年齢計算・社保・表示崩れ | ✅ | 生年1948–64 · 入社60歳以降 |
| **シード再現** | 「同じ手順で再現せよ」監査 | ✅ | Mulberry32 · 同一設定＝同一CSV |
| **地雷（意図的異常値）** | 本番では滅多に出ない境界が未テスト | ✅ | 約5% · 外字 · 超長住所 · 不正メール |
| **ヘッダー1文字差** | SaaSインポートで全列スキップ | ✅ | 列名カスタム · テンプレ3種 |
| **件数不足（ページング）** | 50件ではソート/ページバグが拾えない | ✅ | 100/500/5000 |
| **社員↔給与明細リレーション** | 結合・集計・3ヶ月分の通算 | 🔜 Should | 同一社員IDで明細N行（v0.3） |

### 6.2 入れない — 別ツール・別プロセスの領域

| あるある | なぜ入れないか | 代替 |
|----------|----------------|------|
| **本番DBマスキング・匿名化** | C3違反リスク · 復元可能指摘 · 製品ポリシー | 架空生成のみ · `statements` で明記 |
| **マイナンバー・実口座** | コンプラ · 誤用・監査で一発アウト | 生成しない · 固定文字列もv0.3以降検討のみ |
| **社保等級・源泉の厳密境界値セット** | 法改正追随コスト · 専門TDM領域 | 基本給はレンジのみ · 境界は手動1行足す |
| **100万件・負荷試験** | ブラウザ・単一HTMLのスコープ外 | k6 · SnowFakery · 本番相当TDM |
| **SQL INSERT / DB直投入** | 開発者向け · Handy/SnowFakeryが強い | CSVコピー · 先方スクリプト |
| **勤怠×給与×賞与の完全整合** | 3表以上 · 月度締め · 個社ロジック | v0.3は社員+明細の**最小リレーション**のみ |
| **部門・事業所・社保事業所マスタ** | 顧客ごとに列が違いすぎる | ヘッダーカスタム後に様子見 |
| **50型ビルダー** | 差別化が薄れる · 容量 | 業種プリセット垂直 |

### 6.3 将来検討 — インタビューでGO/NO

| あるある | 保留理由 | GO条件 |
|----------|----------|--------|
| **xlsx DL（先頭0落ち）** | JSライブラリ容量 | 「郵便番号が60…になる」がFAQ上位 |
| **控除・手当の独自列** | 社ごとに無限 | ヘッダーカスタム＋固定列テンプレ1種 |
| **Unicode正規化（NFC/NFD）** | ニッチ | インポート失敗の再現報告 |
| **性別「その他」・未入力** | 法・SaaS差 | ペルソナから要望 |
| **重複社員番号（地雷）** | レア | DB一意制約テストの声 |
| **最低賃金・通勤15万境界の1行** | 法改正追随 | 「境界値1行欲しい」が複数ヒアリング |

### 6.4 ドキュメントだけ足す（製品外）

現場・情シス向け FAQ / `statements` に書き、**ツールは生成しない**。

- ダウンロード後のCSV保管・廃棄（利用者責任）
- Excelで開くときの「データからテキスト」取り込み手順
- 本番データを「仮名化」してもC3に触れうる旨
- プロキシで `sugudasu.com` 自体がブロックされる場合がある旨
