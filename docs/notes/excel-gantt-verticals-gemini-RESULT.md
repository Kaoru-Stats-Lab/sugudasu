### §6 勝ち筋の結論（Gemini）

**第一 PoC 垂直：広告・WEB制作ディレクター向け「スケジュール逆算・共有バッファ」**

- 二極化（Asana/Monday vs スプレッドシート）の **中間**。
- 最大ペイン：**クライアント確認1日遅れ → 後続全日程の手動再計算 + 最新版 PDF 再送**。
- SUGUDASU 適合：行変更の後続連鎖 · URL1本 · 案件単位少額課金。

**第二・第三（様子見）：建設（内装・リノベ）· 映像・動画制作**

- 建設：SEO 魅力大だが図面・日報等の機能要望で ANDPAD 比較されやすい。
- 映像：香盤表（分刻み）との境界が近い → WEB/広告で検証後に拡張。

**やらない垂直：エンタープライズ IT 開発 · 大規模製造の生産管理**

- Jira/MS Project/ERP 必須領域。API 連携・法規と思想が衝突。

### §8 リスク・誠実開示（Gemini）

1. **S2 同期の実装済み誤認** → LP で設計中ラベル厳守
2. **ガント SaaS 過剰期待** → 依存矢印・要員管理は非対応と明示
3. **コア無料とのブランド混同** → ドメイン・規約分離
4. **セル結合・マクロのパース失敗** → コピペ/CSV 限定を UI 開示
5. **短期パージ誤認** → retain_until · エクスポート促し
6. **本番中 Checkout** → 事前チケット消費のみ
7. **社外秘・実名の送信不安** → E2EE 等は要確認と開示
