## 0. なぜこの文書があるか

シフト表は現場によって **データの形がまったく違う**。病棟の24時間体制と、8席の飲食店と、3人のサーバ当番では、同じ「シフトツール」に押し込むと、どこかで必ず嘘になる。

SUGUDASU は **向いている人に正直に使ってもらい、向いていない人には別の手段を勧める** 方が、請求書やテストデータと同じ誠実さだと判断した。  
あわせて **スコープ内の人からのフィードバックは、開発の燃料** として歓迎する（後述 §6）。

## 1. プロダクトの一文

**小さなチームの「来月、誰がいつ出るか」を、ログインなし・ブラウザだけで組み、A4で貼れる表にする。**

| IS | NOT |
|----|-----|
| 5〜30人程度のパート・アルバイト中心 | 数百人・多拠点の人員管理 |
| 月次カレンダー · 印刷 · 仮→FIX | 勤怠打刻 · 給与計算 · 承認ワークフロー |
| 「偏りすぎない」自動シャッフルのたたき台 | 労基法・36協定・資格要件の自動適合証明 |
| データは端末内（LocalStorage） | クラウド同期 · 監査ログの正本 |

**隣接ツールとの境界**

| ツール | 違い |
|--------|------|
| **イベント進行**（`timeline`） | **当日のコマ**（何時に何をするか）。人の勤務表ではない |
| **Schedule**（Sync） | **工事・制作の工程**（タスクの日付）。人員シフトではない |
| **班分け**（`group-split`） | **1回きりのグループ分け**。月次の繰り返し勤務ではない |

## 2. 向いている人（スコープ内 · ここから FB ほしい）

次のような現場・役割が **v2 の主ターゲット**。名前は仮称。実在の業態に合わせて FB で磨く。

### 2-1. 小規模飲食・カフェの店長（8〜20席 · スタッフ5〜12人）

- **状況:** 早番・遅番・キッチンとホールを同じ時間帯に複数人入れたい。Excel はコピペ地獄。
- **欲しいこと:** ランチ帯に「キッチン2 · ホール2」くらいを表にして、休憩室に貼る。公平に土日を回したい。
- **SUGUDASU が担う:** たたき台の自動配置 → 手直し → 印刷。**法令適合の証明はしない。**

### 2-2. 美容室 · ネイル · 小売のシフト担当（スタッフ3〜15人）

- **状況:** 曜日定休 · 扶養内の子がいるスタッフ · 連勤上限を頭で計算している。
- **欲しいこと:** 2ヶ月見通し · 偏りの可視化 · FIX して PDF。
- **現行でも近い:** 扶養内上限 · 連勤制限 · 土日平準化は既にある。v2 で **時間帯×ポジション** を足す。

### 2-3. イベント · 学習塾 · 小規模オフィスの「当月の当番」（5〜20人）

- **状況:** 受付当番 · 清掃当番 · 電話当番を月単位で回す。専用 SaaS は重い。
- **欲しいこと:** 今日サッと表を作って Slack にスクショ。データを外に出したくない。
- **用語:** 「枠」ではなく **当番 · 役割 · 時間帯** と呼ばれることが多い。

### 2-4. 社内情シス · 小規模インフラの「サーバ当番」（3〜10人）

- **状況:** 夜間・休日の **primary / backup** を月で回す。PagerDuty までは要らないチーム。
- **欲しいこと:** 誰が今週末か一目 · 印刷 · ローテの偏りを減らす。
- **スコープの線:** **当番表まで**。**アラート連携 · エスカレーション · SLA 管理は別製品**（§3）。

### 2-5. 副業フリーランスがバイト先を複数持つ「自分用メモ」（1人）

- **状況:** 本業とは別に、店のシフトを自分でメモしたい（店長ツールの利用者ではないが、閲覧・印刷用途）。
- **限界:** 複数人の公平性ロジックは薄い。**個人メモ＋印刷** くらいの期待に留める。

## 3. 向いていない人（Non-goal · 別ツールを勧める）

ここは **はっきり書く**。使えないと分かったうえで選んでもらうため。

| 誰 | なぜ SUGUDASU シフトではないか | 現実的な代替 |
|----|-------------------------------|--------------|
| **病棟 · 診療所の看護師シフト担当** | 24H×365 · 人員基準 · スキル · 法定休憩 · 夜勤手当 — **法的・運用上の要件に耐えない**。ブラウザ単体では責任の取り方が成立しない | 病院向けシフト管理 · 勤怠SaaS（専売品） |
| **中規模以上の飲食チェーン（多店舗）** | 店舗横断 · 人員不足の補充 · 本部承認が必要 | ジョブカン · KING OF TIME · シフトール等 |
| **製造 · 物流の大規模シフト** | ライン人数 · 資格 · 変形労働 — 最適化と監査が主戦場 | 勤怠・シフト専用 ERP / SaaS |
| **大規模 DC · 24/7 NOC** | チーム×リージョン×オンコール階層 · インシデント連携 | PagerDuty · Opsgenie · 社内運用ツール |
| **「労務が法的に問題ないか自動判定してほしい」担当** | 本ツールは **配置のたたき台** であり、労基適合の証明装置ではない | 社労士確認 + 勤怠SaaS |

**画面 FAQ（予定）でも同趣旨を書く。** ユーザーを遠ざけるためではなく、**ミスマッチによるトラブル**（「法令チェックしたつもりだった」等）を防ぐため。

## 4. 用語 — 「枠」をやめる

| 旧（実装寄り） | 新（ユーザー向け） | 意味 |
|----------------|-------------------|------|
| 枠1 / 枠2 / t1,t2 | **時間帯** | 早番 · 遅番 · ランチ帯 など、その日の勤務ブロック |
| （欠落） | **ポジション** | キッチン · ホール · primary · 受付 など、**同じ時間帯に並ぶ役割行** |
| セル1人 | **配置** | その時間帯×ポジションに入る人（v2: **複数可**） |

**データモデル（v2 目標）**

```
日付 (YYYY-MM-DD)
  └─ 時間帯[] （名称可変 · 1〜4程度）
       └─ ポジション[] （名称可変 · 0〜6程度 · 省略可＝ポジションなし1行）
            └─ 人[] （スタッフID または 自由入力）
```

現行の `t1/t2/t3` × `val: string` は **移行期間** として残し、v2 で `positions` + `assignments[]` に拡張（`BACKLOG` §12-2 · FB-20260617-002）。

## 5. v2 Must（優先順）

| # | 要件 | 備考 |
|---|------|------|
| V2-1 | **時間帯あたり複数名** | 飲食 · 当番の最低ライン |
| V2-2 | **ポジション行**（任意） | 時間帯の下に「キッチン」「ホール」等。名称はユーザー編集 |
| V2-3 | **未充足の可視化** | ポジションに最低1人欲しい等 — 赤表示。自動充足は後回し可 |
| V2-4 | **UI 用語置換** | 「枠」→ 時間帯 / ポジション |
| V2-5 | **印刷** | セル複数行 · A4横 · 月改ページ（現行維持） |
| V2-6 | **自動生成** | まず **平準化＋制約の範囲を狭く** 再定義。全ポジション自動は Phase 2 可 |
| V2-7 | **繁忙期 · 繁忙日の人数増** | §5-1 参照。V2-1（複数名）の上に載せる |

### 5-1. 繁忙期 · 繁忙日（あるある · v2 候補）

現場では **平日3名 · 土日5名 · クリスマス前は全帯+1** のように、日によって必要人数が変わる。v1 では **日別備考＋手動でセルを埋める** のみ。

**v2 で想定する形（要件ドラフト）**

| パターン | 例 | UIイメージ |
|----------|-----|------------|
| **曜日ルール** | 土日は各時間帯 +1名 | 「毎週土日 → ホール最低2名」 |
| **特定日** | 12/24 · 12/31 · バレンタイン | 臨時休業日UIと同型で「繁忙日」リスト |
| **期間** | GW · お盆 · 年末年始 | 開始〜終了日でルールを上書き |
| **時間帯別** | ランチ帯だけ +2 | ポジション行（V2-2）とセット |

**自動生成:** 繁忙日は「その日の必要人数」を先に満たしてから平準化、が自然。未充足は V2-3 の赤表示と連動。

**FB で知りたいこと:** 業態ごとの繁忙パターン（飲食の金土夜 · 小売の週末 · 塾の冬期講習 等）。

### v2 Won't（明示的にやらない）

- 労基法 · 36協定 · 夜勤間隔の **適合判定**
- 多店舗 · 本部承認 · 希望休の承認フロー
- 資格マトリクス（看護師免許 · 危険物取扱 等）
- クラウド同期 · 他端末引き継ぎ（現行どおり LocalStorage のみ）
- 給与 · 勤怠打刻との連携

## 7. 現行（v1.1）の限界 — 嘘をつかないための注記

| 項目 | 現状 |
|------|------|
| 1日の構造 | 時間帯 **最大3** · 各 **1人のみ** |
| 職種タグ | スタッフの `tag` は **集計表示** と **同日の重複NG** 程度 |
| 同一人物 | 自動生成では **1日1時間帯まで** |
| 繁忙期 | **日によって必要人数を変えるルールなし**（土日+1名 · 年末強化 等は v2 候補 · §5-1） |
| 回避策 | セル「自由入力」で複数名を手打ち · **日別備考**に「本日5名体制」と書く |

→ 飲食店の「同時間帯にキッチン＋ホール＋レジ」は **v1 では表現しきれない**。v2 で §5 を実装する。
