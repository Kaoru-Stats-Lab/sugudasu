## 2. 用語

| 用語 | 定義 |
|------|------|
| **プロジェクト（シート）** | 1つの工程表。日ベース or 時間ベースは **作成時に固定** |
| **モード · 日ベース** | 開始/終了は **日付**（稼働日カレンダー適用）。連動単位は **日** |
| **モード · 時間ベース** | 開始/終了は **日時**（分精度）。夜間作業・DB移行など **日付をまたぐ** 想定 |
| **親ID** | 行が追随する先行行の ID。指定された行だけが親の終端に連動 |
| **クリティカルパス（CP）** | 親IDの **終端** に、CP指定の **子行** の開始/終了が **元の期間分** スライド |
| **ロック** | チェック列。ON の行は **連動で動かない** |
| **稼働日** | プロジェクトごとの曜日設定 + 手動で追加した稼働日 |

## 3. データモデル

### 3-1. プロジェクト

```json
{
  "id": "proj_xxx",
  "mode": "day | datetime",
  "minutePrecision": true,
  "accountDefaultMode": "day",
  "workingWeekdays": [1, 2, 3, 4, 5],
  "extraWorkingDates": ["2026-06-28"],
  "frozenColumnCount": 3,
  "columns": [],
  "rows": []
}
```

| フィールド | 説明 |
|------------|------|
| `mode` | **`day` or `datetime`** — 作成時選択 · シート内混在不可（SCH-F03） |
| `workingWeekdays` | 0=日 … 6=土。非稼働は日数カウントからスキップ（SCH-A03） |
| `extraWorkingDates` | 途中で手動追加した稼働日（ISO 日付） |

**アカウント既定（SCH-A01）:** 新規作成時のデフォルトモードを **アカウント設定に保存**。プロジェクト作成時に上書き可。

### 3-2. 列定義

```json
{
  "id": "col_assignee",
  "label": "担当",
  "kind": "text | number | select",
  "options": ["A", "B"],
  "widthPx": 120,
  "system": false
}
```

| システム列（固定 · 左Frozen） | 説明 |
|------------------------------|------|
| タスク名 | 必須 |
| 開始 | 日 or 日時（mode 依存） |
| 終了 | 日 or 日時 |
| 所要 | 日数 or 分（mode 依存 · 算出可） |
| 親ID | 追随先行 ID（空=直列のみ） |
| CP | クリティカルパス子として親終端にスライド（bool） |
| ロック | チェックボックス |

ユーザー列: **無制限追加** · kind は **テキスト / 数値 / 選択** の3種（SCH-C01）。

### 3-3. 行

```json
{
  "id": "row_001",
  "cells": { "col_assignee": "山田" },
  "start": "2026-06-26",
  "end": "2026-06-28",
  "duration": 2,
  "parentId": "row_000",
  "criticalPath": false,
  "locked": false
}
```

時間ベース例: `"start": "2026-06-26T22:00"`, `"end": "2026-06-27T06:00"`（夜間・日跨ぎ · SCH-A04）

## 4. 連動ルール

### 4-1. 直列（SCH-B02）

- **並行タスク概念なし**。表は **上から下** が基本の直列連鎖。
- **親ID 未指定** の行: 直上の非ロック行の **終了** の翌稼働境界から **開始** が決まる（日ベースは翌稼働日、時間ベースは終了直後）。

### 4-2. 親ID（SCH-B01）

- `parentId` を持つ行は、**親行の終了** に追随して **開始/終了を再計算**（自行の **所要** は維持が既定）。
- **CP（criticalPath）= true** かつ親IDあり: 親の **終端変更** に応じ、子の開始/終了を **元の期間分** まとめてスライド。

### 4-3. ロック（SCH-B04）

- `locked: true` の行は **連動計算で移動しない**。
- ロック行の **手動編集** は許可。下流への伝播は §4-4・§5 に従う。

### 4-4. 行挿入（SCH-B03）

- 挿入時 **開始/終了は空欄**。
- 提督が日付/日時を入れた **瞬間** に連動エンジン実行。

### 4-5. 稼働日（SCH-A03）

- プロジェクトごとに **稼働/非稼働曜日** を固定指定。
- **extraWorkingDates** で途中から稼働日を追加可能。
- 日ベースの「+N日」は **稼働日のみカウント**（土日祝スキップ · 手動稼働日はカウント）。

### 4-6. 時間精度（SCH-A02）

- 時間ベースは **分単位**（秒は切り捨て表示）。

## 5. 終了日の前倒し・後ろ倒し（SCH-C02 · 提督回答）

**採用方針: 「編集したセルが何の意味か」で分岐**

| ユーザーが編集したもの | 挙動 |
|------------------------|------|
| **所要**（日数/分） | 終了を再計算 → **下流・親ID追随・CP子** を連鎖 |
| **開始** | 所要維持で終了を再計算 → 同上 |
| **終了**を直接変更 | **終了を確定値（ピン）** とみなし、**所要 = 終了 − 開始** を逆算 → 下流・追随を連鎖 |
| **ロック ON** | 当該行は連動で動かさない。手動変更のみ |

**原則:** 直列表なので「終了を前倒し」= **所要が短くなる** か **開始が繰り上がる** のどちらかが起き、**その差分が下流に伝播**する。Excel の「終了だけ触って式が壊れる」を、エンジン側で **所要の逆算** により吸収する。

**UI ヒント（v1 最小）:** 終了列を直接編集した直後、トースト1行「終了を基準に所要を更新しました」（過剰ダイアログは出さない）。

## 8. xlsx Export（SCH-F）

| 項目 | 仕様 |
|------|------|
| 値 | **すべて文字列**（例: `'2026/06/26` · 先頭 `'` 相当のテキストセル）（SCH-F01） |

| オフライン | **localStorage** 一時保存 → オンライン復帰で同期。複数端末時は **編集権は1端末**（SCH-H03） |
| 履歴 | **v1 なし** — DB は **最新状態1つのみ**（SCH-H04） |

### 9-1. 編集開始 · 保存 · 編集終了（同義ではない）

| 操作 | 意味 | サーバー | 排他ロック |
|------|------|----------|------------|
| **編集開始** | この端末が編集モードに入る | ロック取得 API | **取得** — 他者は閲覧のみ |
| **保存** | いまの表を DB に書き込む | **PUT 最新1版** | **維持**（編集セッション継続） |
| **編集終了** | 編集モードを抜ける | **未保存なら先に保存を促す**（下記） | **解放** — 他者が編集開始可 |

**編集終了 ≠ 保存（同義にしない）**

- **保存だけ** — 長時間編集中に途中経過をサーバーへ残す。**ロックは握ったまま**。
- **編集終了** — **ロック解除が主目的**。未保存変更があるときは  
  `保存して終了` / `保存せず終了（破棄）` / `キャンセル` の3択（v1 最小）。

**編集終了だけ押して DB が更新されない** 状態を許すと、他者が古い版を見たまま編集開始できるため、**既定は「保存して終了」を推奨ボタン**にする。

**閲覧者:** 編集開始ボタンは disabled。「〇〇が編集中（閲覧のみ）」表示。

**オフライン（SCH-H03）:** 編集開始中は localStorage にも書く。オンライン復帰時 **保存** でサーバーへ。編集終了時も未同期があれば保存を促す。

**タイムアウト（v1.1 検討）:** 編集開始のまま放置 → ロック自動解放は **v1 では任意**（提督が要らなければ出さない）。
