## 1. seed と名簿指紋だけでは復元できない

画面に出る例:

```text
seed: 41502a127f4809593750bb123fc5f744
名簿指紋: f5e94abbcc06c372…（SHA-256 先頭16桁）
```

| 項目 | 何か | 単体で班を復元できるか |
|------|------|------------------------|
| **seed** | 乱数の種。同じ入力なら同じシャッフル順 | **いいえ** — 名簿と条件が別途必要 |
| **名簿指紋** | 実行時の名簿テキスト全体の SHA-256 | **いいえ** — 「今貼った名簿が当時と同じか」の**照合**用 |

班をもう一度同じにするには、最低限 **名簿全文 · 分割モード（N人1組 or M組）· 属性条件 · 名前ルール · seed** の5つが揃っている必要があります。

**正攻法:** 実行直後に **セッションJSON** をコピー/保存する（名簿・条件・seed が1ファイルに入る）。

## 2. 班を復元する（PC → 別端末 · 翌日）

### 手順 A — セッションJSON + 同期メモ（推奨）

名簿は SUGUDASU に送らない。**自分のクラウドメモ**に JSON を貼るだけ（提督の同期アカウント内）。

#### おすすめの同期メモ（OS横断）

| アプリ | 向き | iOS / Mac / Win / Linux |
|--------|------|-------------------------|
| **Google Keep** | 第一候補 · Googleアカウントがあれば | アプリ + **Web**（Linux は Web） |
| **Google ドキュメント** | JSON が長い · 研修名簿込み | **ブラウザで全OS** |
| **Microsoft OneNote** | M365 · 社内環境 | Win / Mac / iOS / Android / Web |
| **Notion** | 既にチームで使っている場合 | 全プラットフォーム |
| Slack 自分用 DM | インストール済みなら | 全OS |

> **Appleメモ**は Apple 端末間のみ — PC 幹事 → Android 会場には向きません。

1. 班分け実行後、結果の **「JSONをコピー（同期メモへ）」**  
2. **Google Keep** または **Google ドキュメント** に貼る  
3. 会場のスマホで同じメモを開き、JSON 全文をコピー  
4. `/group-split` の **「同期メモから JSON を貼り付け」** → **読み込んで再実行**  
5. **名簿指紋**が記録と一致するか確認  

### 手順 A' — セッションJSON（ファイル）

1. 班分け実行後、結果の **「セッションの保存 · 読込」** を開く  
2. **「JSONをコピー」** または **「JSONを保存」**  
3. 別の PC / スマホで `/group-split` を開く  
4. 同じアコーディオン内のテキスト欄に JSON を貼る  
5. **「読み込んで再実行」** を押す  
6. 表示された **名簿指紋** が記録（`f5e94abb…`）と一致すれば、同じ名簿で再現できている  

JSON の中身（例）:

```json
{
  "version": 1,
  "tool": "group-split",
  "roster": "氏名\t所属\n…",
  "mode": "groupCount",
  "param": 23,
  "constraints": { … },
  "columnMapping": { … },
  "seedHex": "41502a127f4809593750bb123fc5f744"
}
```

### 手順 B — 名簿を手元に残している場合

1. 当時と同じ名簿を左のテキスト欄に貼る  
2. 属性条件・名前ルール・組数を**当時と同じ**にする  
3. **セッションJSON を読み込む**（seed が入っている）か、同一タブで実行直後のまま再実行  
4. 名簿指紋が記録と一致するか確認  

> **注意:** seed を手入力する欄はありません。seed だけ Slack に残しても、名簿と条件がなければ復元できません。**JSON か結果 TSV を一緒に保管**してください。

### やってはいけないこと

- **「再シャッフル」** — 新しい seed が生成され、**別の班**になる（欠席対応にも使わない）

## 3. ハッカソン当日 — 欠席をスマホタップで再編（M02）

### 想定フロー（幹事がスマホを持つ）

**事前（PC · 前日〜当日朝）**

1. Excel 名簿で班分け実行（職能列 · 各組エンジニア必須など）  
2. **JSONをコピー（同期メモへ）** → **Google Keep** 等に貼る  
3. 参加者向けに Slack 用テキスト or TSV を投稿  

**当日（会場 · スマホ）**

1. スマホで `https://sugudasu.com/group-split` を開く  
2. **Google Keep** 等から JSON をコピー → 貼り付け欄へ → **読み込んで再実行**  
3. 欠席が判明したら、結果の **グループ内の名前チップをタップ**  
4. 確認ダイアログで OK → 名簿からその人が削除され、**同じ seed で再構成**（制約は維持）  
5. 緑トースト「○○ を欠席扱い · N名で再構成しました」  
6. **Slack用テキストを再コピー**してチャンネル更新 · スイッチャーへ差分を連絡  

### 同一端末で結果を開きっぱなしの場合

- JSON の読み込みは**不要**（`lastResult` に seed が残っている）  
- **名前タップだけ**で再編できる  

### タップ再編の中身（実装）

1. `removeNamesFromRoster` — 左の名簿テキストから該当行を削除  
2. `pruneConstraintsAfterNameRemoval` — 固定班・離すペアからも除去  
3. `executeSplit({ seedHex: lastResult.seedHex })` — **seed 維持**で再実行  
4. 班構成は変わる（人数が減ったため）が、乱数系列は同じ起点  

### スイッチャーへの連携

ツールは Zoom BR は動かさない。幹事が:

1. 更新後の Slack 班表を投稿  
2. 「G3 から田中欠席 · エンジニア全組維持」など **1行 DM**  
3. スイッチャーが Zoom 上で BR を手動調整

## 4. よくある誤解

| 誤解 | 正しい理解 |
|------|------------|
| seed をメモれば班が戻る | 名簿・条件も必要。**JSON が最短** |
| 名簿指紋から名簿が復元できる | 指紋はハッシュ。**逆算不可** |
| タップ再編で全員同じ班のまま | **欠席者以外で再割当**。人数が変わるので班は変わる |
| 再シャッフル = 欠席対応 | **新 seed** になる。欠席は **名前タップ** |
