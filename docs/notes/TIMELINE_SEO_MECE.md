## §1 刺さる「場面（シーン）」の MECE 分類

| 大分類 | 中分類（場面） | 巻き・押しの要因 | SUGUDASU Timeline が刺さる理由 |
|--------|----------------|------------------|--------------------------------|
| **A. 社内** | 研修・ワークショップ | ワーク白熱・質疑延長 | 機密予定を外部 SaaS に送らない。±5 で再計算 |
|  | 全社キックオフ・総会 | 挨拶延長・表彰遅れ | アンカー（昼食・バス）への干渉を即可視化 |
|  | 社内懇親会・忘年会 | 乾杯遅れ・ビンゴ | スマホ片手で終了時刻の帳尻 |
| **B. 対外** | ウェビナー・セミナー | 機材トラブル・登壇遅刻 | 1コマ遅延で終了予定が即更新→アナウンス文コピー |
|  | ハッカソン・勉強会 | アイデアソン延長・飛び入り LT | PC なしで会場歩きながら更新 |
|  | 展示会・ポップアップ | 滞在超過・ブース転換遅れ | 休憩・交代への食い込みを逆算 |
| **C. 公共・教育** | 学校行事・OC | 移動遅れ・来場波 | 紙進行表から脱却・チャットへコピペ |
|  | 学会・研究発表 | 発表超過・ディスカッション | 次セッションまでの残り時間 |
|  | 地域イベント・祭り | 雨天順序変更・搬入遅れ | 予算なし団体でも無料 |
| **D. 私的** | 結婚式・二次会 | 入場遅れ・余興押し | 歓談を何分削るか即算 |
|  | 同人即売・自主ライブ | 設営遅れ・機材 | Excel 数式崩壊を回避 |

## §2 刺さる「職業・役割（ペルソナ）」の MECE 分類

| 大分類 | 役割 | Pain | 刺さる機能 |

## §3 aha! moment（FAQ 本文に織り込む）

1. **司会:** 挨拶を 5→10 分に変えた瞬間、後続 15 コマが一斉再計算され撤収可否が 1 秒で判明。
2. **人事:** 休憩 60→50 分に減らすと午後が定時 17:00 に戻り、安心してアナウンス。
3. **幹事:** PC なし会場で URL を開き、Slack に最新タイムテーブルを貼付。

## §4 競合 FAQ の SUGUDASU 適応方針（Rundown 等）

| 競合の問い | SUGUDASU の正直な答え |
|------------|----------------------|
| スプレッドシートで足りない？ | 連動再計算・モバイル・コピー共有に特化。秒単位・リアルタイム同期・テレプロは **コア外** |
| 料金・イベント課金 | **無料** · 登録不要（コア） |
| チーム vs ゲスト | コアは **1 端末 + コピー共有**。複数端末同期は将来 Sync ライン（未掲載） |
| 本番信頼性 | 静的配信 · オフライン同等のブラウザ処理。変更は手動コピーで共有 |
| 研修必要？ | 不要。3 タップ（選択→±5→コピー）想定 |
| 返金・VAT・教育割引 | **該当なし**（無料ツール）— FAQ に載せない |
