## 1. 意思決定ログ（ナラティブの変遷と死因分析）

本ツールは、複数AI（Grok、ChatGPT）および提督とのドロドロとした泥仕合を経て、極限まで機能が削ぎ落とされた。開発者は、過去に却下された以下の「誘惑」を絶対に再実装してはならない。

### 【変遷1】 `/budget-pie`（自動比例配分）の死

* **初期案：** 1つのスライダーを動かすと、他項目が比率を維持して「自動縮小」するオシャレな円グラフUI。
* **却下理由：** 「比例配分は人間の政治的思考とズレる」という致命的バグ。現場の会議では、広告費を増やしたからといって交通費まで均等に削られるのは困る（営業が止まる）。特定の項目だけを生贄に捧げるのがリアルな政治であり、ツールの自動介入はただのお節介であるため、**自動配分機能を完全剥ぎ取り**。

### 【変遷2】 `/budget-limit`（Bar ✕ Limit Line）の死

* **中期案：** 円グラフを捨て、上限線（Limit Line）に対して各項目（Bar）がどれだけ突出しているかを可視化する棒グラフUI。
* **却下理由：** グラフィック表現そのものが「見栄えの罠（デザインの病）」であった。会議の戦場で人間が凝視しているのは、美しいグラフではなく「生々しい数字」と「あと何円（何時間）で檻に収まるかという超過額」そのもの。また、項目名が長い場合に横幅が崩壊するリスクがあり、10年メンテナンスフリーの思想に反する。**グラフ表現（Pie、Barともに）を完全剥ぎ取り**。

### 【変遷3】 `/limit-counter`（汎用総量カウンター）の死

* **後期案：** 予算に閉じず、工数、時間、シフトなどあらゆる総量規制に対応する汎用的なカウンター。
* **却下理由：** 「なんでもできる」は「誰にも刺さらない」と同義。実務家は「総量規制」という言葉では検索しないし、課題を認知できない。入り口は「予算の削り合い」「評価の枠取り」という最も血が流れる泥臭い具体ドメインに固定すべきという結論に至り、**抽象的な汎用ブランディングを完全焼き払い**。

### 【変遷4】 自動Sync（クラウド・リアルタイム同期）の死

* **直近案：** 人事が各部署の原案を事前に集約するため、WebSocketやP2Pでリアルタイム同期する部屋を作る機能。
* **却下理由：** 「社員名 ✕ 仮評価」というSクラスの極秘情報を、ログインも認証もない外部の同期空間に流すなど、コンプライアンス的に狂気の沙汰。大企業のセキュリティ監査によって一撃で違法ツールに指定され、全社プロキシで遮断される。**外部通信ロジックを完全焼き払い**。

### 【変遷5】 JSON等による物理ファイルダウンロード保存の死

* **直近案：** 完全非送信を守るため、下書き状態を `.json` などの物理ファイルとしてローカルPCにダウンロードさせる機能。
* **却下理由：** ファイルの生成は実務上、最悪の摩擦を引き起こす。「ダブルクリックで開けない（ブラウザにドラッグ＆ドロップし直す必要がある）」という非IT層への認知の壁、および大企業の「機密情報を含む未知のファイルのメール/Slack添付禁止ポリシー」に直撃して流通が止まるため、**ファイル出力機能を完全焼き払い**。

## 2. SUGUDASU憲法チェック

* **【F1】登録不要・ログイン不要・ブラウザ完結：** サーバー不要、静的HTML/JSのみで動作。
* **【F2】完全非送信：** 1バイトも送信しない。機内モード（完全オフライン）でも100%動作する安心の物理的シグナルを担保。
* **【F3】完全静的配信：** インフラ維持コストは永久に $O(0)$。
* **【F4】要塞化の禁止：** データの永続蓄積・ファイル管理は一切行わない。画面を閉じたらすべて消える。
* **【F5】Copy-first：** 手元のExcelからコピペでIN、15分格闘、TSVコピーでExcelへ戻って15秒離脱。

## 3. 絶対にやらないこと（DENYガードレール）

1. **いかなる種類のグラフ（円、棒など）も描画しない。**
2. **WebRTCやWebSocketなどの外部通信、サーバー同期を一切行わない。**
3. **`.json` や `.csv` などのローカルファイルダウンロード機能を作らない。**
4. **給与計算、職能給、号俸テーブルなどの「評価制度」の計算ロジックを持たせない。**
5. **URLのハッシュ（`#`）にデータを埋め込んで変形させない（ブラウザ履歴への残留やSlackクローラーによるセキュリティ誤認を防ぐため）。**

## 4. 機能要件（MUST）

### ① 入力データの超強力デトックス（パース）

ユーザーがエクセルから適当にコピーしてきた行データを、正規表現で強引に `[要素名(String), 仮レーン名(String)]` の配列に変換する。ノイズ（全角、記号）は剥ぎ取る。

### ② スロット（定員制限）付きカンバンレーン

* ユーザーが画面上で評価ランク（S, A, B...）ごとのレーンを作成でき、各レーンの上部に「上限人数（定員）」を直入力できるボックスを配置する（例：`S [ 3 ] 人`）。
* 枠数はオプショナル（空欄なら制限なし）とする。これにより、「枠数がまだ決まっていないマネージャー個人の事前ドラフト作成」と「枠数が決まった後の本番会議」の両方のユースケースを1つの画面で滑らかに処理する。

### ③ 定員オーバー時の「赤色明滅アラート」（政治的妥協のトリガー）

* 要素（付箋）をドラッグ＆ドロップして上限人数を超えた瞬間、レーンのヘッダーが **`🚨 S [ 4 / 3人 ] 1人超過`** と真っ赤に激しく点滅する。
* ツールは自動で数値を戻したりしない。「誰を落とすか決まるまで赤点滅が消えない」というシステム的な視覚的暴力を突きつけることで、マネージャー同士の「じゃあウチの鈴木を落とすから、そっちの佐藤を入れろ」という人間同士のドロドロした政治的妥協（合意形成）を爆速でドライブする。

### ④ 「一発OK（オールグリーン）」の祝福UX

* 事前案のインポート直後、すべてのレーンで超過が $0$ であった場合、画面上部は警告を止め、静かで美しい緑色の **`「✅ 枠内（超過なし）」`** を巨大明示する。会議を開く前に「一発OK」が視覚的に証明され、1秒で会議をクローズできる最大報酬の体験を提供する。

### ⑤ 復元用暗号風テキストコード（完全無痕跡テキスト）による「人間系Sync」

データをURLにもファイルにも保存しない代わりに、可搬性を「テキストのコピペ」に完全還元する。

* **保存時：** 「下書きコードをコピー」を押すと、現在の配置状態をシリアライズした1行のテキスト文字列がクリップボードにコピーされる。
* **可搬（人間系Sync）：** マネージャーはこの「文字」を自分のExcelのメモ欄やSlackの自分用メモに貼り付けて保管する。人事に提出する時も、メール本文にこの文字をコピペして送るだけ。ファイル添付禁止ポリシーに100%引っかからない。
* **復元時：** 人事はメールからその文字列をコピーし、SUGUDASUのインポートエリアに `Ctrl+V` でブチ込むだけで、各部署のドラフト画面を一瞬で完全復元し、Excelマスターへ集約できる。

#### 復元コード形式（v0.1 確定）

```text
SUGUDASU-SLOT- + Base64(UTF-8 JSON)
```

* **v0.1では圧縮アルゴリズムを導入しない**（依存追加ゼロ）。
* Prefix `SUGUDASU-SLOT-` は **必須・固定**（将来互換）。
* JSON は `version: 1` を持つ。**キー名の勝手な短縮・変更禁止**。互換性を壊す変更禁止。
* 復元後はレーン順序・付箋順序・レーン名・上限人数が **完全一致** すること。

#### レーン削除（v0.1 確定）

* レーン削除時、付箋は削除しない。
* すべて **「不明」** レーンへ移動する（存在しなければ自動生成）。
* 削除前にブラウザ標準 `confirm()` で確認する（編集モーダルとは別。インライン編集用モーダルは禁止）。

#### パース不能行・固定レーン名

* パース不能行は捨てない。**「不明」** レーンへ入れる（「要確認」は使わない）。
* 「不明」はカテゴリ名として固定。

#### レーン編集UI

* 編集できるのは **名前** と **上限人数** のみ。**インライン編集**。編集用モーダル禁止。
* 上限が空欄のときは **無制限**（超過判定なし）。

### ⑥ 完全非送信ステータスの常設明示（安心のシグナル）

画面最上部に、緑色の鍵マークとともに以下のテキストを常時明示し、ユーザーが操作するたびに「ローカルで再計算完了」と一瞬ピカッと光る手応え（シグナル）を与える。

> **`🔒 完全非送信ステータス：有効（このツールはインターネットと通信していません。機内モードでも動作します）`**

## 5. UI/UX・レイアウト仕様

長い名前（例：「2026年度第2四半期特別優秀賞候補者（営業部）」）に対する耐性を100%保証するため、グリッドレイアウトによる横並びを制限し、縦スクロールだけで破綻しない堅牢性を死守する。

```text

 こんな会議で使います

 【 🚨 S評定 [ 4 / 3人 ] 1人超過 】
```

## 開発をスタートするAgentへの伝言

このツールは「計算と制限のパレット」**であり、**「意思決定エンジン」ではない。
綺麗に整ったグラフを描画するライブラリを導入しようとしたり、ファイルを生成させようとした瞬間、大企業の人事会議室から叩き出されてプロダクトは即死する。
ただひたすらに、「コピペされた文字を扱い、枠から溢れたら赤く光らせ、復元コードという名のただのテキストで安全に持ち運ばせる」という泥臭いコピペファースト（【F5】）の筋肉だけに、すべてのコードを集中させよ。
