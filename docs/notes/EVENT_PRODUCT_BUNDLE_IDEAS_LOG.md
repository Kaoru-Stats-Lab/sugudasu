### 1.1 ブレスト拡張メモ — ID 02 を「イベント印刷キット」として見る（2026-06-26）

**提督メモ:** 02 は当日だけでなく **事前準備〜事後フォロー** まで含むニーズの入口になりうる。AdSense・幹事 SEO 向きの「印刷物系を厚くする」は、この束全体を指す可能性。

| フェーズ | 印刷物・成果物（案） | 既存 ID / ツールとの関係 |
|----------|----------------------|---------------------------|
| **事前** | 会場オペカード · 搬入導線 · 受付配置図メモ | **02** · **01** スタッフ配置 · **14** 箱ラベル |
| **直前** | 受付チェック表 · 名札 | **07** · `label` |
| **当日** | 緊急連絡カード（ポケット）· 司会カンペ抜粋 | **02** · **05** timeline 印刷 |
| **事後** | **紙アンケート** · 満足度シート · 御礼文の同封用 QR（任意） | **16**（新規案）· **08** サンクス文 · `report` 議事録 |

**紙アンケート（16）の憲法上の切り口（判断ではない）:**

- **やる:** 設問テンプレ → **印刷用 PDF**（Google フォームの代替ではなく **物理回収** 向け）
- **やらない（フル SaaS 化）:** 回答のクラウド集約 · リマインド · 未読管理（掲示板・T10 と同型で重い）
- **任意のコア拡張:** 回収後に幹事が **手入力 or OCR なしで手集計** → 台帳 X09「リッカート手元集計」と統合する案

**02 単体 vs 束ねる案:**

- **A:** 02 のまま「会場オペカード」1本（スコープ小 · MOC 速い）
- **B:** 製品名を **「イベント現場キット」** 的なハブにし、タブ or `sg-segment` で **事前/当日/事後の印刷物** を同じ UI から出す（02 を親 ID にする）

### 1.2 ブレスト拡張メモ — ID 16 紙アンケートのテンプレ設計（2026-06-26）

**提督メモ:** パターンは複数ありそう。**イベント種別ごとのテンプレ**を選び、幹事が最小限カスタムする方が楽。ただし **「満足しましたか」だけの空洞テンプレは不可** — そのイベントの幹事が本当に知りたい問いが最初から入っている必要がある（芯を食ったテンプレ）。

#### テンプレ軸（案 · 未確定）

| テンプレ ID（案） | 想定イベント | 芯になる問い（例 · ブレスト） | 用紙の型（案） |
|-------------------|--------------|------------------------------|----------------|
| `training-internal` | 社内研修・新人研修 | 演習時間は足りたか · 難易度 · 次回テーマ希望 · 持ち帰り1つ | 5段階×4問 + 自由記述1 |
| `study-session` | 社内勉強会・LT会 | 発表の分かりやすさ · 時間配分 · 次回聞きたいテーマ | 短尺（A4半分 or 1/3） |
| `workshop-hands` | ワークショップ・ハッカソン | 班編成の公平感 · ルール説明の明確さ · 設備・Wi‑Fi · 次回参加意向 | `group-split` 利用イベントと親和 |

> **16 vs 17:** 上記 `workshop-hands` は **参加者の事後満足度**。ハッカソン/アイディアソンの **審査員採点・評価シート** は **ID 17**（§1.4）。

| `social-after` | 懇親会・飲み会（社内） | 会場・アレルギー配慮 · 二次会要否 · 金額帯の妥当感 | 匿名推奨 · 質問数少なめ |
| `seminar-external` | 外部向けセミナー・説明会 | 目的達成度 · 資料のわかりやすさ · **個人情報を書かせない** 連絡手段 | 記名/匿名切替 · 会社名任意 |
| `compliance-short` | コンプライアンス・安全研修 | 理解度チェック（○×数問）· 事例の実務関連度 | 記名 · 監査提出を想定した欄 |
| `facilitator-retro` | 幹事・運営者向け振り返り（非公開） | トラブル有無 · 進行表のズレ · スタッフ連絡の詰まり | 印刷は幹事のみ · 参加者配布しない |

#### カスタマイズの境界（案）

| 層 | 幹事が触る | テンプレが固定する |
|----|-----------|-------------------|
| **イベント名・日付・主催** | ○ 入力 | ヘッダー枠の位置 |
| **設問文の微修正** | ○ 1問あたり短文編集可 | 質問の **意図カテゴリ**（時間・難易度・次回など）はテンプレが定義 |
| **設問の追加・削除** | △ 上限あり（A4 1枚に収まる範囲） | 必須コア問いは **削除不可**（芯を食う） |
| **5段階ラベル** | ○「非常に〜」文言 | 尺度の意味はテンプレ注釈 |

#### 「芯を食った」とは（品質メモ · 判断ではない）

- **NG 例:** 「本日の満足度」「ご意見ください」だけ · どのイベントにも当てはまる汎用5問
- **OK 例:** 研修なら「演習→本番のギャップ」「持ち帰りアクション」 · ハッカソンなら「発表までの時間」「メンター充足」
- **SUGUDASU らしさ:** テンプレは **幹事ペルソナ（P-A）の現場語彙** で書く（総務・人事が稟議に貼っても不自然でない）

#### 既存との接続

- **X09** リッカート手元集計 — 5段階設問が多いテンプレとセット想定
- **08** サンクス文 — アンケート同封・メール添付の文案を同じイベント名で揃える
- **02 / イベント印刷キット（B案）** — 事後タブでテンプレ選択 → PDF

### 1.3 ブレスト拡張メモ — 印刷の「最後のライフライン」ネットプリント（2026-06-26）

**提督メモ:** 会場オペカード・アンケート等の PDF は、**自宅・会社プリンタがない幹事** や **当日コンビニで増刷** の逃げ道として、**ネットプリント / プリントネットワーク系への接続** があってもよい（最悪の手段 · 任意）。

#### 想定フロー（案 · SUGUDASU 側は軽く）

```
PDF 生成（ブラウザ内）
  → ① 通常: ブラウザ印刷 / PDF 保存（正規ルート · L3 緑）
  → ② ライフライン: 「コンビニで印刷する」導線（外部サービスへ）
        ユーザーが PDF を手元に保存 or アップロード（SUGUDASU は印刷ジョブを預からない）
```

| サービス（例） | 店舗系 | 接続の現実的な形（案） |
|----------------|--------|------------------------|
| **ネットプリント**（セブン・ローソン等） | セブンイレブン · ローソン等 | 公式アプリ/サイトへ誘導 + **PDF 保存済み** の手順1行。深い API 連携は不要 |
| **プリントネットワーク** | ファミマ等 | 同上 |
| **PrintSmash** 等 | 複数チェーン | 同上 |

**SUGUDASU がやる / やらない（ブレスト · 未確定）:**

| やる（軽量） | やらない（重い） |
|--------------|------------------|
| PDF 出力後の **「コンビニ印刷の手順」** 折りたたみ or モーダル | 自前サーバーで印刷キュー保管 |
| 各サービスの **公式 URL へのリンク**（アフィではない） | OAuth · 店舗在庫 API |
| 用紙サイズ・カラー/モノクロの **事前注意**（A4 · 両面等） | 印刷代の見積もり（店舗依存） |

#### 憲法・信頼との接続（判断ではない）

- **F2 維持:** PDF バイナリを SUGUDASU サーバーに送らない。ユーザー → コンビニサービスは **ユーザー操作の延長**。
- **機密:** 会場オペカード（Wi‑Fi パスワード等）· 記名アンケートは **コンビニ経由に個人情報が載る** 旨を1行注意（幹事判断）。
- **UI 層:** 正規は **ブラウザ印刷（緑）** · ネットプリントは **青リンク or 二次ボタン**（L3 緑を奪わない · `DESIGN_GUIDELINE` 3層アクション）。

#### 適用候補 ID

- **02** 会場オペカード · **07** 受付表 · **14** 箱ラベル · **16** 紙アンケート — **印刷物を出すもの全般**

#### コピー例（画面用 · 案）

> プリンタがない場合: PDF を保存し、[ネットプリント](https://www.printing.ne.jp/) 等からコンビニで印刷できます。パスワード入りの用紙は内容にご注意ください。

### 1.4 ブレスト拡張メモ — ID 17 評価シート・ルーブリック（ハッカソン / アイディアソン）（2026-06-26）

**提督メモ:** アンケート（16）があると、**審査・採点用の評価シート** もセットで欲しくなる。ハッカソン・アイディアソンは特に顕著（審査員複数 · チーム複数 · 発表順）。

#### 16（満足度）と 17（評価）の違い

| | **16 紙アンケート** | **17 評価シート** |
|--|---------------------|-------------------|
| 誰が書く | 参加者全員（またはサンプル） | **審査員・メンター・運営** |
| いつ | 事後 | **デモ後・ピッチ後**（当日） |
| 目的 | 満足度・改善・次回 | **順位・表彰・公平な採点の記録** |
| 形式 | 5段階・自由記述 | **ルーブリック（評価軸×点数）** · チーム名欄 |
| 集計 | X09 手元集計（任意） | **審査員ごとの紙 → 幹事が合算**（サーバー集約なし） |

#### テンプレ軸（案 · 芯を食った評価軸）

| テンプレ ID（案） | 想定 | 評価軸（例 · ブレスト） | 用紙の型（案） |
|-------------------|------|--------------------------|----------------|
| `hackathon-judge` | 社内ハッカソン · 成果物あり | 完成度 · 技術的挑戦 · UX · プレゼン · チームワーク | 各軸1〜5点 + コメント · **チーム名行を N 件** |
| `ideathon-judge` | アイディアソン · 非エンジニア多め | 課題の鋭さ · 解の独創性 · 実現可能性 · ストーリー · 発表の分かりやすさ | バックログ **緩和型**（各組必須なし）と整合 |
| `demo-day-short` | ライトな発表会 · 時間短い | 3軸のみ（内容・伝わりやすさ · 実務への応用） | A5 or 半分 · 1チーム1枚 |
| `peer-feedback` | 参加者相互評価（任意） | 良かった点1 · 改善1 · 学び1 | 匿名 · 記名切替 |
| `mentor-sheet` | メンター所感（非公開） | 技術的助言の要否 · 詰まりポイント | 幹事のみ回収 |

#### `group-split` との接続（案）

- 班分け結果 TSV の **チーム名一覧** を読み込み → 評価シートの **行見出しを自動生成**（手打ちミス防止）
- **M13** BR/Slack 対応表と同じイベント文脈で、「班が決まった → 評価用紙を印刷」

#### カスタマイズ・集計（案）

- 評価軸の文言微修正 · 満点スケール（5点 / 10点）· 審査員名欄
- 回収後: 幹事が各紙の点数を **ブラウザ内に手入力** → 合計・順位表（**X09 拡張** または 17 専用の軽い集計 UI）— サーバーには送らない
- **§1.3** ネットプリント: 審査員人数×チーム数で **枚数が増える** → ライフラインの価値が特に高い

#### 製品の置き方（案）

- **16 と同一ツール** の `sg-segment` タブ「満足度アンケート | 審査評価シート」
- または **17 単体**（SEO: 「ハッカソン 評価シート テンプレ」）

## 2. イベントライフサイクルマップ（構想・未確定）

名簿（TSV/JSON）と進行表（Timeline）を背骨に、コピーと A4 印刷でつなぐ **束ね図のたたき**。実装順・必須ツールではない。

```
【事前：テキスト作成と構造化】
  03 日程調整文案（X02 系） → 調整さん等へ接続
  group-split → 04 M13 BR/Slack 対応表
  warikan → 06 店舗リスト LINE 用整形

【直前：現場インフラの物理パッキング】
  名簿 → label → 07 受付チェック表
  名簿 → 01 スタッフ配置表
  02 会場オペ・緊急連絡カード
  14 備品・箱ラベル

【当日】
  timeline（コア） → Sync（共有・版）
    05 レーン別出力
    09 ピン留め1件
    10 版差分サマリ
    11 Slack スナップショット
    13 緊急アナウンス原稿
    15 残り5分バッファ
  12 リハ/本番（長期）

【事後】
  report · lottery · 08 サンクス文
  16 紙アンケート・満足度シート（印刷）— 任意で X09 手元集計
  17 評価シート・ルーブリック（ハッカソン/アイディアソン · 審査員用）
```

**02 拡張メモ（2026-06-26）:** 直前ブロックの「02 会場オペカード」は **事前〜当日** の物理キットの核。事後は **16** で接続。

**既存ツールとの対応（参考）:** `warikan` · `group-split` · `timeline` · `fair-draw`/`lottery` · `present` · `label` · `report` — `TIMELINE_TOOL_SPEC.md` §1-1 束ね図の拡張案。
