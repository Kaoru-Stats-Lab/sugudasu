## Note — 本文（そのまま投稿可 · 約950字）

カタログやパンフ、広報誌——紙媒体の案件では、進行表を Excel で組み、修正はメールや LINE、確認は紙のゲラ、という組み合わせがまだ多いと思います。

私も（あるいは周りの現場でも）、次のような場面に心当たりがあります。

- サーバーに `_final` と `_確定02` が並んで、**どれを書き出せばいいか分からない**
- 再校で直したはずの赤字が、校了間際のゲラで**古いデータに戻っている**（先祖返り）
- クライアントの校正戻りが遅れただけで、進行表のマス目を**手でずらす夜**が続く
- 朝礼では**A3の壁貼り**を見ているのに、事務所の Excel だけが先に進んでいてズレる

施工管理アプリや IT 向けの進捗ツールを入れようとして、デザイナーが「入力が面倒」「イラレの手が止まる」と定着しなかった——そんな話もよく聞きます。問題はツールの良し悪しだけでなく、**紙の運用（印刷・掲示・ゲラ）とデジタルの正本が二重**になっていることにあるのかもしれません。
