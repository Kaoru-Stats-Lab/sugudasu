# 入力データ非送信 — 主張ポリシー（SSOT）

**更新:** 2026-07-18  
**ステータス:** 方針確定 · **共通バッジ実装済み**（`data-sg-privacy-badge` · `sugudasu-shell.js`）· ユーザー向け矛盾表現の是正済み  
**関連:** [`tools/statements.html`](../../tools/statements.html) · [`DESIGN_GUIDELINE.md`](../DESIGN_GUIDELINE.md) §3.6b · Sync [`STATEMENTS_SYNC_PAGE_DRAFT.md`](STATEMENTS_SYNC_PAGE_DRAFT.md)

## 0. 用語（厳守）

| 用語 | 意味 |
|------|------|
| **入力データ** | ユーザーがフォーム・テキストエリア・ファイル選択等で入れた内容そのもの（名簿・金額・画像・表など） |
| **テレメトリ** | GA4・（予定）AdSense・Google Fonts・ページ配信など、**ページ側が発生させる通信**。入力データを含まない |

**主張してよいこと:** 「入力データを（処理目的で）サーバーに送信しない」  
**主張してはいけないこと:** 「通信ゼロ」「通信が一切ない」「サーバーと一切やり取りしない」「Network でリクエストが出ない」など、**通信全般の不在**

---

## 1. 浅いバッジ（恒久文言）

**核（必須形）:**

```text
{対象物}はサーバーに送信しません
```

例:

- `入力内容はサーバーに送信しません`
- `景品情報はサーバーに送信しません`
- `名簿はサーバーに送信しません`

**禁止:** `外部送信ゼロ` · `通信ゼロ` · `一切送信されません`（対象が曖昧な場合）· 鍵アイコン / SECURE 風装飾の新規追加

**リンク:** バッジ自体はテキストのみ。約束ページへの導線は既存フッターのみ（バッジ周りに「詳しくは」を足さない）。

---

## 2. 深い安心（検証・テレメトリ開示）

- **正本:** `/statements` のみ
- Network 確認で言うべきこと: **入力データをボディに含む POST が出ないこと**（ページ読込・広告・解析・フォントの GET は別）
- テレメトリ開示は「何を・何のため・入力データと無関係」の **3点セット**（§2.1）

### 2.1 約束ページ追記案（骨子 · 文言は提督確認後に確定）

> このサイトは、皆さまに無料でツールを使い続けていただくために、以下の外部サービスを利用しています。いずれも、あなたが各ツールに入力・作成した内容（テキスト・画像・データ）を送信するものではありません。
>
> - **Google Analytics（GA4）:** どのツールがよく使われているかを把握し、今後の開発の優先順位を決めるために利用しています。計測されるのはページの閲覧・操作の統計のみです。
> - **Google AdSense（導入予定）:** 運営を維持し、今後も無料でツールを提供し続けるための広告収益のために利用予定です。表示される広告は入力内容に基づくものではありません。
> - **Google Fonts:** サイトの表示品質を保つために利用しています。フォント読み込みの際に Google への通信が発生しますが、入力内容とは無関係です。

※ AdSense は **予定** と書く（現在形で断定しない）。Fonts は「表示のための通信」まで。IP 等の深掘りは別途確認。

## 3. 共通コンポーネント（実装案 · 承認後）

| 項目 | 案 |
|------|-----|
| 置き場 | **実装済み:** `assets/sugudasu-shell.js` の `initPrivacyBadges()` |
| マークアップ | `<p data-sg-privacy-badge data-subject="入力内容" data-extra="任意の接尾" class="…"></p>` |
| 出力 | `{subject}はサーバーに送信しません`（+ optional ` · {extra}`）。テキストのみ · リンクなし |
| CSS | 呼び出し側の emerald / violet pill クラスを維持。**Sync 用バッジと色・文言・アイコンを共有しない** |
| 検証 | 将来: 禁止語（通信ゼロ等）の HTML ガードを `validate:` に追加可 |

**ハードコード禁止:** 新規ツールは本コンポーネントのみ。個別に文言を書くことを禁止（README · DESIGN_GUIDELINE に明記）。

## 4. Sync ガードレール（設計のみ · コード実装は将来）

Sync（sync.sugudasu.com）は、ユーザーが明示的に有効化したときだけ入力データ等がサーバーへ送られる **初めての例外レーン**。

| # | 方針 |
|---|------|
| S1 | Sync 有効 UI には **専用表示**（例: 「このデータは同期されます」）。コアの非送信バッジと **見た目・色・文言パターンを流用しない** |
| S2 | **デフォルトは Sync オフ**。明示操作なしに自動送信する導線を作らない |
| S3 | 既存無料コアの「入力データ非送信」前提は **変更しない**。Sync はオプトイン追加であり、コア既定動作に制約を課さない |
| S4 | Sync の公開約束はコア `/statements` と混同せず、将来 `sync.sugudasu.com/statements`（[`STATEMENTS_SYNC_PAGE_DRAFT.md`](STATEMENTS_SYNC_PAGE_DRAFT.md)）側で開示する |
