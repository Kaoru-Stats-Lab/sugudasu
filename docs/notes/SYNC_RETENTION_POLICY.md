# SUGUDASU Sync — タイムライン保持・削除ポリシー

**更新:** 2026-06-25（提督確定 — 無期限保持禁止）  
**関連:** `SUGUDASU_SYNC_LINE.md` §3-2 要素 F/H · `SYNC_S1_ARCHITECTURE.md`

---

## 1. 原則

| 原則 | 内容 |
|------|------|
| **無期限クラウド保存はしない** | イベント単位の使い捨て mental model · DB 肥大・コスト防衛 |
| **課金でも同時ルーム上限** | Supabase 行数 cap · 詳細は `SYNC_STORAGE_QUOTAS.md` |
| **長期ログはユーザー JSON** | エクスポート DL · クラウドは短期バッファ |
| **削除は一等市民** | オーナーがいつでもルームごと消せる（S1 必須） |
| **自動パージ** | `retain_until` 経過後にバッチ削除（S2 以降の cron） |
| **アーカイブ ≠ 永久保存** | エクスポート or テンプレ取込 · クラウドに残し続けない |

**コア無料**は従来どおり localStorage — ユーザーがブラウザデータを消すだけ（サーバ負荷ゼロ）。

---

## 2. 削除の種類

| 種類 | 誰 | いつ | 実装 |
|------|-----|------|------|
| **手動削除** | ルームオーナー | いつでも | `DELETE sync_rooms` → `sync_room_states` CASCADE |
| **自動パージ** | システム | `retain_until < now()` | pg_cron / 日次 Edge（S2+） |
| **エクスポート後削除** | オーナー | イベント終了後 | JSON DL → 手動削除（要素 F · `SYNC_STORAGE_QUOTAS.md` §4） |

閲覧 URL はルーム削除と同時に **無効**（404）。

---

## 3. `retain_until`（保持期限）

`sync_rooms.retain_until` — **必ず設定** · NULL 禁止（DB デフォルトあり）。

| 課金・状態 | 保持目安（案） | 根拠 |
|------------|----------------|------|
| **trial** | 最終更新 + **14 日** または作成 + **30 日**（短い方） | 試用 · 放置ルーム掃除 |
| **イベント単位課金**（S3） | `event_date` + **7 日**（§4 課金モデルと整合） | 当日〜翌週の振り返りのみ |
| **延長購入**（将来） | チェックアウトで `retain_until` を延長 | 例外は課金で明示 |

**更新ルール:** 編集者が保存するたびに trial は「最終更新 + 14 日」に **延長しない**（放置検知のため）。有料イベント枠のみイベント終了日基準。

## 4. 「アーカイブ」（要素 H）の再定義

| 誤解 | 正しい意味（Sync） |
|------|-------------------|
| クラウドに何年も残す | **×** |
| 翌年のテンプレとして使う | 終了前に **JSON エクスポート** → コア or 新規ルームへ取込 |
| Pro の履歴一覧 | **削除前のスナップショット数件**（メタのみ · payload は期限付き）— S3+ 検討 · 無期限は不可 |

## 6. コスト防衛

- `payload` jsonb は **ルームあたり1行** — ルーム削除でまとめて消える
-  revision 履歴テーブルは **S2 では作らない**（最新1版のみ · `sync_room_states`）
- 過去版が必要なら **エクスポート**（クライアント）— サーバに版歴を積まない
