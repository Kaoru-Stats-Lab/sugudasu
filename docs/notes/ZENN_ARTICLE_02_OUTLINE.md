## H2 構成

### 1. AI の使い分け — 企画と実装を混ぜない

**書くこと**

| 役割 | 担当 | やること | やらないこと |

| 企画・表 | Gemini 等 | テーマ一覧・スケジュール・比較表 | **本文・コードの遂行** |

- **Vibe Coding** は使わない（読者に伝える必要なし · 自戒）

**避ける:** 各社の内部プロンプト・リーク URL · モデル名の盛り

### 2. Cloudflare Pages を選んだ理由 — サーバー代より「触らない」

**書くこと**

- 静的 HTML + `build-pages.mjs` で `dist/` 生成 · Git push で配信
- 見積ツールは **ブラウザ内 JSON** · 会員 DB なし（#1 と接続可 · 繰り返しは短く）
- 月額を抑えるより、**ユーザーデータを預けない** 設計が先
- Pages Functions は使わない（現状）— 複雑さとコストを避けた

**技術用語（Zenn 向け · 補足付き）**

- **静的サイト生成（SSG）** — ビルド時に HTML を焼く。リクエストごとにサーバーで組み立てない
- **クライアントサイド** — ブラウザの JavaScript が入力を処理

**スクショ候補:** hub · `updates.html` · Cloudflare ダッシュボード（ドメインのみ · 秘密なし）

### 3. Form → changelog の速度 — フィードバックを製品に戻す

**書くこと**

- `updates.html` + フォーム → GAS → `data/changelog.json` → 再ビルド
- 例: 見積→請求転用の案内追加（2026-06-17）— **ユーザー要望が changelog に載る** 流れ
- PR TIMES は見送り、**自分で記事（Zenn）+ X** に寄せた判断（1〜2 文 · 誇らない）
- 修正のたびに長いデプロイ手順を踏まないよう、ビルドスクリプトを1本化

**読者への価値:** 個人開発でも「改善ログを公開する」だけで信頼が積み上がる

### 4. 「スグダス」はスバルが占有している — 同名キーワードと Query Disambiguation

**書くこと（案B · 技術ストーリーとして1節 · 約 400 字）**

- Google / X で **「スグダス」** と検索すると、SUBARU 認定中古車 **SUGDAS** がタイムライン・SERPs を支配している（2026-06 時点の観測）
- 個人開発ツール **SUGUDASU** は読みが同じだが別サービス — 正面から単体キーワードを取りに行かない判断
- **Query Desambiguation（検索意図の分離）** の観点: 複合語（`スグダス 請求書` `スグダス 登録不要`）と構造化データで「実務ツール」意図を育てる
- 実装した対策（事実のみ · 誇張なし）:
  - `hub.html` に `WebApplication` schema · 迷子向け `not-a-car.html`
  - invoice / receipt に業種向け FAQ（中古車販売・整備の帳票文脈）
  - PageSpeed 改善（Tailwind 事前ビルド · フォント preconnect）— 技術優位は補助要因
- **やらなかったこと:** スバル在庫投稿へのリプ寄生 · 車系バイト記事 · 商標ロゴの模倣
- コピー軸: **「走るスグダスじゃなく、書くスグダス」** — X でも複合語セットで発信

**技術用語（補足）**

- **Query Desambiguation** — 同一語に複数の強い意図があるとき、検索エンジンがカテゴリを分けて結果を混ぜる挙動
- **構造化データ（JSON-LD）** — `WebApplication` / `FAQPage` でサービス種別を機械可読に

**スクショ候補:** X「スグダス」検索（車写真と UI スクショの混在 · 個人情報マスク）· `not-a-car.html` · hub バナー

**避ける:** SUBARU 批判 · 「SEO ハックで1位奪取」等の断定 · スバル公式への @

**X 連動:** 本節公開週に `homonym_intro` テンプレ投稿（`docs/x_guideline.md` §11）
