### 本文

見積で固めた明細を、請求書でまた一から打ち直していませんか。私は月に数枚だけ書類を出す立場で、クラウドの会員登録や月額だけのために手間を増やしたくないと思い、**登録不要・クライアントサイドのみ**で動く見積・納品・請求書ツール [SUGUDASU](https://sugudasu.com/invoice?utm_source=zenn&utm_medium=social&utm_campaign=article_01_invoice_convert) を個人開発しました。配信は静的ホスティング（Cloudflare Pages）で、入力処理はブラウザの JavaScript が担います。

この記事では、いちばん使う場面である **「見積 → 請求」** の流れに加え、下書きが **JSON ファイル** としてどう保存されるかも触れます。エンジニア向けの実装メモと、事務担当者向けの手順を同じ記事に載せます。

## アカウント不要で、書類作成を始められる理由

SUGUDASU にはログイン画面がありません。URL を開くと、そのまま見積書・納品書・請求書の入力フォームが表示されます。インボイス制度で必要になりがちな **適格請求書発行事業者の登録番号** や、10%・8%混在の明細も、フォーム上で入力してプレビューに反映できます。

![](https://static.zenn.studio/user-upload/6f52e616cccf-20260618.png)
*入力フォームとプレビュー（見積書）*

ここで言う「アカウント不要」とは、**当方のサーバーに利用者アカウントを作らない** という意味です。書類の下書きを SaaS の DB に預ける設計とは異なります。月に数件しか請求しない人ほど、「とりあえず登録」が負担になりやすいと私は考えています。

なお、最終的な PDF 保存や印刷はブラウザの **印刷ダイアログ**（`window.print()` 相当）を使います。画面上でも PC での確認を推奨しており、スマートフォンは入力や下書きの確認程度を想定しています。

## 下書きは JSON ファイル — 中身と保存の仕組み

取引先に送るのは **PDF** です。一方、編集を続けるための控えが **下書きデータ（`.json`）** です。この2つは別物として扱ってください。

「下書きデータの保存」を押すと、フォームのスナップショットを `JSON.stringify` したテキストが `application/json` の **Blob** としてダウンロードされます。ファイル名は `【見積請求】YYYYMMDD_番号または件名.json` のような形式です。読み込み時は `FileReader` でテキスト化し、`JSON.parse` したあと `format: "sugudasu.invoice"` と `version: 1` を検証してからフォームへ復元します。別ツールの JSON を誤って読み込まないためのガードです。

スキーマのイメージは次のとおりです（ロゴ・印鑑は Base64 の Data URL として `images` に入ります）。

```json
{
  "format": "sugudasu.invoice",
  "version": 1,
  "exportedAt": "2026-06-17T12:00:00.000Z",
  "docType": "estimate",
  "clientName": "株式会社サンプル",
  "items": [
    { "name": "Web制作", "qty": 1, "price": 100000, "taxRate": 10 }
  ]
}
```

![](https://static.zenn.studio/user-upload/d117772c80fd-20260618.png)
*下書きデータの保存・読み込み欄*

`docType` は見積・納品・請求のどれで保存したかを表します。社内フォルダや別 PC へコピーすれば、翌日そのまま続きから再開できます。会計ソフトへそのまま取り込む形式ではありません。

### 補足: JSON とは？

**JSON（ジェイソン）** は、データを「名前: 値」の並びでテキストファイルに書き出すための共通フォーマットです。Excel の表をそのまま保存するのではなく、**プログラムが読み書きしやすいメモ帳** だと思ってください。拡張子 `.json` のファイルをメモ帳で開くと `{` と `"` が並んで見えますが、**SUGUDASU ではツールの「読み込み」ボタンから選ぶだけ** で使えます。中身を手編集する必要はありません。

入力した顧客名・金額・ロゴ・印鑑画像などは、**原則としてブラウザ内で処理** し、当サイトのサーバーへ POST する API はありません（詳細は [プライバシーポリシー](https://sugudasu.com/privacy)）。社内の情報管理ルールに合わせて使ってください。

## 書類種別タブを切り替えて転用する

転用の手順は次のとおりです。

1. 見積書として明細・宛先・金額を入力する  
2. 「下書きデータの保存」で `.json` をダウンロード（任意だが、転用時は便利）  
3. あとから「下書きデータを読み込む」で復元する  
4. 画面上部の **書類種別タブ**（見積書・納品書・請求書）を切り替える — 内部では `docType` が `estimate` / `delivery` / `invoice` に変わります  

![](https://static.zenn.studio/user-upload/7e780a7d3201-20260618.png)
*書類種別タブの切り替え（見積 → 請求）*

タブを変えても、**明細配列 `items`・宛先・金額は維持** されます。変わるのは書類の体裁（タイトル、期限ラベル、備考の初期文など）です。

![](https://static.zenn.studio/user-upload/64572ae07507-20260618.png)
*請求書に切り替えたあとのプレビュー*

ここで一度止まってください。見積番号のまま請求書を出さないように、**請求番号・支払期限・振込先などの備考** は書類に合わせて直してから PDF を保存します。納品書を挟む運用なら、納品書タブを経由してから請求書に切り替えても同じです。

値引き行のように単価が負の明細も入力できます。税率区分ごとに集計し、消費税は切り捨てで計算します。適格請求書としての要件を満たすかどうかは、取引内容ごとにご確認ください（[免責事項](https://sugudasu.com/disclaimer)）。

## まとめ

見積で作った内容を、コピペで請求書に写す代わりに、**JSON 下書きの読み込みとタブ切替** で体裁だけ変える。サーバーにアカウントを作らず、スナップショットを手元のファイルとして持てる — それが SUGUDASU invoice の実務フローです。

同じポータルには割り勘やシフト表などもありますが、まずは一件、見積から請求まで試してみてください。困った点や要望は [更新履歴ページ](https://sugudasu.com/updates) のフォームから送ってもらえると、changelog に反映して改善しています。
