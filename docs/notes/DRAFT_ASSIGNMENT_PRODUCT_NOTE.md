# 希望順位割当（ドラフト会議型）— プロダクトメモ

**更新**: 2026-07-09（§3-3 マーケ棚割 · §9 参考Repo）  
**ステータス**: 整理中（実装未着手）  
**起源**: ルート `Draft会議.md` · `draft_meeting.md`（新卒配属ドラフト会議の実体験からのブレスト）

> **別Agentへ:** 抽選（`fair-draw`）との混同に注意。本テーマは **希望順位 × 定員枠 × 合意形成** で、ランダム抽選とは別ジョブ。

---

## 1. 一言

**第1〜3希望を集め、限られた枠へできるだけ納得感のある割当を作り、会議で手直しするためのシミュレーションボード。**

- 自動計算は「たたき台」。最終決定は人間が画面共有・プロジェクター前で調整する前提（`draft_meeting.md`）。
- データは機密（配属希望 · 社内評価 · 顧客本音）。**非送信・インメモリ**が差別化の核。

## 3. 想定ユースケース（利用シーンは多い）

ブレスト・実務感から、**新卒配属だけでなく横展開余地が大きい**。

### 3-1. 人事 · 教育（高頻度）

| シーン | 誰 × 誰 | なぜ第3希望まで | 機密性 |
|--------|---------|-----------------|--------|
| **新卒一括配属** | 新卒 × 部署（定員上限） | 人気部署の取り合い · 第2・3希望で救済 | 配属希望 · 評価 · 人間関係 |
| **ジョブローテーション** | 社員 × 次期ポスト | 複数候補 · 定員 · スキル枠 | 同上 |
| **ゼミ配属** | 学生 × 研究室・ゼミ | 希望研究室の集中 · 定員オーバー | 成績・研究テーマ希望 |
| **大学・大学院 配属・履修枠** | 学生 × コース/クラス | 履修定員 · 時間割衝突の緩和 | 学籍・希望理由 |
| **研修・ワークショップ分科会** | 参加者 × セッション枠 | 人気枠の平準化 | 関心領域（キャリア本音） |
| **ハッカソン・アイディアソン班** | 参加者 × チーム | 職能・属性の制約（既存 group-split と接続） | 名簿・属性 |

### 3-3. マーケ · 店舗 · 総務（ニッチだが需要あり）

| シーン | 誰 × 誰 | 備考 |

| 配送ルート・エリア割当 | 車両 × エリア枠 | 住所など個人情報 |

#### 3-3-1. マーケ棚割シミュレーション（提督案 · 2026-07-09）

**Pain:** キャンペーンや店頭で、製品・プレゼントを横一列（またはグリッド）に並べて配分を決めるとき、**人気SKUはすぐ枯れ、不人気SKUは最後まで残る**のが常態。事前に「誰に何を割り当てるか」を幹事会で調整したいが、Excelだと視覚的な合意形成が弱い。

| UIイメージ | 内容 |
|------------|------|
| **横軸** | 製品・景品カード（SKU名 · 在庫数 · 人気タグ） |
| **縦軸 / プール** | 顧客セグメント · 会員ランク · 抽選当選者 |
| **フィルター** | **人気順 / 不人気順 / 在庫僅少 / 未配分のみ** — 枯れそうな枠を赤、余りそうな枠をグレーで可視化 |
| **操作** | カードをスロットへ D&D · 定員（在庫上限）超過を即警告 |
| **出力** | 配分表 TSV · JSON（案A/B）· 稟議用サマリ |

→ 人事の「部署スロット」と同型。**供給（SKU枠）× 需要（希望順位）× 人間の手直し** の構造は共通。`draft_meeting.md` §5 ドメイン2（マーケ・販促）の具体化。

## 4. 既存ツールとの関係

```
fair-draw    … ランダム抽選 · 景表法証跡（当選 N 人）
group-split  … 均等班分け · 制約シャッフル（初期案生成）
match-board  … 希望順位割当 · 定員スロット · 手動調整（本メモの対象）
```

| 機能 | fair-draw | group-split 現行 | 本テーマ（match-board） |
|------|-----------|------------------|----------------------|
| ランダム性 | ◎ | ○（シード） | △（初期案の自動マッチのみ） |
| 希望順位 1〜3 | × | × | ◎ |
| 定員 per 枠 | △（当選人数） | ○（班サイズ） | ◎（部署・ゼミ・セッション単位） |
| 手動D&D調整 | × | △ | ◎（コアUX） |
| JSON 持ち出し | ○（証跡） | ○ | **必須級** |

## 9. 参考 Repo · 技術スタック（ChatGPT 調査 · 2026-07-09）

> **注意:** 以下は実装調査メモ。**採用は未FIX。** SUGUDASU コアは静的HTML/JS（React 前提ではない）のため、**思想・パターンの借用**が主。専用「ドラフト会議Repo」は少なく、**カードD&D + 順番 + Room** の組み合わせで足りる、という整理。

### 9-1. ドラフト会議型 UI と Room 構想の相性

本プロダクトはホワイトボードではなく **「カードを空間配置して合意形成するUI」**。SUGUDASU Room（有期限・共同編集）と相性が良い。

```
未指名プール  →  スロット列（部署/SKU/セッション）  →  指名済み
     Player              Team / Slot                    Assigned
```

### 9-2. 参考リポジトリ（優先度付き）

| 優先 | Repo | URL | 借りるもの |
|------|------|-----|------------|
| ★★★★★ | **tldraw** | https://github.com/tldraw/tldraw | Shape モデル · カードD&D · 選択状態 · グループ · **Room概念** · 複数人編集 |
| ★★★★ | **Excalidraw** | https://github.com/excalidraw/excalidraw | URL共有 · Room参加 · 軽量同期 · シンプルUI |
| ★★★★ | **Atlassian Pragmatic Drag and Drop** | https://github.com/atlassian/pragmatic-drag-and-drop | 実務向きD&D · a11y · **プール→列→確定** の3段が相性良い（dnd-kit より実務寄りとの評価） |
| ★★★ | **dnd-kit** | https://github.com/clauderic/dnd-kit | React 定番 · ドラフト会議には十分（本リポは React 非前提） |
| ★★ | **React Flow (xyflow)** | https://github.com/xyflow/xyflow | マインドマップ向け · 配属後のフォーメーション可視化の派生参考 |

**スポーツ戦術ボード系**（Soccer/Basketball tactical board）は ZoneBoard 側の参考。本テーマは選手アイコン・ラインアップ配置の **UXパターン** のみ参照（正本: `C:\asl_dev\zoneboard`）。

### 9-3. Cursor 向け推奨スタック案（Sync / Room フェーズ）

コア（`match-board`）を **静的HTML/JS** で先に ship し、Room は別フェーズで以下を検討：

| 層 | 候補 | 役割 |
|----|------|------|
| UI | tldraw · Pragmatic Drag and Drop | カード配置 · スロットD&D |
| 同期 | **Yjs** · y-webrtc | CRDT · オフライン耐性 |
| 描画 | Konva.js · React Konva | 大量カード時のレイヤー分離（`draft_meeting.md` §4 と整合） |
| 暗号 | WebCrypto | Sync ライン既存方針（`# SUGUDASU Sync — 暗号アーキテクチャ仕様書 v3.md`） |
| 通信 | WebRTC DataChannel | P2P 優先 |
| シグナリング | Supabase Realtime | 既存 Sync インフラ |

**深掘り1本なら tldraw** — Shape · 選択 · 共同編集 · Room 管理が「配置して議論する」基盤として最も近い。

### 9-4. SUGUDASU 実装方針との接続

| フェーズ | 方針 |
|----------|------|
| **Phase 1（コア）** | `match-board`（独立）· 素の HTML5 D&D または軽量ライブラリ · **非送信 · JSON** |
| **Phase 2（Sync）** | Room · 案A/B · 役員閲覧リンク — Yjs + WebRTC を **オプトイン** |
| **見送り** | ドラフト専用Repo探し · Fantasy Draft 系UIの丸コピー |
