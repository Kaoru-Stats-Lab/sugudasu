## 2. 源泉徴収 10.21%（P1-4）

### 2-1. UI

| 要素 | 仕様 |
|------|------|
| チェック | 「源泉徴収（10.21%）を差し引く」（デフォルト ON · 法人宛は OFF） |
| 税抜/税込 | ラジオ — **税抜本体に対象**（デフォルト）/ **税込総額に対象** |
| 端数処理 | セレクト — **切捨て**（デフォルト）/ 四捨五入 / 切上げ |
| 累進 | 100万円超の **20.42%** 自動適用（トグル or 常時 — 実装時に FAQ 明記） |
| 逆算 | 同一画面に「手取りから逆算」タブ（余力） |

### 2-2. 計算（一般例）

**税抜本体に対象（原則）**

```
本体 = 入力（税抜）
消費税 = floor(本体 × 10%)  ※端数ルールは消費税側も切捨てデフォルト
源泉税 = floor(本体 × 10.21%)  ※1円未満切捨てデフォルト
請求総額 = 本体 + 消費税
差引支払 = 請求総額 - 源泉税
```

**100万円超（累進 · 要 FAQ）**

- 100万円以下の部分: 10.21%
- 超える部分: 20.42%
- 実装前に [国税庁 No.2792](https://www.nta.go.jp/taxes/shiraberu/taxanswer/gensen/2792_qa.htm) と突合 · 単体テスト必須

### 2-3. 入力 UX

- 半角数字 · カンマ自動挿入
- 文字混入時は数字抽出して計算継続（ユーザーを止めない）
- 上限目安: **99,999,999円** / 明細 **10行** 程度

### 2-4. 免責（F7 · 採用文案）

> ※本ツールで提供する計算結果は、一般的な税率（10.21%）に基づく計算例です。実際の源泉徴収義務の有無や正確な税額、端数処理の取り扱いについては、契約内容や職種により異なる場合があります。実務でのご利用にあたっては、必ず国税庁の公式情報をご確認いただくか、税理士等の専門家へご相談ください。

## 3. 内税 8% / 10% 混在分離（P1-5）

### 3-1. UI

| 要素 | 仕様 |
|------|------|
| 入力 | `10%対象（税込）` · `8%対象（税込）` フィールド |
| 端数 | 税率ごとに **1回だけ** 端数処理（デフォルト切捨て） |
| 比較 | 検算用「行ごと端数」比較トグル（折りたたみ · デフォルト OFF） |
| 出力 | インボイス記載想定グリッド + ワンクリックコピー |

### 3-2. 計算（一般例）

各税率ブロックについて:

```
税込総額 = 入力
本体   = floor(税込総額 / (1 + 税率))   ※端数処理はここで1回
消費税 = 税込総額 - 本体
```

**禁止:** 明細行ごとに端数してから合計（インボイス制度と不整合 · F7）

### 3-3. 表示項目チェックリスト（FAQ · 断定弱め）

- [ ] 適格請求書発行事業者の氏名・登録番号（T+13桁）
- [ ] 取引年月日 · 取引内容（軽減税率対象の旨）
- [ ] **税率ごとの対価の額と適用税率**
- [ ] **税率ごとの消費税額**（端数は請求書あたり税率ごとに1回）
- [ ] 交付を受ける事業者の氏名

### 3-4. やらないこと

- レシート OCR（サーバー不要維持）
- 外食/持ち帰りの 8%/10% 自動判定（入力はユーザー任せ）

### 3-5. 免責

> 本ツールは税務申告の正確性を保証するものではありません。一般的な計算例としてご利用ください。
