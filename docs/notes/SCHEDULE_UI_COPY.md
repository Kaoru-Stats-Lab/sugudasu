## 1. 結論（粒度の一行）

**現場の言葉はそのまま · IT と SaaS の言葉だけ平易に · 画面に出すのは「Excel の列見出し＋印刷メニュー」レベルまで。**

| 平易にする | 平易にしすぎない |
|------------|------------------|
| preset · 依存エンジン · visibility · Sync · tier | 立会 · 工区 · 占用 · 協力会社 · 段階確認 |
| 「後の工程も日付をずらす」 | 「お仕事をつなげる」（幼い） |
| 「監理に渡す用」 | 「提出用プリセット」（内部名） |

---

## 2. 四段階モデル（どこまで平易か）

画面・文案は **露出頻度** で段を分ける。下ほど毎日見る · 上ほど一度きり or 開発者のみ。

```
D0  コード・SSOT・Agent     submit · prop_machine · maintain_gap — ユーザーに出さない
D1  設定・上級（v1 最小）   「休日を除いて数える」— 年1回触る想定
D2  毎日の操作               提出 / 現場 · 提出用PDF · 工種名 · 開始日
D3  初回のみ・空状態         「ここに工種を追加」· 3ステップ導線
```

### 2-1. 各段のルール

| 段 | 対象 | 語彙レベル | 文の長さ |
|----|------|------------|----------|
| **D3** | 空状態 · 初回オンボ | **いちばん平易** · 敬体 · 専門語なし | 1文 20字前後 |
| **D2** | ツールバー · 表頭 · ボタン · ステータス | **Excel 列見出し** · 現場で通じる漢語 | ラベル **2〜8字** 目安 · ボタンは **動詞** |
| **D1** | チェック · 詳細設定 · 確認ダイアログ | 平易 + **括弧で現場語** 可 | 1行 40字以内 · 危険操作は理由を1句 |
| **D0** | コード · ログ · 仕様書 | 技術語 OK | — |

**v1 原則:** **D1 を増やさない**（設定画面を作らない · §0-5）。迷う機能は D2 の `title` ツールチップ 1行まで。

## 3. 置き換え辞書（NG → OK）

### 3-1. IT / 製品内部語（必ず隠す）

| 出してはいけない（D0） | 画面（D2） | ツールチップ（D1） |
|------------------------|------------|-------------------|
| preset `submit` | **監理に渡す** or **提出用** | 発注者・監理者に出す月間工程表の見え方 |
| preset `site` | **現場用** | 重機・資材など社内で使う見え方 |
| `weekly_sub` | **下請に送る週間表** | 協力会社ごとに抜き出して PDF |
| dependenciesEnabled | **後の工程も日付をずらす** | 先行の終了を変えたとき、続きの工期を保ったまま動かす |
| maintain_gap | （表示しない） | 同上 |
| visibility · tier · ops | （表示しない） | — |
| payload · revision · Sync | （表示しない） | 保存しました / 更新しました |
| 依存 N本 | **つながり N件** または非表示 | 工種の前後関係（画面では矢印なし） |
| プロパティ | **項目** or **列** | — |
| サブアイテム | **内訳行** | 工区のまとめの下の行 |
| ガント · タイムライン | **工程表** · **日程** | 右の横棒は期間の目安 |
| Export | **PDFで出す** · **Excelで出す** | — |

### 3-2. 現場語（そのまま使う · 平易化しない）

監督は **土木建築の業務語** は知っている。ここを「わかりやすく」しすぎると **素人向け** に見えて信頼を損なう。

| そのまま OK（D2 列名） | 補足 |
|------------------------|------|
| 工区 · 測点 | 長い場合は **工区** 列 + ツールチップで測点 |
| 協力会社 | 「下請」併記は任意 |
| 立会 · 段階確認 | 現場の呼び方を優先 |
| 道路占用 | チェック列なら **道路規制あり** も可 |
| 重機 · 資材 | PoC の `投入予定重機・車両` → **重機・車両** で足りる |
| 人工 | 社内列 · 提出に出さない時は列ごと非表示 |

### 3-3. PoC からの改修候補（ゲート A）

| 現状（PoC） | v1 案（D2） |
|-------------|-------------|
| 依存連動 | **後の工程も日付をずらす**（チェック） |
| 依存 3本 | 非表示 or **つながり 3件** |
| 施工工区・位置（測点） | **工区** |
| 配置予定人工数 | **人工**（site のみ表示） |
| 主要資材・予定数量 | **資材** |
| 段階確認・立会種別 | **立会** |
| 道路規制・占用フラグ | **道路規制** |
| 直前変更・遅延理由 | **遅れ・変更理由** |
| 届出区分 | **届出**（site のみ） |
| preset=submit 固定で出力（title） | **監理提出用の PDF** |

## 4. 文の型（テンプレ）

| 用途 | 型 | 例 |
|------|-----|-----|
| **主ボタン** | 動詞 + 対象 | 提出用 PDF · Excel で保存 |
| **トグル** | 名詞 or 短い動詞句 | 現場用 / 監理に渡す |
| **チェック** | オンで何が起きるか | 後の工程も日付をずらす |
| **保存** | 結果だけ | 保存しました |
| **エラー** | 何をすればいいか | 終了日を開始日より後にしてください |
| **空状態** | 次の1アクション | ＋ を押して工種を追加 |

**禁止:** 英語 UI 語のカタカナ直訳（プリセット · エクスポート · デフォルト · オプトイン）· 三行説明 · 「ご利用ありがとうございます」

## 5. 四層モデル（機能）との対応

[`SCHEDULE_GENERIC_VS_VERTICAL.md`](SCHEDULE_GENERIC_VS_VERTICAL.md) との関係:

| 機能層 | 文案 |
|--------|------|
| **L0** | D2 ボタン名は **業界共通**（開始日 · 終了日 · 備考 · PDF） |
| **L1** | 「監理に渡す / 現場用」— **パックごとに表示名だけ差し替え可** |
| **L2** | 列名 = **建設の Excel 見出し**（重機 · 立会） |
| **L3** | LP・SEO のみ別文言 · **アプリ内は L2 で足りる** |

## 6. 検証（ゲート A · dogfood）

提督・監督に見せるときの **3問**（リサーチ不要 · 5分で可）:

1. **このボタンを押すと何が起きるか** 言えるか？（D2）
2. **列名が Excel と違って困る** ものはあるか？（D2）
3. **説明を読まないと怖い** トグルはあるか？（D1 に降ろす or 文言修正）

**合格:** 3問すべて「Excel でやってたことと同じ言葉」で答えられる。
