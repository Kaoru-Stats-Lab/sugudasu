## §1. 調査ログ（アーカイブ · 2026-06-19 時点）

> 以下は Cursor セッションで RevenueCat 2024–2026 を横断整理した**ログそのまま**。改変・要約禁止の参照用。転用判断は §2 を正とする。

RevenueCat [State of Subscription Apps](https://www.revenuecat.com/state-of-subscription-apps-2024/)（2024→2025→2026）を横断すると、**「平均的に維持」は消え、上位が加速・下位が失血する二極化**の中で、次の7本が勝ち筋として一貫しています。

### 一言でいう勝ち筋

> **初回セッションで価値を証明し、課金設計で早くキャッシュを回し、Week 1 から更新を取りに行き、技術的な revenue leak を塞ぐ。AIは集客に使い、継続は習慣とPMFで勝つ。**

### 1. 二極化に乗るか、滑るか（2024→2026で加速）

| 年 | シグナル |
|----|----------|
| **2024** | ローンチ1年後、上位5%の月収は下位四分位の **200倍**超。中央値は月 **$50未満**。[TechCrunch要約](https://techcrunch.com/2024/03/12/most-subscription-mobile-apps-dont-make-money-new-report-shows/) |
| **2025** | その格差は **400倍**に拡大（2024の200倍から悪化）。[2026サマリー](https://www.revenuecat.com/blog/growth/subscription-app-trends-benchmarks-2026/) |
| **2026** | 成長率でも二極化：上位25%はMRR **+80% YoY**、下位25%は **-33%**（113pt差）。中央の5–17%成長は「安全圏」ではなく**下位落ちリスク** |

**勝ち筋:** メンテモード禁止。**オンボーディング・ペイウォール・更新・課金インフラ**をカテゴリベンチマークで継続A/B。月$1,000→$2,500→$5,000は「到達後に次段階に進みやすい」階段（2024）。

### 2. 初回セッション＝勝負（全版共通・2026でより極端）

| 指標 | 数字 |
|------|------|
| トライアル開始 | **80%超がDay 0**（2025: 82%、2026も同趣旨） |
| 3日トライアル解約 | **55%がDay 0**、**84%がDay 0–1**（2026） |
| 年額解約 | **35%が初月**（2026）。Year1解約率は2025の56%→2026 **72%** |

**勝ち筋:** 「3日トライアル」は実質 **1時間トライアル**。オンボーディング＝**最大のリテンション機能**。コア価値の *aha!* を**最初の60分**に入れる。年額は「12ヶ月ある」と思わず、**Week 1–8で更新維持・win-back**。

### 3. ペイウォール設計：ハードが速い、実行が全て

| モデル | D35 課金転換（中央値） | 備考 |
|--------|------------------------|------|
| **Hard paywall** | **10.7%**（2026）/ **12.1%**（2025） | Freemiumの **約5倍** |
| **Freemium** | **2.1%**（2026）/ **2.2%**（2025） | Week 6以降の遅延転換は残る |
| **Day 60 RPI** | Hard **$3.09** vs Freemium **$0.38**（8倍・2026） | |
| **1年後年額維持** | Hard 27% vs Freemium 28% | **ほぼ差なし** |

**勝ち筋:**

- **キャッシュフロー・CAC回収**優先 → Hard + 初回で価値が伝わるコピー/体験
- **長期ファネル** → Freemium + 機能ゲート + タイミング良いアップセル（上位FreemiumはDay 0転換も強い＝**モデルより実行**）
- 2025: トライアル付きペイウォールはA/BでVisual-onlyより勝ちやすい（Adapty引用の二次分析あり）
- Hardは**返金率も高い**（約70%高）→ 過剰期待を生むハードは逆効果

### 4. トライアル長さのパラドックス（データ vs 市場）

| 期間 | トライアル→課金（中央値） |
|------|---------------------------|
| **17–32日** | **42.5%**（2026）/ **45.7%**（2025） |
| **4日未満** | **25.5%**（2026）/ **26.8%**前後（2025） |
| 差 | **約70%高い** |

一方、**4日以下トライアルを採用するアプリは46.5%**（2026）と増加。理由は**3日で課金・実験速度・ダッシュボードの即効性**。

**勝ち筋:** 群衆に盲従しない。**カテゴリ別**に長トライアル実験。短トライアルを使うなら **Day 0価値証明**で補完。2024も「長いほど転換良いが、開発者はCAC短縮で短くする」と同型。

### 5. 年額は「売れた」≠「残る」

- 初月に年額解約の **35%**（2026）
- ユーザーは年額を**1年分の買い切り**と見て、すぐ自動更新OFF
- 解約曲線：初月スパイク → 年中3–10% → **12ヶ月目再スパイク**

**勝ち筋:** Year 2はYear 1の**11ヶ月目**ではなく**1週目**から取りに行く。習慣化・価値再提示・初月win-back。

### 6. Androidは「成長」より「課金エンジニアリング」（2025→2026悪化）

| プラットフォーム | 解約のうち課金失敗 |
|------------------|-------------------|
| **Google Play** | **31%**（2026）/ 28.2%（2025） |
| **App Store** | **14%**（2026）/ 15.1%（2025） |

**勝ち筋:** Android比率が高いなら **dunning・リトライ・grace period** が広告よりROI高い（**15–20%回収**可能、2026サマリー）。2024から継続テーマ。

### 7. AI：集客は強い、継続は弱い（2025→2026で顕在化）

| | AIアプリ | 非AI |
|--|---------|------|
| Year1 LTV | **+41%**（$30.16 vs $21.37） | 基準 |
| 12ヶ月維持 | **36%悪い** | 基準 |

2025時点ではAIの**転換率自体は高くない**が**課金単価は高い**。2026は「試したが定着しない」が表面化。

**勝ち筋:** AIラッピング alone は3ヶ月で死ぬ。**PMF・習慣・バックエンド**にAI収益を再投資。速く作るより**維持と信頼**（2026: *stamina* が差）。

### 2024→2026で変わったこと / 変わらないこと

| 変わらない（勝ち筋の芯） | 2024→2026で強まった |
|-------------------------|---------------------|
| 初回体験・ペイウォールタイミング | 格差（200x→400x、成長率113pt） |
| 長トライアルの転換優位 | Day 0解約の増加（51%→55%） |
| 年額初月解約 | 年額Year1解約悪化（56%→72%） |
| 上位は実行で抜ける | AIの「売れるが残らない」 |
| | 月1.4万アプリ新規（2026）でレッドオーシャン化 |

2024の予測（**サブ単体からハイブリッド課金**、価格上昇、Web/アフィ/広告併用）も2025–2026の「pure subだけでは厳しい」と整合。

### 実務チェックリスト（サブスクアプリ向け · ログ原文）

```
1. インストール直後にペイウォール（80%がDay0に決める）
2. 60分以内に aha!（機能削ぎ・最短ルート）
3. モデル選択：Hard=速いキャッシュ / Freemium=遅延転換（どちらも初回品質が命）
4. トライアル長：カテゴリで17–32日 vs 3日をA/B（群衆は短い方）
5. 年額購入者：Week1から更新維持キャンペーン
6. Android：課金失敗リカバリを最優先インフラ
7. AI：差別化ユースケース + 習慣化。 hype だけで出さない
8. 「good enough」成長（5–17%）を警戒 → 四半期ごとにベンチマーク比較
```

## §2. SUGUDASU 転用 SSOT（正本）

### 2-0. 前提：何を転用し、何を転用しないか

| RevenueCat の世界 | SUGUDASU の世界 |
|-------------------|-----------------|
| App Store / Play サブスク | **Web · 登録不要 · 静的 · ローカル完結** |
| Hard paywall / トライアル | **フリクションゼロ即利用**（憲法 F1–F3） |
| MRR / 更新 / dunning | **AdSense · アフィ · 再訪 · ツール横断**（§2-5 逆算式） |
| 115k アプリ競合 | **ニッチ実務ツール · SEO · Zenn · X** |

**転用するのは「メカニズム」**（初回価値 · 二極化 · 習慣 · 信頼 · ハイブリッド収益）。**転用しないのは「ストア課金そのもの」**（サブスクアプリ化は本 SSOT のスコープ外 · 別プロダクト）。

### 2-1. 抽象原則 → SUGUDASU 翻訳表

| SOSA 原則（§1） | SUGUDASU での意味 | 具体アクション |
|-----------------|-------------------|----------------|
| **二極化（200x→400x）** | ツール海で「平均的」は埋もれる | **1ツール1尖り**（M2差別化1文）。Tier S（invoice / fair-draw）に集中。メンテだけの γ ツール量産禁止 |
| **Day 0 = 80%** | **初回セッション = 唯一のコンバージョン窓** | URL開いて **3秒で入力可能** · **1操作で結果** · 説明よりフォーム優先（DESIGN_GUIDELINE） |
| **60分 aha!** | **1セッション完結**（幹事は待たない） | 結果画面に **PDF / コピー / 次ツール導線**。fair-draw の監査PDF = 「説明の aha!」 |
| **Hard 5x vs Freemium** | SUGUDASU = **逆Hard**（登録・送信なし） | **現状** AdSense・アフィ。**将来 Pro**（広告非表示 · 証跡 PDF 拡張 · 高度機能）可 — **コア実務の登録ウォール** は F1 違反 |
| **Freemium 遅延転換** | **ハブ回遊 · ブックマーク · 社内Wiki** | 1本目で完結 → **hub / 関連ツール / Zenn** で2本目。`data/cta.json` 横断 |
| **長トライアル > 短** | **再訪の理由 = 習慣ではなく「都度の実務」** | 月次更新（changelog）· 季節記事 · **同じPainの別角度**（例: 見積→請求）。サブスク期待しない |
| **年額初月解約** | **「使い切り」プロダクト** | 1回利用で価値完了 = 正常。**再訪は次のPain**。無理なプッシュ通知・メールマガ禁止（F2） |
| **Android 課金 leak** | **該当薄い**（Web） | 転用先 = **PageSpeed · 404 · 広告配置ミス** 等の **技術 leak**（§13 PSI）。計測は GA4 のみ |
| **AI: 売れるが残らない** | **Gemini 直叩き禁止 · 叙述は Colab/バッチ** | ツールに AI チャットを載せない。**確定出力**（PDF/JSON）が信頼。AI は **開発速度** にのみ使う |
| **ハイブリッド課金（2024予測）** | **既に設計済み** | AdSense（C）+ Amazon present（M7）+ SEO（A）+ Zenn（A）。pure 広告のみにしない |

### 2-2. SUGUDASU の「ペイウォール」= 初回摩擦設計

RevenueCat の Hard paywall に**概念的に最も近い**のは、SUGUDASU の **「最初の3秒で使えるか」**。

| 悪い Hard（サブスクアプリ） | SUGUDASU でやるべき逆設計 |
|----------------------------|---------------------------|
| 登録してから使える | **即入力** |
| 説明動画 60秒 | **1行ヒント + placeholder** |
| 機能一覧を見せる | **Step 0 を1択に圧縮**（fair-draw） |
| 期待過多 → 返金 | **免責・黄旗で期待調整**（F7） |

**KPI 翻訳:** `Download→Trial` の代わりに **`Landing→First Input`**（フォームフォーカス）と **`Session→Result Shown`**（結果 or PDF）。

### 2-3. 「更新」= 再訪・横断・信頼の積み上げ

サブスクの「Week 1 更新維持」に相当する SUGUDASU 施策:

| 施策 | 根拠（SOSA） | SUGUDASU 実装 |
|------|-------------|---------------|
| **初回に証跡を渡す** | Day 0 解約防止 = 価値証明 | 監査PDF · コピー文（fair-draw P1-4）· warikan シェア文 |
| **2本目への導線** | Freemium 遅延転換 | hub · ツール間 CTA · 「次は invoice」 |
| **changelog 可視** | stamina / 信頼 | `updates.html` · ツール version badge |
| **被リンク・Zenn** | 上位10%の実行差 | §14 Tier 1 · 実装エッセイ |
| **GTM 手動フォロー** | ニッチ幹事への直接リーチ | §15-5 懸賞天国 SNS（スクレイピング禁止） |

### 2-4. 二極化下の優先度ルール（BACKLOG 判断用）

```
IF ツールが「中央値」帯（使えるが差別化1文が弱い）
  → HOLD または 尖り追加（PDF/FAQ/ニッチ条文）まで実装凍結

IF 初回セッションで結果まで > 3 クリック
  → UX 改修を機能追加より優先

IF AI 機能を「話題性だけ」で提案
  → PRODUCT_IDEA_JUDGMENT_LEDGER で OUT（SOSA §1-7）

IF 収益が AdSense のみ
  → present / invoice 等 M7 整合ツールへ導線を必須化
```

### 2-5. ツール別転用メモ（2026-06-19）

### 2-6. 計測（サブスク KPI → Web KPI）

| SOSA | SUGUDASU（GA4 · 静的） |
|------|------------------------|
