### リード（改訂版 · 1段落 · strong タグ位置は維持可）
「Excelの工程表で十分では？」——企画を通す段階や提出用資料としては確かに強いです。ただ、**クライアント確認が1日遅れただけで後続工程を手動で全部ずらして、気づいたら_vFinal_と_v3_最終_が3つ並んでる**状況は、WEBディレクターあるあるだと思います。制作スケジュール表と案件進行管理シートを回すときに起きやすい版ズレや手動リスケの痛みを、実務目線でまとめました。

### H2「なぜ〜」— 改訂段落（p のみ · 3段落まで）
制作現場ではスケジュールが生き物です。  
クライアントからの戻りが遅れたり、デザイン案の修正が想定以上にかかったりすると、Excelの行を一つずつずらす作業が発生します。  
これを繰り返していると、最新版がどれかわからなくなったり、関係者への共有が遅れたりするんですよね。特にクリエイティブ進行表とスケジュール逆算表を両方持つ案件で顕在化しやすい問題です。

### H2「制作進行でよくある失敗」— 各H3名 → 改訂2段落

**H3: 版管理の混乱**  
「_v3_最終.xlsx が3つある…」  
最新版がどれかわからなくなり、関係者に古いファイルを送ってしまう。原因はファイル名ルールが曖昧で、更新履歴が残らないExcelの特性にあります。

**H3: 手動リスケのミス**  
「1日遅れただけで全部手で直して、結局ダブルブッキングした」  
後続タスクの連動が取れていないと、1箇所の変更が全体に波及してミスが増えます。特に締め切りが近いフェーズで起きやすいです。

**H3: 共有タイミングのずれ**  
「昨日送ったはずの進捗が、クライアントには反映されてない」  
関係者ごとに別ファイルや別シートを使っていると、更新漏れが起こります。

**H3: 逆算表と進行表の乖離**  
「逆算では納品日が合ってるのに、実際の進行表では1週間ズレてる」  
2つの表を別管理すると同期が取れなくなり、全体像が見えにくくなります。

**H3: 担当者不在時の引き継ぎ失敗**  
「急に休まれたときに、どこまで進んでるかわからなかった」  
個人依存の管理だと、急な不在で進行が止まりやすいです。

### H2「よくある質問」— 各FAQ: 質問文はそのまま列挙 → 回答pのみ改訂
**Q1: Excelで十分に管理できないケースは？**  
クライアントの戻りが頻繁に遅れたり、修正が連鎖したりする案件では、手動でのリスケ作業が膨らみやすいです。版管理も煩雑になりがちです。

**Q2: 複数のシートをどう同期させる？**  
逆算表と進行表を別々に持つ場合、変更があったら両方を更新するルールを決めておくか、参照式を活用するとミスが減ります。

**Q3: チームで共有するときの注意点は？**  
共有フォルダを使う場合はファイル名のルールを統一し、更新したら関係者チャットに一言連絡する習慣をつけると混乱が少なくなります。
