## 1. 結論（監督が週間で欲しいもの）

| 問い | 答え |
|------|------|
| **チャートを見せれば足りるか？** | **足りない。** 下請職長はガントを読まない。LINE に貼れる **縦の表・箇条書き** が本命。 |
| **欲しい情報は？** | **いつ・どこで・何を・何が来る・誰に連絡** — 自社に関係する行だけ。 |
| **フォーマットは？** | **① PDF/画像の週間表（LINE）② スマホ用リスト URL** — Excel は副次。 |
| **元請だけの情報は？** | **ある。** 他社工区・人工/見積・遅延理由・監理向け全体 — `visibility` / tier で分離。 |
| **下請 FB は？** | **必要。** コメント・遅延申告・ステータス更新 — ゲスト編集 or 行コメント（v1.1）。 |

---

## 2. なぜチャートだけではダメか

Gemini 実態: 週会で Excel を作り、下請には **LINE や紙** で配る。

| 理由 | 監督の声（典型） |
|------|------------------|
| **スマホで横スクロール** | 現場では片手 · 小さい · 棒グラフの読み取りが苦手 |
| **他社の工程がノイズ** | 「うち以外いらない」 |
| **日付の解像度** | 「月曜から金曜、うちは何があるか」が一目欲しい |
| **LINE 文化** | スクショ1枚 · PDF1枚が届けばよい。URLを開くのは若い職長だけ |
| **印刷・朝礼** | 事務所のプリンタ · 現場掲示用に **縦A4** |

**チャート（分割ペイン）は監督の編集 UI。** 週間配布物は **別ビュー・別 Export** と考える。

## 3. 下請が本当に欲しい情報（受け手目線）

優先度順。v1.1 の週間リスト **必須列** 候補。

| 優先 | 列 / 項目 | 例 | データソース |
|------|-----------|-----|--------------|
| P0 | **日付**（週内の該当日） | 6/2（月） | `start`/`end` · 週フィルタ |
| P0 | **工種・作業名** | 掘削 · 重機入場 | `title` |
| P0 | **工区・位置** | 第1工区 K0+120 | `prop_scope` |
| P1 | **時間帯**（あれば） | 午前 · 終日 | v1.1 プロパティ or 備考 |
| P1 | **重機・車両** | BH 1台 · 10t車 | `prop_machine` · ops 行 |
| P1 | **資材・納入** | 生コン 15m³ · 鉄板12枚 | `prop_material` · `prop_qty` |
| P1 | **監督連絡** | 佐藤 · 090-xxxx | 現場固定 + 担当 |
| P2 | **立会・検査** | 配筋検査立会 | `prop_inspection` |
| P2 | **道路規制** | 車線規制あり | `prop_road` |
| P2 | **前後の注意** | 盛土完了後に着手 | 親行 · `備考` |
| P3 | **ステータス** | 未着手 / 予定変更 | `status` |

**不要（下請向け週間から除外）:**

- 全体工期の月間ガント全体像
- 他社の人工数・見積根拠
- 監理提出用のクリーンな提出行だけの世界
- 社内遅延理由の詳細（監理に言えない内容）

## 4. フォーマット案（配布チャネル別）

### 4-1. 本命: 週間 **表** PDF / PNG（LINE 用）

```
┌─────────────────────────────────────────┐
│ ○○工事 週間予定  2026/6/2〜6/8          │
│ 協力会社: △△土木（この会社向けにフィルタ） │
├──────┬────────┬──────┬────────┬──────────┤
│ 日付 │ 工種   │ 工区 │ 重機等 │ 備考     │
├──────┼────────┼──────┼────────┼──────────┤
│ 6/2  │ 掘削   │ 1工区│ —      │          │
│ 6/3  │ 重機入場│ 1工区│ BH    │ 門番連絡 │
│ …    │        │      │        │          │
└─────────────────────────────────────────┘
監督: 佐藤  /  版: 2026-06-01 17:00
```

- **縦A4 or スマホ幅の PNG** — LINE でそのまま送信
- ヘッダに **週の範囲 · 協力会社名 · 版日時**（版ずれ防止）
- 行は **その下請に関係するものだけ**（フィルタ）

### 4-2. 次点: 閲覧 URL（リスト表示 · 読取専用）

- ガントではなく **表だけ** のモバイルページ
- ゲストログイン or トークン URL（フェーズ C/D）
- 常に最新 — PDF 版との使い分け（PDF=正式週間配布、URL=直前変更）

### 4-3. 副次: xlsx

- IT に強い下請 · 自社で印刷したい場合
- 列は PDF と同型

### 4-4. やらない（週間配布として）

- 月間ガントのスクショ丸投げ
- 編集可能な全体 Schedule 画面の共有（情報漏洩 · 誤操作）

## 5. 元請が Schedule に載せたいが下請に見せたくないもの

既存モデル（`visibility` · `tier` · preset）の延長で設計する。

| 情報 | 元請（site） | 下請週間 | 分離方法 |
|------|--------------|----------|----------|
| 重機・資材 ops 行 | ○ | △ 自社分のみ | `prop_subcontractor` フィルタ |
| 人工数・見積根拠 | ○ | **×** | `tier: ops` · 週間 Export 列から除外 |
| 遅延理由（監理向け） | ○ | **×** | `prop_delay` · `visibility: internal`（**v1.1 追加検討**） |
| 他社の工種詳細 | ○ | **×** | 協力会社フィルタ · または `visibility: subcontractor` 行のみ配布 |
| 監理提出用の全体工程 | submit preset | **×** | submit は監督のみ Export |
| 予備日・天候待ち | ○ | △ マスク可 | 提出同様 · 下請には「調整中」表記のみ |
| 進捗％・コスト | 将来 | **×** | v1 なし |
| 他現場・ポートフォリオ | 組織内 | **×** | ゲストは招待現場のみ（§3-5） |

### 5-1. 推奨: 第4 preset `weekly_sub`（v1.1 案）

| preset | 誰が使う | 行 | 列 |
|--------|----------|-----|-----|
| `submit` | 監督 → 監理 | 提出行のみ | official |
| `site` | 監督編集 | 全現場行 | official + ops |
| **`weekly_sub`** | 監督 → 下請配布 | `subcontractor` 可 + 当該会社フィルタ | 週間必須列のみ（人工数等除外） |

`visibility` 拡張案:

| 値 | 意味 |
|----|------|
| `submit` | 監理提出に載る |
| `site` | 現場のみ（既存） |
| `both` | 両方（既存） |
| **`internal`**（v1.1） | 元請監督のみ — 下請週間・提出ともに非表示 |
| **`subcontractor`**（v1.1） | 下請週間に載せる（site 行のうち配布対象） |

既存 `site` 行の多くは `both` or `subcontractor` で足りる場合もある — dogfood で検証。

## 6. 下請からのフィードバック（FB）

監督が週間配布 **後** に欲しいこと。

| FB 種別 | 例 | 実装案 | フェーズ |
|---------|-----|--------|----------|
| **確認** | 「見ました」 | 行の `ack` フラグ · 任意 | v1.2 |
| **遅延申告** | 「雨で1日遅れ」 | ゲストが `prop_delay` or コメント | v1.1 |
| **ステータス更新** | 進行中 / 完了 | ゲストが `status` 編集 | D（ゲスト編集） |
| **日程変更依頼** | 「水曜に変更希望」 | コメント → 監督が承認して日付変更 | v1.1 |
| **自社工程の追記** | 下請が自社内の細分工を追加 | ゲスト編集 · `contractorTag` 行 | v1.1 |
| **リアルタイムチャット** | LINE で十分 | **やらない** | — |

**原則:** FB は **行に紐づく**（チャットアプリ化しない）。監督が承認してから工程表本体が変わる（勝手に全体が動かない — Q-INS-01 A と整合）。

ゲストは v1 **現場全体編集可**（MASTER §3-5）— v1.1 で **行フィルタ（contractorTag）** を足すと FB と整合しやすい。

## 7. 監督ワークフロー（週次ループ）

```
金曜 or 週初
  1. site ビューで来週を確認・更新
  2. 「週間リスト出力」— 協力会社ごとに PDF/PNG
  3. LINE で各下請に送信（会社別ファイル）
  4. 必要なら閲覧 URL も送る

週中
  5. 変更があれば site で修正 → 版を上げて再送 or URL で追随
  6. 下請 FB（コメント・遅延）を行に反映

月末
  7. submit Export は別ルート（監理）— 週間リストとは混ぜない
```

## 8. 製品への落とし込み（v1.1 提案）

| # | 機能 | 説明 |
|---|------|------|
| W-1 | **週範囲ピッカー** | 来週月〜日 · 今週残り |
| W-2 | **協力会社フィルタ** | `prop_subcontractor` 単位で Export 分割 |
| W-3 | **週間表 PDF/PNG** | ガントなし · 縦表 · 版日時ヘッダ |
| W-4 | **列プリセット `weekly_sub`** | 人工数等を落とした列セット |
| W-5 | **行コメント** | 下請 FB · 監督承認フローは手動でも可 |
| W-6 | **閲覧 URL** | リスト表示 · read-only デフォルト |

**v1 キラー（月間提出）と混同しない** — Export メニューは「提出用」「週間・○○社向け」を分離。
