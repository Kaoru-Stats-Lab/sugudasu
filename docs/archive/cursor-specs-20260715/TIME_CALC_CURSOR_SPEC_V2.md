# time-calc 実装仕様書（Cursor向け・v2）

対象: SUGUDASU 新規ツール `time-calc`（労働時間 h:mm合計＋時給計算）
ステータス: 公開済み（アルファ版）。本書 v2 は「合計時間の10進小数表記」を追加する差分を含む
主ペルソナ = バイト・パート本人（個人利用）。追加差分の主ペルソナ = 給与監査・請求書作成で10進小数の時間表記を必要とする事務担当
前提: 静的サイト・登録不要・ブラウザ内完結。サーバー通信は一切行わない。

## 0. ガードレール（Cursorへの厳守事項）

Cursorは指示がないと以下をやりがちなので、明示的に禁止する。

- **フレームワーク・ビルドツールを追加しない。** React/Vue/npm依存の追加は禁止。素のHTML/CSS/JSのみ。既存の他id（例: normalize系）と同じ構成に合わせる。
- **状態管理ライブラリ・クラス設計を持ち込まない。** 状態はグローバル変数1〜2個 + 純粋関数で十分。Reduxパターンやカスタムイベントバスは不要。
- **ファイルを分割しすぎない。** `time-calc.html` 1ファイルに HTML/CSS/JS を収める（既存id踏襲）。`parser.js` 等への分離は不要。
- **抽象化を先回りしない。** 「将来の拡張性のため」にオプションオブジェクトや設定ファイル、プラグイン機構を作らない。今回のスコープ（後述4）だけを実装する。
- **バリデーションは1関数に集約する。** 行ごとのパース処理は `parseLine(line: string): ParsedLine` のみ。正規表現を分散させない。
- **UIコンポーネント化しない。** Web Componentsやテンプレートエンジンは不要。DOM操作は素の `innerHTML` / `textContent` で十分。
- **外部ライブラリを追加しない。** 日付/時刻ライブラリ（day.js等）は不要。パース対象がシンプルな文字列なので標準JSで完結する。
- **コメントは最小限。** 関数の意図が名前で分かるようにし、行コメントの乱発をしない。JSDocは主要関数のみ。
- **console.logを残さない。** デバッグ用ログは実装完了時に全て削除する。
- **既存の共通CSS/デザイントークンを再利用する。** 新規に色・余白の値を定義せず、他id（normalize等）で使われているクラス・変数をそのまま使う。差分がある場合のみ質問すること。
- **テストコードは仕様の受け入れ基準（6章）をそのまま使う。** 独自のテストフレームワーク導入は不要。ブラウザのconsoleで手動確認できる形で十分。

## 1. スコープ

**やること**
- 複数行のテキスト入力（1行=1日分の勤務）から合計時間（h:mm）を算出
- 合計時間を10進小数（例: `14.75時間`）でも併記する（v2で追加。給与監査・請求書作成で「45÷60」を都度手計算している摩擦を解消するのが目的。既存の合計計算ロジックには一切手を入れず、表示を1項目追加するだけ）
- 時給を入力すると概算支給額を表示
- 入力エラーのある行をハイライトし、処理は継続する

**やらないこと（Out — LP/UIに明記必須）**
- 残業代・割増賃金・法定労働時間の自動判定
- 休憩時間の自動控除（休憩は入力側で引き算してもらう前提。将来検討）
- 複数人分のシフト集計（店舗オーナー向けの一覧管理machineは対象外）
- 案件別の請求書連携（invoiceとは独立）

## 2. 画面構成（左入力・右出力の2カラム）

既存SUGUDASU系ツールの「左に入力、右に結果」パターンを踏襲する。

```
+------------------------+------------------------+
| 入力（縦に複数行）       | 結果                     |
| 1行 = 1日分              | 各行の解釈結果一覧         |
|                          | 合計時間                  |
| [時給の入力欄]            | 概算支給額                |
+------------------------+------------------------+
```

- 入力欄: `<textarea>`、複数行貼り付け対応、placeholder に入力例を3パターン表示
- 時給欄: 入力欄の直下（または右カラム上部）に `<input type="number">` を1つ
- 右カラム: パース結果を1行=1テーブル行で表示（元の入力／解釈した時間／エラー表示）
- 最下部: 合計時間（h:mm）＋10進小数表記（例: `14:45（＝14.75時間）`）＋支給額（時給入力時のみ表示）

## 3. データモデル

```js
// 1行分のパース結果
// {
//   raw: string,        // 元のテキスト（そのまま表示用）
//   minutes: number|null, // 解釈できた分数。エラー時はnull
//   status: 'ok' | 'warning' | 'error',
//   message: string|null  // warning/error時の表示文言
// }
```

- `status: 'ok'` — 正常にパースできた
- `status: 'warning'` — パースはできたが曖昧さがあった（例: `1.30` を1.3時間として解釈したが誤入力の可能性あり）
- `status: 'error'` — パース不能。合計計算から除外する

---

## 4. パース仕様（`parseLine`）

入力1行に対して、以下の優先順位で解釈する。空行はスキップ（結果に表示しない）。

| 優先度 | 形式 | 例 | 解釈 |
|---|---|---|---|
| 1 | `H:MM-H:MM`（時刻範囲） | `9:00-18:00` | 終了時刻 − 開始時刻。日をまたぐ場合（終了<開始）は翌日跨ぎとして+24時間 |
| 2 | `H:MM`（コロン区切りの時間量） | `1:30` | 1時間30分 = 90分 |
| 3 | `N分` | `90分` | 90分 |
| 4 | 小数 | `1.5` | 1.5時間 = 90分 |
| 5 | 上記いずれにも一致しない | `1.30`（小数か時刻か曖昧） | `status: 'warning'`、小数として暫定解釈しつつ「1:30の意味では？」と注記 |
| — | 完全に数値変換できない文字列 | `abc` | `status: 'error'` |

**時刻範囲のバリデーション**
- 開始・終了とも `H:MM` または `H` 形式を許容（`9-18` も可）
- 終了 < 開始の場合は日またぎとみなし24時間加算（例: `22:00-6:00` → 8時間）。ただし警告表示は出す（意図しない入力の可能性があるため）

**行フォーマットの補足要素（任意）**
- 行末に半角/全角スペース区切りでラベルを許容: `9:00-18:00 出勤` → ラベル部分は表示用にのみ保持し、計算には使わない

---

## 5. 計算仕様

- 合計時間 = 全行の `minutes`（`status !== 'error'`）の合計を h:mm 形式に変換
- **10進小数表記（v2追加）** = 合計分数 ÷ 60 を計算し、小数点第2位で四捨五入して表示（例: 870分 → 14.5時間、875分 → 14.583...→ 14.58時間）。丸め誤差は表示上の問題のみで、単純な算術のため税務・労務上の断定表現には該当しない（F7対象外）
- 支給額 = 合計時間（分）÷ 60 × 時給。**円未満は切り捨て**
- 表示形式: 合計時間は `12:30`（12時間30分）のように表示。0埋めは時のみ不要（`1:05`でOK）

## 6. 受け入れ基準（テストケース）

Cursorはこの表をそのままテストケースとして実装・確認すること。

| 入力 | 期待される分数 | status |
|---|---|---|
| `9:00-18:00` | 540 | ok |
| `9:00-13:00` | 240 | ok |
| `22:00-6:00` | 480（日またぎ） | warning（日またぎ注記） |
| `1:30` | 90 | ok |
| `90分` | 90 | ok |
| `1.5` | 90 | ok |
| `1.30` | 78（1.3時間として暫定解釈） | warning |
| `abc` | null | error |
| （空行） | 表示自体をスキップ | — |
| `9:00-18:00 出勤` | 540（ラベルは無視） | ok |

**合計計算の確認例**
入力:
```
9:00-18:00
9:00-13:00
1:30
```
→ 合計 540+240+90 = 870分 = `14:30`（＝**14.5時間**）
時給1200円入力時 → 870/60×1200 = 17400円

**小数表記の確認例（v2追加）**
| 合計分数 | h:mm表示 | 10進小数表示 |
|---|---|---|
| 885分（14時間45分） | `14:45` | `14.75時間` |
| 870分（14時間30分） | `14:30` | `14.5時間` |
| 875分（14時間35分） | `14:35` | `14.58時間`（14.5833...→四捨五入） |

## 9. v2差分についての注意（Cursorへの指示）

- `time-calc`は既に公開済みのため、本差分は**新規実装ではなく既存ファイルへの追記**である。実装前に必ず既存の `time-calc.html`（合計時間の計算・表示ロジック）を読み込み、現状の実装方法を報告してから着手すること
- 新しいツール（`time-convert`等）や新しいモードを作らない。既存の合計表示の隣に小数表記を1項目追加するだけに留める
- 既存の合計計算ロジック（分の積算処理）には手を入れない。小数変換は表示直前の1ステップとして追加する
- 新しい入力欄・新しいパース処理は追加しない（既存の`parseLine`・入力欄はそのまま）
