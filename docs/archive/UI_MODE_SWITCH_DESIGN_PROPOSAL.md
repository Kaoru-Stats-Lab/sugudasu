# UI 提案 — モード切替とボタン配色の統一

**Status:** ARCHIVED（2026-06-19 統合完了 · 2026-07-03 `docs/archive/` へ移動）  
**正本:** [`DESIGN_GUIDELINE.md`](../DESIGN_GUIDELINE.md) §0–§3.3 · `assets/sugudasu.css`（`.sg-segment*`）

## 1. いま何が起きているか

### 1.1 モード切替が見えにくい理由

| 現象 | 原因 |
|------|------|
| 選択中と未選択の差が弱い | 白 vs 薄グレー + 影だけ（彩度ゼロ） |
| モードを変えた感がない | フォーム全体の色・トーンがほぼ不変 |
| 何が「モード」か分かりにくい | 書類種別・出力形式・算法・AI連携が**同じ見た目の部品**で混在 |

対象ツールとモードの対応:

| ツール | モードの意味 | 現状の部品 |
|--------|-------------|------------|
| invoice | 見積 / 納品 / 請求 | 3分割 · 白/灰 |
| receipt | 1枚 / A4マルチ | 2分割 · 白/灰 |
| warikan | 傾斜係数 / 固定額 | 2分割 · 白/灰 |
| report | ローカル整形 / Gemini | 2分割 · 白/灰 |
| reverse | 辞書 / Gemini | 2分割 · 白/灰 |
| label | ラベル / 名刺 | 2分割 · **青アクティブ**（他と不一致） |

invoice だけプレビュー側で `honorBgClass`（見積=slate / 納品=slate-700 / 請求=indigo）が変わるが、**タブ自体は色が変わらない**。

### 1.2 青ボタンと緑ボタンが共存する理由（現行ルール）

`DESIGN_GUIDELINE.md` では意図的に分離されている:

| 色 | 役割 | 例 |
|----|------|-----|
| **青** `blue-600` | 画面内の**主アクション**（生成・計算・検索） | 「整形する」「計算して精算」「シフト自動生成」 |
| **緑** `emerald-600` | **出力固定**（印刷 / PDF）· 1画面1つ | ヘッダー「印刷/PDF」· receipt の PDF ボタン |

問題はルール違反というより、**セグメント（モード）が無色のため、緑・青のCTAと同列に見えて階層がフラット**なこと。

## 2. 提案するデザインスキーム — 「3層アクション」

ユーザーが瞬時に判別できるよう、操作を **3層** に固定する。

```text
┌─────────────────────────────────────────┐
│ L1 モード選択（sg-segment）              │  … 状態を変える · 破壊的操作ではない
│     見た目: トラック + スライドピル      │
├─────────────────────────────────────────┤
│ L2 主アクション（sg-btn-primary）        │  … 青 · 1画面1つ · 「実行」
├─────────────────────────────────────────┤
│ L3 出力（sg-btn-print）                  │  … 緑 · 印刷/PDFのみ · ヘッダー or 結果直下
└─────────────────────────────────────────┘
```

**原則:** モード切替に青・緑は使わない。青緑は **L2/L3 専用**。

## 3. コンポーネント案 — `sg-segment`

### 3.1 見た目（全ツール共通）

```text
[ トラック: slate-100 · rounded-xl · p-1 ]
   ┌──────────────┐
   │ スライドピル   │  … bg-white · shadow-sm · ring-1 ring-blue-600/20
   └──────────────┘
   [ ラベル: 非選択 text-slate-500 / 選択 text-blue-800 font-bold ]
```

- **スライドアニメーション:** `transform: translateX` · 200ms ease — 「切り替わった」が体感できる
- **アクティブラベル:** 文字色 `blue-800`（面全体を青塗りにしない）
- **任意 · 下線アクセント:** invoice だけタブごとに 2px 下線色を変える（見積=slate-600 / 納品=slate-700 / 請求=blue-700）— **ツールテーマ色の乱立はしない**

### 3.2 モード変更時の「余韻」（軽量）

| 手法 | 適用 | コスト |
|------|------|--------|
| セグメント下の **説明1行** がフェード差し替え | 全ツール | JS 数行 |
| プレビュー領域 `opacity 0.6→1` 200ms | invoice · receipt · warikan | CSS |
| フォーム先頭に **薄いティント帯**（`blue-50/50`）がモード連動 | invoice のみ（既存 indigo 案内を統合） | 小 |

### 3.3 アクセシビリティ

- `role="tablist"` / `role="tab"` / `aria-selected`
- キーボード: ← → で隣接モード
- フォーカスリング: `ring-2 ring-blue-500/40`

## 4. ツール別の整理（実装時のチェックリスト）

| ツール | L1 ラベル改善案 | L2 主CTA（青1つ） | L3 出力（緑） |
|--------|----------------|-------------------|---------------|
| invoice | 見積·納品·請求 + 下線色 | （暗黙: 入力即プレビュー） | ヘッダー印刷のみ |
| receipt | 1枚·A4マルチ | — | **receipt 内 emerald をヘッダーに寄せる検討**（緑の二重化解消） |
| warikan | 傾斜·固定額 | 「計算して精算」 | — |
| report | ローカル·Gemini | 「整形する」 | — |
| reverse | 辞書·Gemini | 「検索して類語を」 | — |
| shift | （枠数は別UI · セグメント化しない） | 「シフト自動生成」 | ヘッダー印刷 |
| label | ラベル·名刺 | — | ヘッダー印刷 |

**receipt の緑ボタン:** ガイドライン上「1画面1つ」のはずが、ヘッダー emerald + フォーム内 emerald が **二重**。統一案は **フォーム内を青セカンダリ（アウトライン）に落とし、緑はヘッダー1つ** に集約。

## 5. ボタン階層の再定義（DESIGN_GUIDELINE 追記候補）

| クラス名 | 色 | 用途 | 画面上限 |
|----------|-----|------|----------|
| `sg-segment` | 無彩色 + 青文字アクティブ | モード切替 | セクションあたり1組 |
| `sg-btn-primary` | blue-600 | 実行・生成・計算 | **1つ** |
| `sg-btn-print` | emerald-600 | 印刷/PDF | **1つ**（ヘッダー優先） |
| `sg-btn-secondary` | 白枠 | コピー・保存・読込 | 複数可 |
| `sg-btn-x` | 黒 | Xシェア | 1つまで推奨 |

**禁止:** セグメントの選択肢を `bg-blue-600` 塗りつぶし（label.html の現状）— L2 と混同する。

## 7. やらないこと

- ツールごとにテーマカラーを変える（indigo/amber 乱立 — ガイドライン違反）
- モード切替のたびに全面リロード感のある派手アニメ
- グラデーションボタンの復活
- セグメントと主CTAを同じ青塗りにする

---

## 8. 期待される UX 効果

1. **モード = 上の無彩色トラック**、**実行 = 青**、**出力 = 緑** が身体記憶になる  
2. スライドピルで「今どの文脈か」が視線誘導される  
3. invoice のプレビュー色変化とタブ下線が連動し、転用フローが理解しやすくなる  
4. Zenn #1「書類タブ切替で転用」と UI が一致し、記事・製品の説明コストが下がる
