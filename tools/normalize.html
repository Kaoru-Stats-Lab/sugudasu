外部送信ゼロ · 事務 · EC登録

社内規程で無料の全角半角サイトに名簿を貼れない — 手打ちや Excel 関数地獄を、貼付 → 用途ボタン → コピーでここで終わらせます。

顧客名簿 · EC一括登録 · メールリストの全角半角 · 空白 · 改行をブラウザ内だけで整えます。変換後は入力 N 行 → 出力 M 行を必ず表示（行ズレ防止）。

社員マスタのダミーCSVは テストデータ から「全角半角整えで開く」でも渡せます。 手順の読み物は 実務ガイド（都内総務OL編） · 班分け前の名簿は 班分け · 複数列の表は 表変換 へ。

全角半角整えのよくある質問

        
使い方のあとに、社内規程・データの扱いに関する質問を載せています。場面別の読み物は 実務ガイド（都内総務OL編）。

        

            

                社内規程で無料の全角半角変換サイトが禁止でも使えますか？
                

                    
当社サーバーへテキストを送りません。 処理はお使いのブラウザ内だけです。社内規程の最終判断は貴社の情報セキュリティ担当にご確認ください。一般的な「外部 Web コンバータに貼る」行為とは、データの行き先が異なります。詳細は SUGUDASU の約束 · プライバシーポリシー。

                

            

            

                Excel の関数でもできますか？
                

                    
はい。Excel / スプシの ASC · TRIM · SUBSTITUTE · TEXTJOIN 等で、シート内なら同様の変換は可能です。本ツールは Excel の代替ではなく、列をコピーして EC 登録 · CRM · 別 PC へ貼る途中での行ズレ · 変換ミスを減らすためのものです。毎週同じブックで補助列を回すなら、Excel テンプレの方が向くことが多いです。

                    
Excel · Web コンバータ · ブラウザ内処理の使い分けは、Zenn 記事「Excel列コピーと全角半角サイトのあるある」（2026/7/7 公開予定）で詳述します。公開後は 更新履歴 からリンクします。

                

            

            

                Excelに貼り戻したら行が上下にズレました
                

                    
セル内改行（Alt+Enter）や、Excel側のフィルター・非表示行が原因のことが多いです。本ツールはデフォルトで行数を変えませんが、貼り付け先もご確認ください。

                

            

            

                先頭ゼロ付きコード（00123）は消えますか？
                

                    
消えません。Excel のように数値型に変換せず、文字列のまま処理します。

                

            

            

                カタカナの伸ばし棒（ー）がハイフンになりますか？
                

                    
なりません。カタカナ直後の長音「ー」は保護し、型番などのハイフン類だけを統一します（CSV名簿プリセットで ON の場合）。

                

            

            

                変換ボタンを押し忘れて古いデータをコピーしませんか？
                

                    
「コピー」ボタンは押した瞬間に最新設定で再変換してからクリップボードへ保存します。入力欄の生テキストはコピーしません。

                

            

            

                メール・電話・氏名の伏字は同時に使えますか？
                

                    
はい。「伏字」パネルで複数チェックできます。1行にメールと電話が混在するリストでも、選んだ種類だけを順に伏せます。用途プリセット（全角半角など）とも併用できます。

                

            

            

                テキスト伏字と画像の黒塗りはどう使い分けますか？
                

                    
メール・電話・氏名のリストは本ツールの伏字プリセット向きです。行数は変わらず、コピーしてそのまま貼れます。スクショ・マニュアル画像の機密は マスク（Canvas 黒塗り · ぼかし）を使ってください。どちらもブラウザ内だけで処理し、当社サーバーへ送信しません。

                

            

            

                メール伏字はどの部分が残りますか？
                

                    
@より前の先頭1〜2文字とドメイン（@以降）は残し、それ以外を *** に置き換えます。例: yamada.taro@example.com → ya***@example.com。完全匿名化や復元不能の保証はしません。社内規程に合わせて確認してください。

                

            

            

                電話伏字のルールは？
                

                    
ハイフン付き番号は中央4桁を **** にします（例: 090-1234-5678 → 090-****-5678）。数字のみ11桁・10桁も同様に中央4桁を伏せます。国際番号や内線など特殊形式は意図どおりにならない場合があります。

                

            

            

                Excel のタブ区切りをカンマ1行にできますか？
                

                    
はい。タブ→カンマ プリセットで、Excel から横にコピーしたタブ区切りをカンマ区切りの1行にまとめられます。SQL IN プリセットとの違いはクォート（'）を付けない点です。メール一括入力やスプシの TEXTJOIN 前処理向けです。

                

            

            

                空行や前後の空白だけを整理できますか？
                

                    
はい。「行の整理」で各行の前後空白を削除 · 空行を削除を選べます。ソートや重複削除と併用できます。行数が変わるため、コピー前に行数バッジを確認してください。

                

            

            

                先頭行が「名前」「メール」などの見出しのときは？
                

                    
貼付時に黄バナーでお知らせします。見出し行は変換対象に含めないよう、データ行だけを貼るか、変換後に先頭行を手動で削除してください。

                

            

            

                SQL の IN 句用に 'id1', 'id2' 形式へまとめられますか？
                

                    
はい。SQL IN プリセットで、改行またはタブ区切りのリストを 'value1', 'value2' の1行にします。値に含まれる単一引用符は SQL 慣習どおり '' にエスケープします。本番IDを外部サイトに貼れない場合向けです。

                

            

            

                改行をカンマ区切りの1行にできますか？
                

                    
はい。「改行→カンマ」プリセットで、縦並びのリストをカンマ区切りの1行にまとめられます。このとき行数は変わるため、コピー前に行数バッジを確認してください。

                

            

            

                並び替えで「10」が「2」より前に来ます
                

                    
デフォルトは文字列としての並び替えです。「1」「10」「2」の順になります。数値の大小順にしたい場合は「行の整理」で数値としてにチェックを入れてください。全角数字や先頭スペースがある行は意図しない位置になることがあります。

                

            

            

                重複削除はどの行が残りますか？
                

                    
デフォルトは完全一致のみ削除し、上から見つかった1行目を残します。「大文字小文字・全角半角を区別しない」は任意オプションです。表記ゆれを残して確認したい場合はオプションをオフのままにしてください。

                

            

            

                行フィルタで正規表現は使えますか？
                

                    
いいえ。v1 ではキーワードの含む / 含まないのみです。ドメインや ID の一部を指定して行を絞り込めます。

                

            

            

                データは外部に送信されますか？
                

                    
いいえ。 すべてブラウザ内だけで処理されます。オフラインでも動作します。永続保存もしません。DevTools の Network タブでも当方ドメインへの POST は発生しません（SUGUDASU の約束）。

                

            

            

                Mac から Windows へコピペすると文字化けしますか？
                

                    
化けた文字（� や ?? など）を検知した場合、画面上部に警告を表示します。元のテキストを確認してください。
