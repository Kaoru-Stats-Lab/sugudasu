閉域環境・外部CSV持ち込み禁止の現場で、給与計算SaaSや労務システムのインポート検証用データを作りたいとき向けです。 Mockaroo 等の外部SaaSや、ネットのダミーCSVを使えない情シス・受託SI・マニュアル担当の下位レイヤーが、Excel手打ちをやめるためのツールです。 社員マスタ（氏名 · フリガナ · 給与 · 住所）をブラウザ内だけで生成 — サーバーへデータを送りません。 郵便番号と住所は整合済み。シード固定で再現可能。マイナンバー · 実口座は生成しません。 大規模試験はCSVダウンロードが本線です（マスク 等と用途が異なります）。

閉域・テストデータのよくある質問

閉域環境や外部サイト禁止の現場でも使えますか？

はい。テストデータの生成からCSVダウンロードまで、端末のブラウザ内だけで完結します。氏名・給与などの入力データを当社サーバーへ送りません。

次のような現場向けです。

  • 外部ファイル持ち込み禁止 — ネットのダミーCSVを落とせない
  • クラウドSaaS禁止 — Mockaroo 等の外部テストデータサービスが使えない
  • 本番・個人情報の持ち出し禁止 — マスキング export も審査が重い
  • 閉域網・顧客VPN内 — ページが開ければ生成処理はオフラインで動く（初回はHTML/JSの読み込みが必要）

社内規程の最終判断は貴社の情報セキュリティ担当にご確認ください。送信がないことの確認手順は SUGUDASU の約束（DevTools） · 情報管理向けまとめ をご覧ください。

プロキシのホワイトリスト申請や広告で止まる場合は、下の FAQ「社内のホワイトリスト」「広告があると」を参照してください。

なぜ SUGUDASU テストデータがあるのですか？

閉域で、今日、整合済みのテストデータが欲しいのに、手段がない — このギャップのためです。

エンタープライズのテストデータ管理（TDM）や本番DBマスキングの代替ではありません。次の2つの現場向けです。

① 非エンジニア（社労士事務所 · マニュアル担当 · 総務） — Excelで社員マスタを手打ちしている · ネットのダミーCSVを持ち込めない · 整合（郵便番号↔住所 · 社員↔給与）を自分では揃えられない。

② エンジニア（受託SI · 社内QA · 情シス兼務） — 閉域なら自前スクリプトやFakerでも作れるのはその通りです。それでも次で止まることが多いです。

  • 自作ツールの社内配布・稟議が重い（「このURLは説明資料付きの静的ツール」）
  • 労務・給与SaaS向けの列整合（再雇用 · 給与3ヶ月連動）を毎回自前で書くのが面倒
  • 本番コピー禁止 · 外部SaaS禁止で、今日・この端末でCSVが欲しい
  • 監査でシード再現を求められる

Excel手打ちのペルソナは入口の痛みの絵です。実際の利用者にはエンジニアが混ざりますが、彼らに「Excelやめよう」と売るより、稟議と整合済みCSVを短時間でが刺さります。

他のダミーデータ・テストデータ生成ツールと何が違いますか？

閉域・持ち込み禁止の現場で、労務・給与向けに「整合済みCSVをブラウザだけで」 — この楔に絞っている点が違います。用途に応じて使い分けてください。

手段 向くこと SUGUDASUとの違い
ネットのダミーCSV 手軽にファイルが欲しい 外部DL不要 · 出所を説明しやすい · 社内ID/列名を指定可
Mockaroo 等のクラウドSaaS 多型・大量 · API連携 当社へデータを送らない · アカウント不要 · 社内SaaS禁止でも説明しやすい
Excel手打ち 数行だけ急ぎ 郵便番号↔住所 · 社員↔給与明細の整合 · 最大25万件 · シード再現
本番DBマスキング · エンタープライズTDM 本番に近い大規模データ 代替ではない · 架空生成のみ · 受託SI・情シスの下位レイヤー向け
汎用のブラウザ型ジェネレータ 氏名・住所など汎用fake 労務・給与プリセット（再雇用 · 地雷 · 給与3ヶ月連動）· 日本の住所整合 · 非送信の説明資料あり

50型ビルダー全面競合はしません。よく事故る労務インポートに深く刺さる範囲だけを増やします。

給与明細は社員マスタと連動しますか？

はい。同じシード・件数で社員を生成し、社員番号に紐づく3ヶ月分の明細行を出力します。基本給・通勤手当は社員マスタと一致します。

生年月日・入社日の形式は変えられますか？

YYYY/MM/DD · YYYY-MM-DD · YYYYMMDD から選べます。インポート先の日付列仕様に合わせてください。

全角半角整えへそのまま渡せますか？

500行以下のサンプルだけです（ヘッダー含む）。「全角半角整えで開く」で 全角半角整え のCSV名簿プリセットに handoff します。

大規模エクスポート（2.5万〜25万件）や5,000件超は CSVダウンロードが正 — normalize とは連携しません。全半角の一括変換はインポート先の仕様に合わせて生成時の形式で対応するか、ご自身のスクリプトで行ってください（全角半角整え は Excel 列コピー500行向けの別ツールです）。

入社年月日や社員番号の「年」は固定ですか？

いいえ。基準年はデフォルトでブラウザの現在年になり、入社年月日の上限に使います。入社開始年も指定できます。過去の改修テストでは基準年を下げてください。

再雇用・高齢層のデータは含まれますか？

雇用形態に再雇用（約7%）があり、生年月日は1948〜1964年帯・入社は60歳以降を想定しています。定年後再雇用や高年齢雇用の計算・表示テスト向けです。

給与SaaS（マネーフォワード・freee等）にそのまま入りますか？

列名は画面の「列名（CSVヘッダー）」で変更できます。テンプレート（従業員番号表記 · 給与SaaS寄り）を選んでから微調整してください。製品ごとに1文字違う場合もここで合わせられます。

本物の個人情報は使われますか？

いいえ。すべて架空のデータです。実在の口座番号・マイナンバーは生成しません。

同じデータを再現できますか？

はい。シード値を同じにすれば、同じ件数・設定で同じCSVが再現されます。バグ修正後の再検証や、監査で「同じ手順で再テスト」と言われたときに使えます。

地雷データとは何ですか？

外字 · 超長住所 · 不正形式のメールなど、画面やバリデーションの境界を試すデータを約5%混入するオプションです。きれいなダミーだけでは拾えないバグを意図的に炙り出すテスト向けです。

データはサーバーに送信されますか？

いいえ。生成からCSVダウンロードまで端末のブラウザ内だけです。これが閉域・非送信ポリシーの核心です。

Chrome の DevTools → Network で、生成時に当社ドメインへPOSTが飛ばないことをご自身で確認できます。手順は SUGUDASU の約束（DevTools） · 情報管理向けまとめ。

※ 初回アクセス時の HTML/CSS/JS 読み込みは通常のWebページと同様です。プロキシで sugudasu.com 自体がブロックされる場合は別問題です。

なぜ「通信なし」でデータセットが作れるのですか？

「通信ゼロ」ではなく、「生成のたびにクラウドへデータを送らない」という意味です。Mockaroo のように1件ずつサーバーが返すのではありません。

3ステップで理解できます（情シス向け · エンジニアは②で足りることが多い）:

  1. 初回だけ — ページとプログラム（JavaScript）· 氏名・住所などの辞書データが端末のブラウザへ届く（通常のWebページ読み込みと同じ通信）
  2. 生成ボタン以降 — 届いたプログラムが端末内で乱数とルールを回し、社員行を組み立てる。この間、氏名・給与などを当社サーバーへ送る API は呼びません
  3. ダウンロード — できた CSV は端末からファイル保存。アップロードはしません

エンジニア向けの一言: 埋め込み辞書 ＋ クライアント側 RNG（シード再現可）＋ POST なし。25万件もチャンク単位で端末内生成です。

「完全オフライン」とは言いません。初回の HTML/JS 取得 · 広告 · フォント取得は 通信レイヤ の②③と別問題です。生成処理が①（端末内）であることを DevTools で確認できます。

社内のホワイトリスト（閲覧許可）に載せられますか？

貴社の情報セキュリティ担当が決めます。当社が代わりに承認することはできません。ただし、申請しやすいよう 情報管理向けまとめ にコピー可の申請メモと DevTools 手順を公開しています。

エンジニア・情シスがよくやる流れ（型の例）:

  1. 社内規程で「外部Webツール」が許されるか確認
  2. 申請書に https://sugudasu.com/test-data · 業務目的（テスト用CSV生成）· 入力データをPOSTしない を記載
  3. プロキシで sugudasu.com を許可（フォント用に fonts.googleapis.com も求められることがあります · UI用 · 生成には必須ではありません）
  4. 許可後、DevTools で生成時に POST が出ないことを確認
  5. インターネット自体が使えない現場は、ビルド済み静的ファイルの社内ミラー（下のFAQ）が現実解

受託SIの情シスはこの型に慣れていることが多いですが、事務職だけの現場では型がなく、申請で止まることもあります。

広告があると、閉域現場で使えませんか？

必ずしも使えません、とは言い切れません。 SUGUDASU コア（sugudasu.com）は無料運営のため Google AdSense の自動広告タグを全ページに設置しています（サイト承認は Google 側で進行中）。一方、CSVの生成処理には広告は不要です（通信レイヤの③ と①は別）。

よくあるパターン:

  • ページ本体だけ許可 → 広告ドメインはブロックされても生成は動く（枠が空 · 見た目だけ劣化）
  • 広告カテゴリ全面ブロック → 情シスに「広告は業務に不要」と伝える · 申請メモ を添付
  • 外部ドメイン全面禁止 → ホワイトリストではなく 社内ミラー（静的HTMLを社内サーバに置く）

登録・同期が必要な SUGUDASU Sync（sync.sugudasu.com）は広告なしの別製品です。テストデータ楔とは用途が違います。

インターネットが使えない現場ではどうすればいいですか？

社内ミラーが現実解です。社外で一度 npm run build:pages してできた静的ファイル（dist/）を、貴社の検証用Webサーバや閉域PCに配置し、社内URLから開きます。広告タグはミラーに含めません。

手順の細部（どのファイルをどこへ置くか · 更新頻度）は貴社情シスと合意してください。ダウンロードしたCSVの保管・廃棄も利用者側の責任です。

「ページが一度開ければ、生成はオフラインでも動く」環境もありますが、初回のHTML/JS取得がボトルネックになる場合はミラーの方が確実です。

エンタープライズのテストデータ管理ツールの代替ですか？

いいえ。本番DBマスキングや大規模負荷試験基盤の代替ではありません。受託開発・社内テスト・マニュアル担当が、Excel手打ちや怪しいDLをやめるための軽量ツールです。
