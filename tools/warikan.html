傾斜割り勘・飲み会精算のよくある質問

使い方のあとに、データの扱いと信頼に関する質問を載せています。

幹事が嫌でも、手間は減りますか？

幹事役そのものは楽になりませんが、レジ後の按分・丸め・LINE清算文をここで終わらせられます。「幹事の計算が怪しい」という疑念も、清算文を共有すれば減らしやすくなります。幹事を増やすツールではなく、幹事の負担を少し軽くするツールです。

2次会・3次会は、参加者だけでもう一度割り勘すればいい？

はい。別途このツールをもう一度使ってください。任意参加の2次会・3次会は、その回のお会計総額と参加者だけ入れ直して計算する運用です。1次会から全部まとめる機能はありません。

上司と新人で飲み会の割り勘を公平にするには？

「傾斜係数モード」でグループごとに係数と人数を設定します。役員・上司は2.0、一般は1.0のように、立場の塊で按分するため人数の多い飲み会でも入力が速いです。遅刻・非飲酒は係数を手動で下げて調整できます。

合コンで女性だけ固定額（例3,000円）、男性は残りを割り勘したい

「固定額モード（合コン等）」を選び、女性側に1人あたりの固定金額と人数を入力。男性側の1人あたり欄は空欄（残り按分）のまま人数だけ入れると、総額から自動計算されます。

金額や人数はサーバーに送られますか？

将来 URL で入力状態を共有する機能を入れた場合も同様です。 金額・人数・傾斜はサーバーに送りません。ブラウザ内で計算し、SUGUDASU に保存する設計にはしていません（幹事王の「URL内のみ」と同じ考え方）。清算文はコピーして LINE / Slack へ貼り付けて共有してください。URL共有は現時点未実装（将来もサーバー非送信を前提に検討）。詳細は SUGUDASU の約束 · プライバシーポリシー。

ページを閉じると入力は消えますか？

消えます。localStorage による自動復元もありません。再訪問時は空のフォームからです。記録が必要なら清算文をコピーして幹事が自行保管してください。URL や JSON ファイルでの入力復元は未実装です（ロードマップ候補 — 実装してもサーバー非送信の方針は変えません）。

端数や「調整のすき間」は誰の負担ですか？

なぜ幹事に余り・不足が出ることがある？ 丸め単位（500円・1,000円など）で1人あたりを切り下げると、理論上の端数が集金総額とお会計の差として幹事の過不足表示にまとまります。100円単位では最大剰余法（LRM）でグループ間に端数を分散し、集金総額とお会計を一致させます。それでも調整のすき間が出る場合（例: 1,000円単位でお会計が割り切れない）は、幹事の自腹とは決めつけません。「不足分はどうしましょうか？」と表示し、先輩・上司のご負担や次回への回しなど現場で決める前提です。飲み会は100円単位推奨。

幹事が得をしていると思われませんか？

按分結果を清算文として全員に見える形で共有できます。計算後は集金総額とお会計が一致するかを画面で確認でき（LRM適用時）、根拠をそのまま LINE に貼れます。多めのご負担へのお礼文も添えられます。目的は幹事個人の得ではなく、「幹事の独断」ではなく計算根拠の透明化です。幹事が悪者に見えにくいよう、不足・余りも決めつけない文案にしています。

LINEやSlackに貼れる清算文は作れますか？

計算後に清算テキストが自動生成されます。「ワンクリックコピー」でグループチャットにそのまま貼り付けできます。入力データはサーバーに送信されません。

多めに払う上司・先輩へのお礼も清算文に入れられる？

はい（推奨ON）。傾斜で多くご負担いただくグループへ、幹事から感謝の一文をグループ全員に見える形で添えられます。「端数を払って」ではなく「多めのご負担へのお礼」として共有する運用です。若手幹事が個別に頼むプレッシャーを減らし、先輩・上司の方も公の場で気前よく見えやすくなります。

上司に「いくら出せばいい？」と聞かれたら？

計算後の「個別に聞かれたとき」欄に、グループごとの答えが出ます（例:「役員・上司の方は、お一人 ¥17,500 でちょうどよいです。」）。1名のグループならそのまま上司への返答に使えます。コピーしてDMに貼るだけでOKです。

神精算をやったら、次回も幹事を任され続けますか？

その可能性は高めです。合計が一致する計算根拠をそのまま共有できるため、場の空気を守れる幹事として信頼されやすくなります。
