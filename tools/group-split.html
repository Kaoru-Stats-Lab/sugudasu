班分け · グループ分けのよくある質問

        
SUGUDASU が想定する「班分け」は、研修 · ブレイクアウトの幹事が名簿を公平に分割し、Slack · Excel で共有 · 記録することです。上段は幹事向けの基本、下段は本ページのツール操作です。


        
班分けの基本（幹事 · 人事向け）

        

            

                
                    研修やブレイクアウトで班分けが必要なのはどんなとき？
                    
                
                

                    
新入社員研修のグループワーク、異動者研修、ハッカソンのチーム編成、社内ワークショップの卓割りなど、参加者全員をいくつかの組に分け、事前または当日に共有したい場面です。SUGUDASU の班分けは「その場の演出」より、幹事の事務作業（割当 · 共有 · 説明責任）を助ける位置づけです。

                

            

            

                
                    ランダムに分けると公平になりますか？
                    
                
                

                    
完全なランダムは「恣意的な偏りを避ける」手段として有効ですが、所属の固まりや特定ペアの回避など、現場のルールも必要になります。公平性は「誰が決めたか説明できること」と「ルールを守った結果を記録できること」も含みます。本ツールはシード値と名簿指紋で再現可能にします。

                

            

            

                
                    Excel の RAND() や手作業との違いは？
                    
                
                

                    
RAND() はセル再計算で班構成が変わりうるほか、固定班 · 離すペア · 所属分散などの制約実装が手間です。手作業は小規模なら可能ですが、再現説明と共有用テキストの作成に時間がかかります。班分けツールは「1回決めた結果を TSV/Slack 用にそのまま出す」ことに特化します。

                

            

            

                
                    投影用ルーレットと事務向け班分けの違いは？
                    
                
                

                    
教室やイベント会場向けのルーレット演出は、画面に映して盛り上げる用途向けです。SUGUDASU の班分けは、名簿を非送信で処理し、Excel TSV や Slack 用テキストをコピーして共有する「幹事の2次利用」向けです。どちらも正しい使い方があり、目的が異なります。

                

            

            

                
                    所属や性別など属性を考慮した班分けは？
                    
                
                

                    
「同じ部署が固まらない」「各組に役員を1名以上」などは、氏名だけでなく所属 · 性別 · 役職などの列を使った条件になります。年齢平均の完全一致などは専門ソルバの領域ですが、本ツールでは Excel 名簿の属性列（最大3）で分散 · 各組必須を指定できます。

                

            

            

                
                    当日欠席が出たときはどうする？
                    
                
                

                    
ハッカソン想定（M02 · スマホタップ再編）:

                    
    
                        
  1. 事前（PC） — 実行後 JSONをコピー → Google Keep 等の同期メモに貼る。
    2. 
                        
  2. 当日（スマホ） — 同期メモから JSON をコピーし、結果欄の貼り付け欄へ → 読み込んで再実行。
    3. 
                        
  3. 欠席が出たら — 結果の名前をタップ → 確認 → 名簿から自動削除 · 同じ seed で再構成（制約は維持）。
    4. 
                        
  4. 共有 — 更新後の Slack / TSV をコピーし、スイッチャーへ。Zoom 上の手動移動は別作業。
    5. 
                    

                    
同一端末で結果画面を開いたままなら JSON 不要でタップだけで再編できます。「再シャッフル」は新 seed のため欠席対応には使いません。名簿を手で貼り直して再実行する方法も有効です。

                

            

        


        
SUGUDASU グループ分けツール

        

            

                
                    名簿は外部に送信されますか？
                    
                
                

                    
いいえ。処理はすべてブラウザ内で完結し、参加者名簿の POST は発生しません。

                    
設計思想と但し書き（広告 · 解析等）の詳細は SUGUDASUの約束 をご覧ください。

                

            

            

                
                    使い方の流れは？（Step ①〜④）
                    
                
                

                    
① Excel 名簿を貼り付け → ② 複数列なら属性条件（所属の分散 · 各組必須など）→ ③ 必要なら名前ルール（固定班 · 離すペア · 組番号固定）→ ④ 人数または組数を指定して実行。結果は TSV · Slack · 告知文を1クリックコピーできます。

                

            

            

                
                    属性条件（②）と名前ルール（③）の違いは？
                    
                
                

                    
② は列ベース（例: 所属=営業をばらす · 各組に役員1名）。③ は氏名ベースの例外（この2人は同組 · 別組 · 1組固定）。多くの研修班分けは ② だけで足り、③ は少数の例外用です。

                

            

            

                
                    同じ班構成を再現できますか？
                    
                
                

                    
同じ名簿 · 同じ条件 · 同じシード値なら同じ班構成になります。

                    
    
                        
  1. 復元したいとき — JSONをコピー → Google Keep 等の同期メモ → 会場スマホで貼り付け「読み込んで再実行」（名簿 · 条件 · seed 一式）。
    2. 
                        
  2. 照合したいとき — 手元の名簿を貼り直して実行し、表示された名簿指紋が記録と一致するか確認します（seed だけでは名簿は復元できません）。
    3. 
                        
  3. 記録用 — Slack 等に seed と名簿指紋を残す場合は、セッションJSON または結果 TSVも合わせて保管してください。
    4. 
                    

                    
「再シャッフル」は新しいシードを生成するため、同じ班にはなりません。

                

            

            

                
                    人数が割り切れないときは？
                    
                
                

                    
制約なしの均等分割では、各グループの人数差は最大1名です（例: 10名×4人1組 → 4+3+3）。固定班や属性条件がある場合は、目標定員からずれることがあります（超過理由を表示）。

                

            

            

                
                    目標定員より多いグループになりますか？
                    
                
                

                    
固定班を分割しない · メンター配置など、制約を満たすために目標定員を超える組が出る場合があります。超過組には人数バッジと理由を表示します。

                

            

            

                
                    Excel や Slack にそのまま貼れますか？
                    
                
                

                    
はい。Excel 用 TSV（グループ · 氏名 · 属性列）がデフォルトです。Slack/Teams · LINE · 告知 · JSON も別形式でコピーできます。名簿上限は250名（研修 · ブレイクアウト規模向け）。
