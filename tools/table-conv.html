確認手順: SUGUDASU の約束（DevTools）

Excel の複数列コピー / CSVを、サーバーに送らず Markdown 表 · JSON · CSV · TSV に変換します。

社内規程でオンライン変換サイトに貼れない表も、ブラウザ内だけで処理。Shift-JIS の CSV も FileReader で端末内デコード。

1列リストの整形は 全角半角整え · 請求書の内税分離は 請求書 をご利用ください。

入力

CSV ファイルをドロップ / クリック

最大 5,000 行 · 2MB 目安

1列のみ検出 — 全角半角整え の方が向いている場合があります。

結合セルは左上の値のみ · 数式は表示値のみ · セル内改行は Markdown で <br> に置換します。

出力

表変換のよくある質問

Excel やスプレッドシートの表を Markdown に変換できますか？

できます。Excel や Google スプレッドシートで範囲をコピーするとタブ区切り（TSV）で貼り付けられます。入力欄に貼って「変換」→ Markdown タブをコピーしてください。

出力は最小の GitHub 風 Markdown 表（| col | + 区切り行）のみです。Zenn · Qiita · Notion 固有の拡張記法は出力しません。

CSV と JSON は相互に変換できますか？

Excel 貼付 / CSV ファイルを1回入力すれば、JSON · CSV · TSV · Markdown を出力タブで切り替えられます（同じ表から複数形式を取り出すイメージ）。

Markdown 表を貼って TSV に戻す逆変換は対応しません（列ズレ・方言混在のリスク）。Excel に戻すときは TSV タブの出力をそのまま貼り付けてください。

データは外部に送信されますか？

いいえ。読み込み · 変換 · コピーまですべてお使いのブラウザ内で完結します。機密の社内表でも、サーバーへアップロードされません。

自分で確認するには F12 → Network タブを開き、変換やコピー時に表データを含む POSTが出ないことを見てください。ページ読込 · 解析等の通信は別です。手順は SUGUDASU の約束。TableConvert もブラウザ内処理を謳っていますが、当ツールは日本の Shift-JIS CSV と社内規程向けの入力データ非送信に特化しています。

Shift-JIS の CSV が文字化けします

「Shift-JIS（CSVファイル）」をオンにしてから D&D またはファイル選択で読み込んでください。

UTF-8 の BOM 付き CSV はそのまま読めます。Excel 向けに再出力するときは「CSV 出力に UTF-8 BOM」をオンにすると文字化けしにくくなります。

① · ㈱ など CP932 拡張文字はブラウザ依存で置換文字（）になる場合があります — 警告が出たら元データを確認してください。

TableConvert など他サイトとの違いは？

TableConvert は 30 以上の形式 · 表エディタ · API など多機能です。SUGUDASU 表変換は社内規程で外部変換サイト NGの現場向けに、

  • 表データはサーバーに送信しません · 確認手順は約束ページ
  • Shift-JIS CSV（日本の実務データ）
  • Excel 貼付 → MD / JSON / CSV / TSV の最短ルート

に絞っています。LaTeX · XML · PDF 画像出力 · 会員 API はありません。

変換結果を続けて別の形式に変換できますか？

同じ入力からは出力タブ（Markdown / JSON / CSV / TSV）を切り替えるだけで OK です。

出力を入力に戻して再変換する場合は、出力欄をコピーして入力欄に貼り直してください（NOVEBLO の「→入力へ」に相当する操作。専用ボタンは今後検討）。

1列だけのリスト・名簿は使えますか？

2列以上の表向けです。縦1列の名簿 · ソート · 重複削除は 全角半角整え をご利用ください。

会員登録・課金・オフラインは？

会員登録も課金も不要です。一度ページを読み込めばオフラインでも変換できます（静的 HTML + ブラウザ内 JS）。データの永続保存はしません。

結合セル・数式・色は再現されますか？

いいえ。クリップボードの表示値だけを扱います。結合セルは左上の値のみ、セル内改行は Markdown で <br> に置換します。書式・数式の復元はしません。

対応しないこと（意図的にやらない理由）

本ツールは表データの運搬（Excel/CSV → 素朴な Markdown·JSON·TSV）に絞っています。次は対応しません。

  • Markdown の方言（Zenn の ::: ブロック · Qiita/Obsidian 固有記法など）— サービスごとに解釈が違い、変換しても貼り先で崩れる。サポート範囲が説明不能になり、混乱だけが増えるため。
  • LaTeX · Mermaid · XML · PDF 出力 — 論文・README 図表向け。当ツールのペルソナ（総務 · Web 担当 · Notion 貼付）の Pain ではないため。
  • リッチ表（結合セル復元 · セル内箇条書き · 色・列幅）— クリップボードに載らない情報は復元できない。出力は GitHub 風の | col | 最小形式のみ。
  • Markdown 表の汎用逆変換（MD 貼付 → TSV）— パイプ表でもエスケープ・区切り行の揺れ・方言混在で列ズレしやすい。v1 は非対応（Excel 往復はTSV タブ出力を Excel に貼るで足りる）。
  • 表エディタ · API · 30形式超 — TableConvert 等の領域。非送信・Shift-JIS · 最短ルートに集中するため。

参考: Markdown の方言乱立 — Zenn（plusone）。当ツールは方言に寄せず、壊れにくい最小表だけを出します。一覧は 開発ロードマップ（対象外） も参照してください。

本ツールは一般的な表変換例を提供するものです。税務・会計データの正確性は保証しません。要専門家確認。
