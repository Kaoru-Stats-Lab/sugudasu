進行表の版管理トラブルを防ぐ安全設計

短期バッファによる自動消去

進行表データは永続保存せず短期バッファとして処理。期限（retain_until）経過後はサーバーから物理削除。

※消去後の復旧は不可

行レベルセキュリティ（RLS）

Supabase の RLS を採用。他アカウントからの不正アクセスを構造的に隔離。

※安全性を保証するものではありません

実名データの送信回避設計

サーバー側は ID と属性（時間等）の処理に特化。実名の他端末表示・暗号化連携は設計中。

※E2EE 等の詳細は要確認

よくある質問

コア無料版（sugudasu.com）との違いは何ですか？

1人で進行表を組むコア無料版はデータ非送信です。Sync 有料版は共有 URL を発行し、チーム間で進行表共有とリアルタイム共有を可能にします。

送信した進行表データはいつまでサーバーに保存されますか？

永続保存はされません。イベントごとの期限（retain_until）経過後、バックアップ含め物理削除され、復旧不能になります。

イベント本番中にクレジットカードで支払う必要がありますか？

不要です。事前チケット購入制を採用しています（一部準備中）。当日はチケット消費で同期を開始する想定です。

現在のリアルタイム同期機能（S2）の状況を教えてください。

リアルタイム同期（S2）は設計中です。利用登録と閲覧 URL 発行を行う S1 の稼働準備を進めています。

URLを知っていれば誰でも見られる仕様で、版管理は安全ですか？

共有 URL は複雑な ID で保護されますが、URL の漏洩管理はお客様の責任です。実名データの暗号化通信の詳細は要確認です。
